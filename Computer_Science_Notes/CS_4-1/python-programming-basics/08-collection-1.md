## 2. 콜렉션의 이해

### 2-1. 콜렉션의 정의

- 콜렉션은 여러 데이터를 묶어 하나의 구조로 다루기 위해 제공되는 자료구조의 총칭임
- 데이터 저장 방식과 접근 방식에 따라 다음과 같이 구분함

| 구분   | 자료구조                 |
|------|----------------------|
| 시퀀스  | 리스트(list), 튜플(tuple) |
| 비시퀀스 | 세트(set)              |
| 매핑   | 딕셔너리(dictionary)     |

### 2-2. 콜렉션의 필요성

- 다수의 데이터를 효율적으로 저장·관리할 수 있음
- 알고리즘을 통해 신속한 프로그램 구현이 가능함
- 활용 분야:
    - 데이터 분석
    - 네트워크 통신
    - 파일 처리
    - 다양한 프로그래밍 영역의 기본 도구

### 핵심 정리

- 콜렉션은 여러 데이터를 하나의 단위로 관리하는 자료구조임
- 리스트, 튜플, 세트, 딕셔너리가 대표적인 콜렉션임

---

## 3. 리스트(list)

### 3-1. 리스트의 개념

- 리스트는 데이터를 순서 있게 저장하는 **가변(mutable) 시퀀스**임
- 서로 다른 데이터 타입을 함께 저장할 수 있는 유연한 구조임
- 원소를 추가, 삭제, 수정할 수 있도록 설계됨

### 3-2. 예시

    body = [181, 78, "dark brown", "black"]
    body.append(78)

### 코드 해석

- `body`
    - 여러 데이터를 순서대로 저장한 리스트
- `append(78)`
    - 리스트의 끝에 `78`을 추가함
- 리스트는 인덱스를 통해 각 원소에 접근할 수 있음

### 핵심 정리

- 리스트는 순서가 있고 수정 가능한 자료구조임
- 값 추가, 삭제, 수정이 가능함
- 서로 다른 자료형도 함께 저장할 수 있음

---

## 4. 튜플(tuple)

### 4-1. 튜플의 개념

- 튜플은 데이터를 순서 있게 저장하는 **불변(immutable) 시퀀스**임
- 생성 후 원소가 바뀌지 않도록 설계된 안정적인 저장 구조임
- 불변성으로 인해 의도하지 않은 데이터 변경을 방지할 수 있음
- 리스트보다 메모리 효율과 접근 속도 면에서 유리할 수 있음

### 4-2. 예시

    body = (181, 78, "dark brown", "black")

### 주의

- 튜플은 생성 후 원소 변경이 불가능하므로 다음과 같은 추가 연산은 사용할 수 없음

  body.append(78)

### 핵심 정리

- 튜플은 리스트처럼 순서가 있지만 수정할 수 없음
- 변경되면 안 되는 데이터를 저장할 때 유용함

---

## 5. 세트(set)

### 5-1. 세트의 개념

- 세트는 중복을 허용하지 않고 순서를 유지하지 않는 콜렉션임
- 수학적 집합 개념을 구현한 데이터 구조임
- 원소 간의 고유성(uniqueness)을 보장함
- 중복된 값을 자동으로 제거하므로 데이터 정제에 유용함

### 5-2. 예시

    items = ["apple", "banana", "apple", "orange"]
    unique_items = set(items)

### 코드 해석

- `items`
    - 중복된 `"apple"` 값을 포함한 리스트
- `set(items)`
    - 중복을 제거한 세트 생성
- 결과적으로 `"apple"`, `"banana"`, `"orange"`만 남음

### 핵심 정리

- 세트는 중복 제거에 유용함
- 순서가 중요하지 않고, 고유한 값만 필요할 때 사용함
- 빈 세트는 `{}`가 아니라 `set()`으로 생성하는 것이 안전함

---

## 6. 딕셔너리(dictionary)

### 6-1. 딕셔너리의 개념

- 딕셔너리는 `key-value` 쌍으로 데이터를 저장하는 **매핑(mapping) 자료형**임
- 키에 대응되는 값을 빠르게 검색할 수 있도록 설계된 구조임
- 키는 중복되지 않아야 함
- 값은 어떤 타입의 데이터도 저장할 수 있음

### 6-2. 예시

    body = {
        "키": 181,
        "몸무게": 78,
        "눈": "dark brown",
        "머리": "black"
    }

    print(body["눈"])

### 코드 해석

- `"키"`, `"몸무게"`, `"눈"`, `"머리"`
    - 딕셔너리의 key
- `181`, `78`, `"dark brown"`, `"black"`
    - 각 key에 대응되는 value
- `body["눈"]`
    - key `"눈"`에 해당하는 값인 `"dark brown"`에 접근함

### 핵심 정리

- 딕셔너리는 key를 이용해 value를 찾는 구조임
- 인덱스가 아니라 key로 데이터에 접근함
- 검색이 빠르고 의미 있는 이름으로 데이터를 관리하기 좋음

---

## 7. 콜렉션 정리

| 데이터 구조       | 순서 | 변경  | 생성              | 예시                              |
|--------------|----|-----|-----------------|---------------------------------|
| `list`       | 있음 | 가능  | `[]`, `list()`  | `[181, 78, "dark brown"]`       |
| `tuple`      | 있음 | 불가능 | `()`, `tuple()` | `(181, 78, "dark brown")`       |
| `set`        | 없음 | 가능  | `{값}`, `set()`  | `{181, 78, "dark brown"}`       |
| `dictionary` | 있음 | 가능  | `{}`, `dict()`  | `{"키": 181, "눈": "dark brown"}` |

### 핵심 정리

- 리스트와 튜플은 순서가 있는 시퀀스 자료형임
- 리스트는 수정 가능, 튜플은 수정 불가능함
- 세트는 중복을 제거하고 순서를 유지하지 않음
- 딕셔너리는 key-value 구조로 데이터를 저장함

## 8. 리스트의 활용

### 8-1. 리스트 원소의 접근

- 리스트 원소에 접근할 때는 접근 연산자 `[]`와 인덱스 번호를 사용함
- 양수 인덱스는 왼쪽부터 접근함
- 음수 인덱스는 오른쪽부터 접근함
- 인덱스 번호는 0부터 시작함

### 8-2. 예시

    hei_list = [1, 5, 14, 26, 31]

    print(hei_list[0])
    print(hei_list[2])
    print(hei_list[-1])

### 코드 해석

- `hei_list[0]`
    - 첫 번째 원소 `1`
- `hei_list[2]`
    - 세 번째 원소 `14`
- `hei_list[-1]`
    - 마지막 원소 `31`

### 핵심 정리

- 리스트의 첫 번째 원소는 인덱스 `0`임
- 마지막 원소는 `-1`로 접근할 수 있음

---

## 9. 리스트 관련 연산자

### 9-1. 주요 연산자

| 연산자      | 사용 예         | 의미                            |
|----------|--------------|-------------------------------|
| `in`     | `x in s`     | 원소 `x`가 시퀀스 `s`에 포함되어 있는지 확인  |
| `not in` | `x not in s` | 원소 `x`가 시퀀스 `s`에 포함되지 않았는지 확인 |
| `+`      | `s1 + s2`    | 두 시퀀스를 연결                     |
| `*`      | `s * n`      | 시퀀스 `s`를 `n`번 복사하여 연결         |
| 비교 연산자   | `s < t` 등    | 두 시퀀스를 사전식으로 비교               |

### 9-2. 예시

    hei1 = [1, 5, 14, 26, 31]
    hei2 = [1, 7, 45]

    print(26 in hei1)
    print(hei1 + hei2)
    print(hei2 * 3)
    print(hei1 < hei2)

### 코드 해석

- `26 in hei1`
    - `26`이 `hei1` 안에 있는지 확인함
- `hei1 + hei2`
    - 두 리스트를 이어 붙임
- `hei2 * 3`
    - `hei2`를 3번 반복하여 연결함
- `hei1 < hei2`
    - 두 리스트를 앞 원소부터 사전식으로 비교함

### 핵심 정리

- 리스트는 포함 여부 확인, 연결, 반복, 비교 연산이 가능함

## 10. 리스트 관련 내장 함수

### 10-1. 주요 내장 함수

| 함수       | 의미               |
|----------|------------------|
| `len(s)` | `s`의 길이, 즉 원소 개수 |
| `min(s)` | `s`에서 가장 작은 원소   |
| `max(s)` | `s`에서 가장 큰 원소    |
| `sum(s)` | `s`의 모든 원소의 합    |

### 10-2. 예시

    hei1 = [1, 5, 14, 26, 31]

    print(len(hei1))
    print(min(hei1))
    print(max(hei1))
    print(sum(hei1))

### 핵심 정리

- 리스트의 길이, 최솟값, 최댓값, 합계는 내장 함수로 쉽게 구할 수 있음
- `sum()`은 숫자 데이터로 구성된 리스트에서 사용함

## 11. 리스트 메소드

### 11-1. 메소드의 개념

- 메소드는 객체지향 프로그래밍 패러다임의 기능임
- 메소드는 클래스 내부에 정의된 함수임
- 해당 객체의 데이터에 접근하여 특정 작업을 수행하는 명령문의 집합임

### 11-2. 메소드 사용 형식

    객체의이름.메소드이름(입력값)

### 11-3. 예시

    hei_list = [1, 5, 14, 26, 31]
    hei_list.append(27)

### 코드 해석

- `hei_list`
    - 객체의 이름
- `append`
    - 메소드 이름
- `27`
    - 입력값 또는 파라미터
- `append(27)`
    - 리스트 끝에 `27`을 추가함

---

## 12. 리스트 주요 메소드

| 메소드                | 의미                        |
|--------------------|---------------------------|
| `append(x)`        | 원소 `x`를 리스트의 끝에 추가        |
| `count(x)`         | 리스트 내 원소 `x`의 출현 횟수를 반환   |
| `index(x)`         | 원소 `x`가 처음 나온 위치의 인덱스를 반환 |
| `pop(i)`           | 주어진 위치 `i`의 원소를 반환하고 제거   |
| `insert(index, x)` | 원소 `x`를 주어진 인덱스 위치에 삽입    |
| `remove(x)`        | 리스트에서 원소 `x`를 제거          |
| `reverse()`        | 리스트 내 원소들의 순서를 역순으로 변경    |
| `sort()`           | 리스트 내 원소들을 오름차순으로 정렬      |

### 예시

    hei_list = [1, 5, 14, 26, 31]

    hei_list.append(27)
    hei_list.insert(1, 3)
    hei_list.remove(14)
    hei_list.sort()

### 핵심 정리

- 리스트 메소드는 리스트 객체의 내용을 직접 변경하는 경우가 많음
- `append()`, `insert()`, `remove()`, `sort()`는 시험에 자주 나올 수 있음

## 13. 리스트의 조작

### 13-1. 리스트 슬라이싱의 개념

- 리스트 슬라이싱은 대상 리스트의 부분 리스트를 생성하는 방법임
- 원하는 범위의 원소를 잘라 새로운 리스트처럼 사용할 수 있음

### 13-2. 구문 형식

    리스트[시작인덱스:끝인덱스:간격]

### 13-3. 예시 1

    hei_list = [1, 5, 14, 26, 31]

    sub_hei = hei_list[:3]
    sub_hei2 = hei_list[-3:]

### 코드 해석

- `hei_list[:3]`
    - 처음부터 인덱스 3 전까지 추출
    - 결과: `[1, 5, 14]`
- `hei_list[-3:]`
    - 뒤에서 세 번째 원소부터 끝까지 추출
    - 결과: `[14, 26, 31]`

### 핵심 정리

- 슬라이싱에서 끝 인덱스는 포함되지 않음
- 시작 인덱스를 생략하면 처음부터 시작함
- 끝 인덱스를 생략하면 마지막까지 선택함

---

## 14. 리스트 슬라이싱 응용

### 14-1. 간격 사용

    hei_list = [1, 5, 14, 26, 31]

    sub_hei1 = hei_list[::2]
    sub_hei2 = hei_list[::-1]

### 코드 해석

- `hei_list[::2]`
    - 처음부터 끝까지 2칸 간격으로 선택
    - 결과: `[1, 14, 31]`
- `hei_list[::-1]`
    - 리스트를 역순으로 선택
    - 결과: `[31, 26, 14, 5, 1]`

### 핵심 정리

- 세 번째 값은 간격을 의미함
- `[::-1]`은 리스트를 역순으로 가져올 때 자주 사용됨

## 15. 리스트 컴프리헨션

### 15-1. 리스트 컴프리헨션의 개념

- 리스트를 간결하게 생성하고 조작하는 문법임
- 반복문과 조건문을 한 줄에 표현할 수 있음

### 15-2. 구문 형식

    [표현식 for 변수 in 리스트]

    [표현식 for 변수 in 리스트 if 조건]

### 15-3. 특징

- 코드가 짧아지고 가독성이 향상됨
- 파이썬다운 코드 작성 방식임
- 내부 최적화로 일반 `for`문보다 빠른 속도를 기대할 수 있음
- 필터링으로 필요한 데이터만 추출 가능함

---

## 16. 리스트 컴프리헨션을 이용한 리스트 생성

### 16-1. 일반 반복문 방식

    hei_list = []

    for i in [1, 5, 14, 26, 31]:
        hei_list.append(i)

### 16-2. 리스트 컴프리헨션 방식

    hei_list = [i for i in [1, 5, 14, 26, 31]]

### 핵심 정리

- 리스트 컴프리헨션은 반복문을 이용한 리스트 생성을 한 줄로 표현할 수 있음
- 간단한 리스트 생성에는 리스트 컴프리헨션이 효율적임

---

## 17. 리스트 컴프리헨션 사용

### 17-1. 변환

    hei_list = [1, 5, 14, 26, 31]

    hei_square = [h ** 2 for h in hei_list]
    hei_len = [len(str(h)) for h in hei_list]

### 코드 해석

- `hei_square`
    - 각 원소를 제곱한 리스트
- `hei_len`
    - 각 원소를 문자열로 바꾼 뒤 글자 수를 저장한 리스트

### 17-2. 조건 적용

    hei_list = [1, 5, 14, 26, 31]

    hei_even = [h for h in hei_list if h % 2 == 0]
    hei_hi = [h for h in hei_list if h > 20]

### 코드 해석

- `hei_even`
    - 짝수 원소만 추출
    - 결과: `[14, 26]`
- `hei_hi`
    - 20보다 큰 원소만 추출
    - 결과: `[26, 31]`

### 핵심 정리

- 리스트 컴프리헨션은 원소 변환과 조건 필터링에 유용함
- `[표현식 for 변수 in 리스트 if 조건]` 형식을 기억하면 됨

## 18. 핵심 요약

### 18-1. 콜렉션

- 여러 데이터를 하나의 구조로 묶어 다루는 자료구조의 총칭임
- 대표 콜렉션:
    - 리스트
    - 튜플
    - 세트
    - 딕셔너리

### 18-2. 콜렉션 비교

| 자료구조 | 순서 | 변경 가능 | 중복        | 접근 방식 |
|------|----|-------|-----------|-------|
| 리스트  | 있음 | 가능    | 가능        | 인덱스   |
| 튜플   | 있음 | 불가능   | 가능        | 인덱스   |
| 세트   | 없음 | 가능    | 불가능       | 원소 자체 |
| 딕셔너리 | 있음 | 가능    | key 중복 불가 | key   |

### 18-3. 리스트

- 순서가 있는 가변 시퀀스임
- 대괄호 `[]`로 생성함
- 원소는 콤마로 구분함
- 인덱스는 0부터 시작함
- 양수 인덱스는 왼쪽부터, 음수 인덱스는 오른쪽부터 접근함

### 18-4. 리스트 활용

- 포함 여부 확인:
    - `in`
    - `not in`
- 연결:
    - `+`
- 반복:
    - `*`
- 내장 함수:
    - `len()`
    - `min()`
    - `max()`
    - `sum()`
- 주요 메소드:
    - `append()`
    - `count()`
    - `index()`
    - `pop()`
    - `insert()`
    - `remove()`
    - `reverse()`
    - `sort()`

### 18-5. 리스트 조작

- 슬라이싱:
    - `리스트[시작:끝:간격]`
- 리스트 컴프리헨션:
    - `[표현식 for 변수 in 리스트]`
    - `[표현식 for 변수 in 리스트 if 조건]`

- 딕셔너리는 key를 이용해 value에 빠르게 접근할 수 있음

### Q2. 다음 코드의 실행 결과로 옳은 것은?

    hei_list = [1, 5, 14, 26, 31]
    print(hei_list[:3])

1. `[1, 5, 14]`
2. `[14, 26, 31]`
3. `[1, 14, 31]`
4. `[31, 26, 14]`

- 정답: **1**

### 해설

- `hei_list[:3]`은 처음부터 인덱스 3 전까지 선택함
- 인덱스 3의 값 `26`은 포함되지 않음
- 따라서 결과는 `[1, 5, 14]`임

---

### Q3. 다음 코드의 출력값으로 옳은 것은?

    hei_list = [1, 5, 14, 26, 31]
    hei_even = [h for h in hei_list if h % 2 == 0]
    print(hei_even)

1. `[1, 5, 31]`
2. `[14, 26]`
3. `[1, 14, 31]`
4. `[26, 31]`

- 정답: **2**

### 해설

- `h % 2 == 0`은 짝수 조건임
- `hei_list`에서 짝수는 `14`, `26`임
- 따라서 출력 결과는 `[14, 26]`임

## 20. 시험 직전 체크포인트

- 콜렉션은 여러 데이터를 하나의 구조로 묶어 관리하는 자료구조이다.
- 리스트와 튜플은 시퀀스 자료형이다.
- 리스트는 변경 가능하고, 튜플은 변경 불가능하다.
- 세트는 중복을 허용하지 않고 순서를 유지하지 않는다.
- 딕셔너리는 key-value 쌍으로 데이터를 저장한다.
- 리스트 인덱스는 0부터 시작한다.
- 음수 인덱스는 오른쪽부터 접근한다.
- `in`은 원소 포함 여부를 확인한다.
- `+`는 리스트를 연결하고, `*`는 리스트를 반복한다.
- `len()`은 원소 개수, `min()`은 최솟값, `max()`는 최댓값, `sum()`은 합계를 구한다.
- `append()`는 리스트 끝에 원소를 추가한다.
- `pop(i)`는 인덱스 `i`의 원소를 반환하고 제거한다.
- 슬라이싱에서 끝 인덱스는 포함되지 않는다.
- `리스트[::-1]`은 리스트를 역순으로 가져온다.
- 리스트 컴프리헨션은 리스트를 간결하게 생성하거나 필터링할 때 사용한다.
