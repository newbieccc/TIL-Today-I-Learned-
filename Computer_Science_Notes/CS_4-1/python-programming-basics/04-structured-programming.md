## 2. 구조적 프로그래밍 패러다임

### 2-1. 명령형 프로그래밍 패러다임

- 1950~60년대 확립된 패러다임
    - Algol
    - Fortran
- 컴퓨터에게 무엇을 하라고 순서대로 지시하는 방식
- 프로그램은 **상태(state)를 가지는 시스템**으로 볼 수 있음
- 코드는 그 상태를 단계적으로 바꾸는 **명령들의 연속**임
- 할당문은 상태 변화를 일으키는 기본 수단임
- 프로그램은 연속된 명령문의 실행을 통해
  **초기 상태에서 목표 상태로 진전**함

### 2-2. `goto`의 문제

- `goto`는 프로그램의 흐름에 대한 **직접 통제**를 어렵게 만듦
- 프로그램의 실행 흐름이 복잡해지면
  이해와 검증이 어려워짐
- 모든 프로그램은 `goto`가 없는 프로그램으로 표현 가능함
- 이에 따라 절차적, 구조적 프로그래밍 패러다임으로 발전함

### 2-3. 제품 생산 공정과 프로그래밍 구조

- 프로그래밍 패러다임은 제품 생산 공정처럼 표현할 수 있음
- 제품 생산 공정은 보통 다음 흐름을 가짐

    - 설계
    - 생산
    - 조립
    - 검사
    - 정상/불량 판단
    - 배송 또는 재생

- 이 흐름은 프로그램의 기본 제어 구조와 연결됨

    - 순차 구조
    - 선택 구조
    - 반복 구조

### 2-4. 구조적 프로그래밍 패러다임

- 구조적 프로그래밍 패러다임은 **명령형 프로그래밍 패러다임의 하위 개념**임
- `goto` 문을 사용하지 않고 프로그램을 다음 3가지 제어 흐름만으로 구성함

    1. 순차(sequence) 구조
    2. 선택(selection) 구조
    3. 반복(iteration) 구조

- 제어 흐름을 블록 단위로 계층화하고 지역화함
- 프로그램 실행 흐름이 간결하고 작은 규모로 조직화되기 쉬움
- 프로그램의 예측 가능성과 검증 가능성을 보장함

### 핵심 정리

- 명령형 프로그래밍은 명령을 순서대로 실행하여 상태를 바꾸는 방식임
- 구조적 프로그래밍은 `goto` 없이
  순차, 선택, 반복 구조만으로 프로그램을 구성하는 방식임
- 구조적 프로그래밍은 코드의 흐름을 명확하게 만들어
  이해, 디버깅, 검증을 쉽게 함

## 3. 순차 구조의 이해

### 3-1. 순차 구조의 정의

- 순차 구조는 실행의 흐름을
  **주어지는 명령의 위치적 흐름에 따라 수행하는 구조**임
- 명령 라인 위에서 아래로 흐르는 가장 직관적인 구조임
- 사람이 이해하고 디버깅하기 쉬움

### 3-2. 순차 구조 예시

- 제품 생산 공정에서 다음과 같은 흐름은 순차 구조와 비슷함

    1. 설계
    2. 생산
    3. 조립

- 프로그램에서도 위에서 아래로 명령을 차례대로 실행하는 방식이 순차 구조임

## 4. 원뿔의 부피와 겉넓이 계산 프로그램

### 4-1. 기능

- 사용자로부터 반지름과 높이를 입력받고
  원뿔의 부피와 겉넓이를 계산하는 프로그램

### 4-2. 공식

- 부피 = 1/3 · π · r² · h
- 겉넓이 = π · r² + π · r · √(r² + h²)

### 4-3. 예제 코드

    # 원뿔 부피 및 겉넓이 계산 프로그램
    rad = 10
    hei = 24

    sla = pow(rad ** 2 + hei ** 2, 0.5)

    # 부피 & 겉넓이 출력
    print(1 / 3 * 3.14 * rad ** 2 * hei)
    print(3.14 * rad ** 2 + 3.14 * rad * sla)

### 4-4. 코드 해석

- `rad`
    - 반지름 값
- `hei`
    - 높이 값
- `sla`
    - 원뿔의 빗변 길이를 저장하는 변수
- `pow(rad ** 2 + hei ** 2, 0.5)`
    - 제곱근을 계산하는 표현
- `print()`
    - 계산 결과를 화면에 출력함

## 5. 사용자 입력

### 5-1. `input()` 함수

- `input()`은 사용자로부터 데이터를 입력받는 내장 함수임
- 키보드 입력을 한 줄의 **문자열**로 읽어 반환함
- `input()` 함수의 파라미터는 입력을 안내하기 위한 **프롬프트** 역할을 함

### 5-2. 예시

    rad = input("반지름을 입력하세요: ")

### 5-3. 코드 해석

- `"반지름을 입력하세요: "`
    - 사용자에게 입력을 안내하는 문구
- 사용자가 값을 입력하면
  그 값이 `rad` 변수에 저장됨
- 단, `input()`의 결과는 기본적으로 문자열 타입임

### 핵심 정리

- `input()`은 사용자 입력을 받는 함수임
- 입력값은 문자열로 반환됨
- 숫자 계산에 사용하려면 데이터 타입 변환이 필요함

## 6. 프로그램 오류

### 6-1. 프로그램 오류의 의미

- 프로그램 오류는 설계 결함, 논리적 실수, 문법 오류 등으로
  프로그램이 의도한 대로 결과를 생성하지 못하는 상황임

### 6-2. 구문 오류

- 구문 오류(syntax error)는 문법 체계에 적합하지 않은 명령문을 입력할 때 발생함
- 프로그램이 실행되기 전,
  인터프리터 또는 컴파일러가 탐지함

### 6-3. 실행 오류

- 실행 오류(runtime error)는 프로그램이 실행되지만,
  실행 중 잘못된 상황이 발생하여 프로그램이 중단되는 오류임

### 6-4. 의미 오류

- 의미 오류(semantic error)는 프로그램의 의도와 다른 잘못된 결과를 생성하는 오류임
- 논리적 또는 의미적으로 잘못 해석되는 명령문 작성 시 발생함
- 프로그램이 실행은 되지만 결과가 기대와 다를 수 있음

### 핵심 정리

| 오류 종류 | 발생 시점        | 설명                   |
|-------|--------------|----------------------|
| 구문 오류 | 실행 전         | 문법에 맞지 않아 실행 전 탐지    |
| 실행 오류 | 실행 중         | 실행 중 잘못된 상황으로 중단     |
| 의미 오류 | 실행 후 결과 확인 시 | 실행은 되지만 의도와 다른 결과 생성 |

## 7. 데이터 타입 변환

### 7-1. 데이터 타입 변환의 의미

- 한 데이터를 다른 데이터 타입의 데이터로 변환하는 것

### 7-2. 데이터 타입 변환 내장 함수

| 함수        | 설명          |
|-----------|-------------|
| `str()`   | 문자열 타입으로 변환 |
| `int()`   | 정수 타입으로 변환  |
| `float()` | 소수 타입으로 변환  |

### 7-3. 예시

    rad = 30

    rad = int(input("반지름을 입력하세요: "))

### 7-4. 코드 해석

- `input()`은 문자열을 반환함
- 숫자 계산을 위해 `int()`로 정수 타입 변환을 수행함
- 사용자가 입력한 값을 계산에 사용하려면
  `int()` 또는 `float()` 변환이 필요함

### 핵심 정리

- `input()` 결과는 문자열임
- 숫자 계산을 하려면 반드시 타입 변환을 고려해야 함

## 8. `print()` 함수의 확장

### 8-1. 여러 개의 데이터 출력

- `print()` 함수는 여러 개의 데이터를 단일 함수로 출력할 수 있음
- 파라미터로 데이터를 콤마(`,`)로 구분하여 입력함

### 8-2. 기본 출력 방식

    print(vol, suf)

- 여러 데이터를 콤마로 구분하면 데이터 사이에 공백이 자동으로 추가됨

### 8-3. `sep` 옵션

- `sep` 옵션을 변경하여 데이터 사이의 공백을 조정할 수 있음
- 기본값은 공백임

### 8-4. 예시

    print("원뿔의 부피는", vol, "입니다.", sep="")

### 8-5. 코드 해석

- `sep=""`
    - 데이터 사이에 아무 문자도 넣지 않고 붙여서 출력함
- 즉, 여러 데이터를 원하는 형식으로 이어 붙여 출력할 수 있음

### 핵심 정리

- `print()`는 여러 데이터를 한 번에 출력 가능함
- 콤마로 구분하면 기본적으로 공백이 들어감
- `sep` 옵션으로 데이터 사이의 구분 문자를 바꿀 수 있음

## 9. 핵심 요약

### 9-1. 구조적 프로그래밍

- `goto` 없이 프로그램을 다음 3가지 구조로 표현하는 패러다임임

    2. 선택 구조
    3. 반복 구조

- 프로그램 흐름을 단순하고 명확하게 만들어
  예측 가능성과 검증 가능성을 높임

### 9-2. 순차 구조

- 위에서 아래로 명령문이 차례대로 실행되는 구조임
- 가장 직관적인 구조이며 이해와 디버깅이 쉬움

### 9-3. 사용자 입력

- `input()`은 사용자 입력을 문자열로 반환함
- 숫자 계산을 하려면 `int()` 또는 `float()`로 변환해야 함

### 9-4. 데이터 타입 변환

- `str()`
    - 문자열 변환
- `int()`
    - 정수 변환
- `float()`
    - 소수 변환

### 9-5. 프로그램 오류

- 구문 오류
    - 문법 오류
- 실행 오류
    - 실행 중 발생하는 오류
- 의미 오류
    - 의도와 다른 결과를 만드는 오류

### 9-6. `print()` 확장

- 여러 데이터를 콤마로 구분하여 출력 가능함
- 기본적으로 데이터 사이에 공백이 들어감
- `sep` 옵션으로 공백 또는 구분 문자를 바꿀 수 있음

## 10. 연습문제 정리

### Q1. 다음은 어떤 제어구조에 대한 설명인가?

> 특정 영역 내의 명령문에 대한 실행 여부를 프로그램 실행 과정 중 조건에 따라 결정하는 구조

1. 선택 구조
2. 반복 구조
3. 순차 구조
4. 명령 구조

- 정답: **1**

### 해설

- 조건에 따라 실행 여부를 결정하는 구조는 **선택 구조**임
- `if`문처럼 조건이 참인지 거짓인지에 따라 실행 흐름이 달라지는 구조에 해당함

### Q2. `number` 변수에 저장되어 있는 정수를 사용하여

`상자에사과가n개들어있습니다.`라는 메시지를 출력하려고 한다.  
올바른 명령문은?

- 정답: **4**

### 정답 코드

    print("상자에", "사과가", number, "개", "들어있습니다.", sep="")

### 해설

- 여러 데이터를 `print()` 함수에 콤마로 구분하여 전달할 수 있음
- 기본적으로 콤마 사이에는 공백이 들어감
- 문제의 출력은 공백 없이 붙어서 출력되어야 하므로 `sep=""` 옵션을 사용해야 함

### Q3. 논리적으로 실행 불가능한 명령문 작성 시 발생 오류를 무엇이라고 하는가?

1. 의미 오류
2. 실행 오류
3. 구문 오류
4. 구조 오류

- 정답: **2**

### 해설

- 논리적으로 실행 불가능한 명령문은 프로그램 실행 중 문제가 발생하게 함
- 따라서 실행 중 발생하는 오류인 **실행 오류(runtime error)**에 해당함

## 11. 시험 직전 체크포인트

- 구조적 프로그래밍은 `goto` 없이 순차, 선택, 반복 구조로 프로그램을 구성한다.
- 순차 구조는 명령문이 위에서 아래로 차례대로 실행되는 구조이다.
- 선택 구조는 조건에 따라 특정 명령문 실행 여부를 결정한다.
- `input()` 함수는 사용자 입력을 문자열로 반환한다.
- 숫자 계산을 하려면 `int()` 또는 `float()`로 변환해야 한다.
- 구문 오류는 실행 전에 문법 문제로 발생한다.
- 실행 오류는 실행 중 잘못된 상황으로 발생한다.
- 의미 오류는 실행은 되지만 의도와 다른 결과가 나오는 오류이다.
- `print()`는 여러 데이터를 콤마로 출력할 수 있다.
- `sep=""`를 사용하면 데이터 사이의 공백 없이 붙여 출력할 수 있다.
