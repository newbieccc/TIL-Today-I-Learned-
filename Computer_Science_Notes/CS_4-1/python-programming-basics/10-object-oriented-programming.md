## 2. 객체지향의 이해

### 2-1. 객체지향의 개념

- 객체지향 프로그래밍은 **객체와 객체 사이의 상호작용**으로 프로그램을 구성하는 프로그래밍 패러다임임
- 프로그램을 유연하고 변경하기 쉽게 만들어 대규모 소프트웨어 개발에 사용됨

### 2-2. 객체지향 패러다임의 특징

| 특징  | 의미                   |
|-----|----------------------|
| 추상화 | 공통의 속성이나 기능을 도출      |
| 캡슐화 | 데이터 구조와 데이터의 연산을 결합  |
| 상속  | 상위 개념의 특징이 하위 개념에 전달 |
| 다형성 | 유사 객체의 사용성을 그대로 유지   |

### 핵심 정리

- 객체지향은 프로그램을 객체 단위로 나누고,
  객체들이 서로 상호작용하도록 구성하는 방식임
- 유지보수와 확장이 쉬워 대규모 프로그램 개발에 적합함

## 3. 클래스 정의

### 3-1. 클래스의 구문 형식

    class 클래스이름:
        초기자 정의
        메소드 정의

### 3-2. 메소드

- 메소드(method)는 객체에 대한 행동 또는 연산을 정의함
- 함수의 정의 및 사용 방법과 거의 동일함
- 클래스 내부에 정의된 함수라고 볼 수 있음

### 3-3. 초기자

- 초기자(initializer)는 객체의 상태를 초기화하는 특수 메소드임
- 항상 `__init__`으로 명명함
- 객체가 생성될 때 자동으로 호출됨

### 핵심 정리

- 클래스는 객체를 만들기 위한 설계도임
- 클래스 안에는 객체의 상태를 초기화하는 `__init__`과 객체의 동작을 정의하는 메소드가 포함됨

---

## 4. 메소드의 정의

### 4-1. 구문 형식

    class 클래스이름:
        def 메소드이름(self, 매개변수 리스트):
            명령 블록

### 4-2. `self` 매개변수

- `self`는 모든 메소드의 첫 번째 매개변수임
- 메소드의 구현에는 사용되지만, 메소드 호출 시에는 직접 전달하지 않음
- 객체 자신을 참조하여 클래스 정의에 포함된 멤버에 접근할 때 사용함

### 핵심 정리

- 클래스 안의 메소드는 첫 번째 매개변수로 `self`를 가져야 함
- `self`는 현재 객체 자신을 의미함
- 객체의 데이터 필드나 다른 메소드에 접근할 때 `self.이름` 형식을 사용함

## 5. `self` 매개변수의 역할

### 5-1. 원뿔 클래스 예시

    # 원뿔 클래스 정의
    class Cone:
        def __init__(self, radius=20, height=30):
            self.r = radius
            self.h = height

        def get_vol(self):
            return 1 / 3 * 3.14 * self.r ** 2 * self.h

        def get_surf(self):
            sla = pow(self.r ** 2 + self.h ** 2, 0.5)
            return 3.14 * self.r ** 2 + 3.14 * self.r * sla

### 5-2. 코드 해석

- `class Cone:`
    - 원뿔을 표현하는 클래스
- `__init__(self, radius=20, height=30)`
    - 반지름과 높이의 기본값을 가지고 객체를 초기화함
- `self.r`
    - 객체의 반지름 데이터 필드
- `self.h`
    - 객체의 높이 데이터 필드
- `get_vol(self)`
    - 원뿔의 부피를 계산하여 반환하는 메소드
- `get_surf(self)`
    - 원뿔의 겉넓이를 계산하여 반환하는 메소드

### 핵심 정리

- `self.r`, `self.h`처럼 작성해야 객체 전체에서 사용할 수 있는 데이터 필드가 됨
- 메소드 내부에서 객체의 데이터 필드에 접근할 때는 `self.`를 붙여야 함

## 6. 클래스 설계

### 6-1. UML 클래스 다이어그램

- UML 클래스 다이어그램을 통해 다음 요소의 표현 방법을 표준화할 수 있음
    - 데이터 필드
    - 생성자
    - 메소드

### 6-2. 표현 형식

| 구분     | 표현 형식                          |
|--------|--------------------------------|
| 클래스 이름 | `클래스이름(매개변수이름: 매개변수타입)`        |
| 데이터 필드 | `데이터필드이름: 데이터필드타입`             |
| 메소드    | `메소드이름(매개변수이름: 매개변수타입): 반환값타입` |

### 6-3. `Cone` 클래스 설계 예시

| 구성     | 내용                                      |
|--------|-----------------------------------------|
| 클래스    | `Cone`                                  |
| 데이터 필드 | `r: int`, `h: int`                      |
| 생성자    | `Cone(radius=20: int, height=30: int)`  |
| 메소드    | `get_vol(): float`, `get_surf(): float` |

### 핵심 정리

- 클래스 설계 시 데이터 필드, 생성자, 메소드를 구분해 정리하면 구조를 이해하기 쉬움
- UML은 클래스를 설계하고 설명할 때 유용한 표기법임

## 7. BMI 계산 프로그램

### 7-1. BMI 클래스 설계

- 이름, 나이, 몸무게, 키를 사용하여 BMI 수치와 상태를 반환하는 BMI 클래스를 정의함

| 구성     | 내용                                                        |
|--------|-----------------------------------------------------------|
| 클래스    | `BMI`                                                     |
| 데이터 필드 | `name: str`, `age: int`, `weight: float`, `height: float` |
| 생성자    | `BMI(name: str, age: int, weight: float, height: float)`  |
| 메소드    | `get_BMI(): float`, `get_status(): str`                   |

### 7-2. BMI 클래스 예시

    class BMI:
        def __init__(self, name, age, weight, height):
            self.name = name
            self.age = age
            self.weight = weight
            self.height = height

        def get_BMI(self):
            self.BMI = self.weight / ((self.height / 100) ** 2)
            return self.BMI

        def get_status(self):
            if self.BMI >= 25:
                return "비만"
            elif self.BMI >= 23 and self.BMI < 25:
                return "과체중"
            elif self.BMI >= 18.5 and self.BMI < 23:
                return "정상"
            else:
                return "저체중"

    gd = BMI("홍길동", 41, 81, 180)

    print(gd.name + "님의 나이", gd.age, "세의 BMI 수치는",
          gd.get_BMI(), gd.get_status())

### 7-3. 코드 해석

- `BMI("홍길동", 41, 81, 180)`
    - 이름, 나이, 몸무게, 키를 전달하여 BMI 객체 생성
- `get_BMI()`
    - BMI 수치를 계산하여 반환
- `get_status()`
    - BMI 수치에 따라 상태를 문자열로 반환
- `gd.name`, `gd.age`
    - 객체의 데이터 필드에 접근

### 핵심 정리

- 클래스는 데이터와 기능을 함께 묶어 표현할 수 있음
- BMI 예제는 사람의 정보와 BMI 계산 기능을 하나의 객체로 묶은 사례임

## 8. 클래스 정의 시 유의사항

### 8-1. 초기화

- 맨 처음에는 무조건 객체 상태를 초기화해야 함
- 초기화 메소드는 `__init__`으로 명명함

### 8-2. `self` 포함

- 메소드의 첫 번째 매개변수는 반드시 `self`를 포함해야 함
- 객체 전체를 아우를 수 있는 변수를 설정하려면 `self.`를 앞에 포함함

### 8-3. 메소드 내부에서 데이터 필드 사용

- `__init__`에서 정의한 변수를 다른 메소드에서 사용하려면 `self.`를 포함해야 함

### 예시

    class Cone:
        def __init__(self, radius=20, height=30):
            self.r = radius
            self.h = height

        def get_vol(self):
            return 1 / 3 * 3.14 * self.r ** 2 * self.h

### 핵심 정리

- 객체의 데이터 필드는 `self.데이터필드` 형식으로 정의함
- `self`를 붙이지 않으면 지역변수처럼 처리되어 다른 메소드에서 사용할 수 없음

## 9. 클래스와 인스턴스

### 9-1. 값의 할당

### 구문 형식

    클래스이름(초기자 파라미터)

### 9-2. 생성자와 인스턴스

- 클래스의 생성자(constructor)를 통해 클래스의 인스턴스를 생성함
- 객체와 인스턴스는 같은 개념으로 이해할 수 있음
- 생성자는 클래스 이름과 초기자의 매개변수를 사용하여 호출함

### 9-3. 객체 생성 과정

    unitcone = Cone(20, 30)

### 동작 과정

1. `Cone(20, 30)` 호출
2. 클래스에 해당하는 객체 생성
3. `__init__()` 자동 호출
4. 반지름과 높이 값이 객체의 데이터 필드에 저장됨
5. 생성된 객체가 `unitcone` 변수에 연결됨

### 핵심 정리

- 클래스는 설계도이고, 인스턴스는 그 설계도로 만들어진 실제 객체임
- 객체 생성 시 `__init__()`이 자동으로 호출되어 상태를 초기화함

## 10. 객체의 사용

### 10-1. 객체 접근

- 객체의 데이터 필드 접근 및 메소드 호출에는 객체 멤버 접근 연산자 `.`을 사용함

### 구문 형식

    객체참조변수.데이터필드

    객체참조변수.메소드(파라미터)

### 10-2. 객체 참조변수 생성

    객체참조변수 = 클래스이름(초기자 파라미터)

### 예시

    firstCone = Cone(20, 30)

    print(firstCone.r)
    print(firstCone.get_vol())

### 코드 해석

- `firstCone`
    - 생성된 객체를 참조하는 변수
- `firstCone.r`
    - 객체의 반지름 데이터 필드 접근
- `firstCone.get_vol()`
    - 객체의 부피 계산 메소드 호출

### 핵심 정리

- 객체는 참조변수를 통해 사용함
- 객체의 필드나 메소드는 `객체명.멤버명` 형식으로 접근함

## 11. 객체지향의 활용

### 11-1. 데이터 타입과 객체

- 파이썬의 기본 데이터 타입도 객체임
- 문자열, 정수 등도 객체로 취급됨

### 예시

- `id()`
    - 객체의 고유 식별값을 확인함

### 핵심 정리

- 파이썬에서는 정수, 문자열 같은 기본 데이터도 객체임
- 객체는 타입과 고유한 식별값을 가짐

## 12. 데이터 필드 감추기

### 12-1. 데이터 은닉

- 데이터 은닉(data hiding)은 데이터 필드의 직접 변경을 방지하기 위해
  사용자의 직접적인 접근을 차단하는 것임
- `public`과 다른 `private` 데이터 필드로 정의함

### 12-2. private 데이터 필드

- 클래스 내부에서만 접근 가능함
- 이름 앞에 밑줄 두 줄 `__`을 붙여 정의함

### 예시

    self.__r
    self.__h

### 핵심 정리

- private 데이터 필드는 객체 외부에서 직접 접근할 수 없음
- 데이터의 잘못된 변경을 막고 객체 내부 상태를 안전하게 보호함

---

## 13. 접근자와 변경자

### 13-1. private 데이터 필드 접근

- private으로 정의된 데이터 필드는 객체 외부에서 접근할 수 없음
- 따라서 private 데이터 필드에 접근하기 위한 메소드를 따로 정의함

### 13-2. 접근자와 변경자

| 구분  | 영어 표현              | 역할        |
|-----|--------------------|-----------|
| 접근자 | accessor 또는 getter | 데이터 필드 반환 |
| 변경자 | mutator 또는 setter  | 데이터 필드 설정 |

### 핵심 정리

- 접근자는 private 데이터 필드 값을 읽는 메소드임
- 변경자는 private 데이터 필드 값을 수정하는 메소드임
- 객체 외부에서는 데이터 필드에 직접 접근하지 않고 메소드를 통해 접근함

## 14. 원뿔 부피와 겉넓이 계산 프로그램 개선

### 14-1. 개선된 `Cone` 클래스 설계

| 구성             | 내용                                                                                                                              |
|----------------|---------------------------------------------------------------------------------------------------------------------------------|
| 클래스            | `Cone`                                                                                                                          |
| private 데이터 필드 | `__r: int`, `__h: int`                                                                                                          |
| 생성자            | `Cone(radius=20: int, height=30: int)`                                                                                          |
| 메소드            | `get_vol(): float`, `get_surf(): float`, `get_r(): int`, `get_h(): int`, `set_r(radius: int): None`, `set_h(height: int): None` |

### 14-2. 개선된 `Cone` 클래스 예시

    class Cone:
        def __init__(self, radius=20, height=30):
            self.__r = radius
            self.__h = height

        def get_vol(self):
            return 1 / 3 * 3.14 * self.__r ** 2 * self.__h

        def get_surf(self):
            sla = pow(self.__r ** 2 + self.__h ** 2, 0.5)
            return 3.14 * self.__r ** 2 + 3.14 * self.__r * sla

        def get_r(self):
            return self.__r

        def get_h(self):
            return self.__h

        def set_r(self, radius=20):
            if radius > 0:
                self.__r = radius
            else:
                return -1

        def set_h(self, height=30):
            if height > 0:
                self.__h = height
            else:
                return -1

### 14-3. 코드 해석

- `self.__r`, `self.__h`
    - 외부에서 직접 접근할 수 없는 private 데이터 필드
- `get_r()`, `get_h()`
    - private 데이터 필드 값을 반환하는 접근자
- `set_r()`, `set_h()`
    - private 데이터 필드 값을 변경하는 변경자
- 변경자에서는 입력값이 양수인지 확인하여 잘못된 값 저장을 방지함

### 핵심 정리

- 데이터 필드를 private으로 감추면 직접 변경을 막을 수 있음
- 접근자와 변경자를 통해 안전하게 값을 읽고 수정할 수 있음

## 15. 정리하기

1. 객체지향 프로그래밍은 객체와 객체의 상호작용을 중심으로 프로그램을 구성하는 방식이며,
   프로그램을 유연하고 변경하기 쉽게 만듦

2. 객체지향 패러다임의 대표적 특징은 다음과 같음
    - 추상화
    - 캡슐화
    - 상속
    - 다형성

3. 클래스는 데이터와 메소드를 함께 가지는 데이터 타입이며,
   이러한 객체의 구조와 기능은 클래스에 의해 정의됨

4. 클래스는 데이터 필드와 메소드를 포함하며,
   객체의 상태를 초기화하기 위해 `__init__` 초기자를 사용함

5. 인스턴스 생성 시 클래스 이름과 초기자의 매개변수를 사용해 생성자를 호출함

6. 클래스의 입력 초기자 매개변수는 이용하여 인스턴스를 생성할 수 있으며,
   객체와 인스턴스는 같은 의미로 사용됨

7. 생성자는 객체 참조변수를 통해 호출되며,
   입력값은 `__init__`을 이용하여 데이터 필드에 전달됨

8. 클래스 설계는 UML 클래스 다이어그램을 통해 데이터 필드, 생성자, 메소드를 체계적으로 표현할 수 있음

9. 파이썬의 기본 데이터 타입인 `int`, `str` 등도 객체이며,
   `type()`과 `id()`를 통해 타입과 식별값을 확인할 수 있음

10. 데이터 은닉은 데이터 필드의 직접 접근을 방지하기 위해 사용되며,
    private 데이터 필드는 앞에 밑줄 두 줄 `__`을 붙여 정의함

11. private 데이터 필드는 객체 외부에서 직접 접근할 수 없으므로,
    접근자와 변경자 메소드를 이용해 값을 읽거나 수정함

## 16. 핵심 요약

### 16-1. 객체지향

- 객체 사이의 상호작용으로 프로그램을 구성하는 방식임
- 프로그램을 유연하고 변경하기 쉽게 만듦
- 대규모 소프트웨어 개발에 적합함

### 16-2. 클래스와 객체

| 구분         | 의미                        |
|------------|---------------------------|
| 클래스        | 객체를 만들기 위한 설계도            |
| 객체         | 클래스로부터 생성된 실제 데이터         |
| 인스턴스       | 클래스로부터 생성된 객체             |

| 생성자        | 객체 생성 시 초기화를 담당하는 메소드     |
| `__init__` | 파이썬에서 객체 초기화에 사용되는 특수 메소드 |
| `self`     | 현재 객체 자신을 참조하는 매개변수       |

### 16-3. 객체 사용 흐름

1. 클래스 정의
2. 생성자 호출

5. 데이터 필드 초기화
6. 객체 참조변수로 데이터 필드와 메소드 사용

### 16-4. 데이터 은닉

- 접근자와 변경자를 통해 안전하게 값을 다룸

## 17. 연습문제 정리

### Q1. 객체의 상태를 초기화하기 위해 사용하는 특수 메소드는 무엇인가?

1. `__main__`
2. `__init__`
3. `class`
4. `return`

- 정답: **2**

### 해설

- `__init__`은 객체가 생성될 때 자동으로 호출되는 초기자임
- 객체의 데이터 필드를 초기화할 때 사용함

---

### Q2. 다음 코드의 출력값으로 옳은 것은?

    class Cone:
        def __init__(self, radius=20, height=30):
            self.r = radius
            self.h = height

    unitcone = Cone(50, 100)
    print(unitcone.h)

1. `20`
2. `30`
3. `50`
4. `100`

- 정답: **4**

### 해설

- `Cone(50, 100)`을 호출하면
    - `radius = 50`
    - `height = 100`
- `self.h = height`이므로 `unitcone.h`에는 `100`이 저장됨

---

### Q3. 접근자와 변경자에 대한 설명으로 옳은 것은?

1. 접근자는 데이터를 삭제하고, 변경자는 객체를 생성한다.
2. 접근자는 private 데이터 필드를 반환하고, 변경자는 private 데이터 필드를 설정한다.
3. 접근자와 변경자는 모두 클래스 외부에서만 정의된다.
4. 접근자는 객체를 복사하고, 변경자는 객체를 저장한다.

- 정답: **2**

### 해설

- 접근자(accessor, getter)는 private 데이터 필드 값을 반환함
- 변경자(mutator, setter)는 private 데이터 필드 값을 설정함
- private 데이터 필드에 직접 접근할 수 없기 때문에 접근자와 변경자를 사용함

## 18. 시험 직전 체크포인트

- 객체지향은 객체와 객체의 상호작용으로 프로그램을 구성하는 방식이다.
- 객체지향의 특징은 추상화, 캡슐화, 상속, 다형성이다.
- 클래스는 객체를 만들기 위한 설계도이다.
- 객체와 인스턴스는 같은 의미로 사용할 수 있다.
- `class 클래스이름:` 형식으로 클래스를 정의한다.
- 초기자는 `__init__`으로 명명한다.
- `__init__`은 객체 생성 시 자동 호출된다.
- 메소드의 첫 번째 매개변수는 `self`이다.
- `self`는 객체 자신을 참조한다.
- 객체의 데이터 필드는 `self.데이터필드` 형식으로 정의한다.
- 객체의 멤버 접근에는 점 연산자 `.`을 사용한다.
- `객체명.데이터필드`로 데이터에 접근한다.
- `객체명.메소드()`로 메소드를 호출한다.
- 파이썬의 `int`, `str`도 객체이다.
- `type()`은 객체의 타입을 확인한다.
- `id()`는 객체의 고유 식별값을 확인한다.
- private 데이터 필드는 `self.__변수명` 형식으로 정의한다.
- private 데이터 필드는 객체 외부에서 직접 접근할 수 없다.
- 접근자는 private 데이터 필드 값을 반환한다.
- 변경자는 private 데이터 필드 값을 설정한다.
