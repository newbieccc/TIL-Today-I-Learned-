## 1. 프로젝트 개요

### 1.1 프로젝트 이름

이번 강의의 프로젝트 이름은 `텍스트 데이터 분석기`이다.

텍스트 파일을 읽고, 의미 있는 키워드를 추출한 뒤, 키워드 빈도를 시각적으로 표현하고 파일로 리포팅하는 전체 파이프라인을 구현하는 것이 목표이다.

---

### 1.2 프로젝트 목표

| 항목     | 내용                               |
|--------|----------------------------------|
| 입력 데이터 | 텍스트 파일                           |

| 결과 표현  | 키워드 빈도 시각화                       |
| 최종 목표  | Raw Data를 가치 있는 Information으로 변환 |

## 2. 데이터 분석의 생애주기

### 2.1 데이터 분석 파이프라인

데이터 분석은 원시 데이터가 인사이트가 되기까지 표준화된 흐름을 가진다.

    Raw Data
    → Collection
    → Preprocessing
    → Analysis
    → Visualization
    → Information

---

### 2.2 데이터 분석 파이프라인 단계

| 단계            | 의미  | 설명                                |
|---------------|-----|-----------------------------------|
| Collection    | 수집  | 파일 입출력을 통해 외부 데이터를 메모리로 적재        |
| Preprocessing | 전처리 | 분석 가능한 형태로 데이터를 정제 및 변환           |
| Analysis      | 분석  | 통계, 알고리즘, 딥러닝 등을 적용하여 가치 있는 패턴 추출 |
| Visualization | 시각화 | 인지적 부담을 줄이고 정보 전달력을 높이는 그래픽 표현    |

---

### 2.3 데이터 분석의 핵심

데이터 분석은 원시 데이터에서 유의미한 패턴을 찾고, 이를 정보로 변환하는 실질적인 문제 해결 작업 과정이다.

시험에서는 `Raw Data → Information`으로 바꾸는 표준 흐름을 `파이프라인`으로 기억한다.

---

## 3. 전처리의 필요성

### 3.1 GIGO 원칙

GIGO는 `Garbage In, Garbage Out`의 약자이다.

잘못되거나 부정확한 데이터를 입력하면, 아무리 정교한 분석 기법을 적용하더라도 잘못된 결과가 출력된다.

| 원칙          | 의미              |
|-------------|-----------------|
| Garbage In  | 품질이 낮은 데이터 입력   |
| Garbage Out | 품질이 낮은 분석 결과 출력 |

---

### 3.2 전처리의 목적

전처리는 데이터 품질을 높여 분석 결과의 질을 향상시키는 과정이다.

텍스트 분석에서는 특히 불용어 제거, 대소문자 통일, 구두점 제거, 단어 분리 등이 중요하다.

---

## 6. 데이터 전처리

### 6.1 불용어

불용어(stopwords)는 문법적으로는 필요하지만 분석적 의미는 적은 단어이다.

예:

    a, the, is, are, of, at

텍스트 분석에서는 이런 단어들이 자주 등장하지만 핵심 의미를 담고 있지 않으므로 제거하는 것이 좋다.

---

### 6.2 불용어 처리

불용어 처리를 위해서는 적절한 자료구조를 선택해야 한다.

| 자료구조 | 활용                    |
|------|-----------------------|
| 리스트  | 단어 목록 저장 가능           |
| 세트   | 포함 여부를 빠르게 확인할 때 유리   |
| 딕셔너리 | 단어와 빈도처럼 쌍으로 저장할 때 유리 |

---

### 6.3 분석하기 쉬운 형태로 가공

텍스트 분석에서는 같은 단어라도 표기 방식이 다르면 다른 단어로 처리될 수 있다.

예:

    Apple
    apple
    apple!

위 세 단어는 분석 관점에서는 같은 단어로 보아야 한다.

따라서 다음 처리가 필요하다.

| 처리        | 사용 함수 또는 메소드      |
|-----------|-------------------|
| 소문자 통일    | lower()           |
| 불필요 문자 치환 | replace(old, new) |
| 공백 기준 분리  | split()           |

---

### 6.4 전처리 예시 코드

    def clean_text(text):
        stopwords = {"a", "the", "is", "are", "of", "at", "to", "and", "in"}

        text = text.lower()

        for ch in [".", ",", "!", "?", ":", ";", "\"", "'"]:
            text = text.replace(ch, " ")

        words = text.split()

        cleaned_words = []

        for word in words:
            if word not in stopwords:
                cleaned_words.append(word)

        return cleaned_words

코드 해석:

- `lower()`로 대소문자를 통일한다.
- `replace()`로 문장부호를 공백으로 바꾼다.
- `split()`으로 문자열을 단어 단위로 나눈다.
- 불용어는 결과에서 제외한다.

## 7. 빈도 분석

### 7.1 빈도 분석의 의미

빈도 분석은 어떤 단어가 몇 번 등장했는지 저장소에 기록하는 과정이다.

텍스트 데이터 분석에서는 단어별 등장 횟수를 계산하여 주요 키워드를 찾는다.

---

### 7.2 딕셔너리와 Key-Value 모델

단어 빈도는 딕셔너리로 저장하는 것이 적합하다.

| 구성    | 의미          |
|-------|-------------|
| Key   | 고유한 식별자, 단어 |
| Value | 데이터의 속성, 빈도 |

예:

    {
        "holmes": 2926,
        "watson": 1500,
        "case": 800
    }

---

### 7.3 매 단어 처리 흐름

단어를 하나씩 확인하면서 딕셔너리에 빈도를 누적한다.

| 경우             | 처리      |
|----------------|---------|
| 딕셔너리에 이미 있는 단어 | 카운트 + 1 |
| 딕셔너리에 없는 단어    | 1로 초기화  |

---

### 7.4 get() 메소드를 활용한 카운팅

`get()` 메소드는 키가 존재하면 해당 값을 반환하고, 없으면 설정한 기본값을 반환한다.

    def count_words(word_list):
        counts = {}

        for word in word_list:
            counts[word] = counts.get(word, 0) + 1

        return counts

코드 해석:

- `counts.get(word, 0)`은 word가 있으면 기존 빈도를 가져온다.
- word가 없으면 0을 반환한다.
- 여기에 1을 더해 빈도를 누적한다.

---

## 8. 데이터 정렬 및 시각화

### 8.1 정렬의 필요성

빈도 순위를 표시하려면 단어 빈도를 기준으로 정렬해야 한다.

딕셔너리는 순서가 보장되지 않고 키 기준으로만 관리하기 때문에, 빈도 기준 정렬을 위해 자료구조 변환이 필요하다.

---

### 8.2 딕셔너리를 리스트로 변환

딕셔너리의 key-value 구성을 값과 키의 리스트로 바꾸면 빈도 기준 정렬이 쉬워진다.

예:

    {
        "holmes": 2926,
        "what": 2291,
        "if": 2049
    }

변환 예:

    [
        (2926, "holmes"),
        (2291, "what"),
        (2049, "if")
    ]

---

### 8.3 빈도 기준 정렬 코드

    def get_sorted_list(counts_dict):
        result = []

        for word, count in counts_dict.items():
            result.append((count, word))

        result.sort(reverse=True)

        return result

코드 해석:

- 딕셔너리를 `(빈도, 단어)` 형태의 튜플 리스트로 변환한다.
- `sort(reverse=True)`를 사용하면 빈도 높은 순으로 정렬된다.

---

### 8.4 데이터 시각화

데이터 시각화는 복잡한 데이터를 시각적인 형태로 표현하는 과정이다.

| 목적         | 설명                   |
|------------|----------------------|
| 데이터 이해도 향상 | 수치만 보는 것보다 패턴 파악이 쉬움 |
| 효과적인 의사소통  | 분석 결과를 쉽게 전달         |
| 의사결정 지원    | 핵심 정보를 빠르게 확인        |

이번 실습에서는 별도의 그래픽 라이브러리 없이 텍스트만으로 데이터의 크기를 표현한다.

---

### 8.5 텍스트 기반 시각화 예시

    def print_save_graph(data):
        for count, word in data[:10]:
            bar = "*" * (count // 100)
            print(word, count, bar)

예상 출력:

    holmes 2926 *****************************
    what 2291 **********************
    if 2049 ********************

코드 해석:

- 빈도가 높은 상위 10개 단어만 출력한다.
- 단어 빈도를 `*` 개수로 표현한다.
- 숫자와 시각적 표현을 함께 보여준다.

---

## 9. 전체 프로그램 흐름 예시

### 9.1 함수 구성

    import os

    def clean_text(text):
        stopwords = {"a", "the", "is", "are", "of", "at", "to", "and", "in"}

        text = text.lower()

        for ch in [".", ",", "!", "?", ":", ";", "\"", "'"]:
            text = text.replace(ch, " ")

        words = text.split()

        cleaned_words = []

        for word in words:
            if word not in stopwords:
                cleaned_words.append(word)

        return cleaned_words


    def count_words(word_list):
        counts = {}

        for word in word_list:
            counts[word] = counts.get(word, 0) + 1

        return counts


    def get_sorted_list(counts_dict):
        result = []

        for word, count in counts_dict.items():
            result.append((count, word))

        result.sort(reverse=True)

        return result


    def print_save_graph(data):
        for count, word in data[:10]:
            bar = "*" * (count // 100)
            print(word, count, bar)

---

### 9.2 실행 흐름

    print("현재 작업 위치:", os.getcwd())

    with open(file_name, "r", encoding="utf-8") as f:
        text = f.read()

    words = clean_text(text)

    counts = count_words(words)

    sorted_data = get_sorted_list(counts)

## 정리하기

1. 데이터 분석은 원시 데이터에서 유의미한 패턴을 찾고 정보로 변환하는 실질적인 문제 해결 작업과정이다.

2. 데이터 분석의 표준 파이프라인은 데이터를 읽어오는 수집, 분석하기 쉽게 다듬는 전처리, 패턴을 추출하는 분석, 직관적으로 표현하는 시각화 단계로 구성된다.

3. 분석 결과의 품질은 데이터의 품질에 직접적으로 비례한다는 GIGO 원칙에 따라, 텍스트 분석 시 불용어를 필터링하는 전처리 과정이 매우 중요하다.

4. 문자열 정제를 위해 모든 문자를 소문자로 통일하는 `lower()`, 불필요한 문자를 치환하는 `replace()`, 공백 등을 기준으로 텍스트를 분리하는 `split()` 등의 내장 메소드를 적절히 조합하여
   활용한다.

5. 프로그램이 실행 중인 현재 작업 위치를 확인하고 외부 텍스트 파일에 접근하기 위해 `os` 모듈을 활용할 수 있다.

6. 핵심 키워드의 출현 빈도는 key-value 구조를 가진 딕셔너리를 통해 기록하며, `get()` 메소드로 단어의 등장 횟수를 안전하게 누적할 수 있다.

7. 분석이 완료된 딕셔너리 형태의 데이터는 가장 많이 등장한 순서대로 순위를 시각화하기 위해 값, 즉 빈도수를 기준으로 정렬 가능한 리스트 형태로 자료구조를 변환해야 한다.

| sort(reverse=True) | 내림차순 정렬                 |

## 시험 직전 체크포인트

### 반드시 암기하기

- 프로젝트 이름: 텍스트 데이터 분석기
- 목표: Raw Data를 Information으로 변환
- 데이터 분석 파이프라인: Collection → Preprocessing → Analysis → Visualization
- GIGO: Garbage In, Garbage Out
- 전처리: 분석 결과 품질 향상을 위한 데이터 정제
- 불용어: 문법적으로는 필요하지만 분석적 의미가 적은 단어
- `lower()`: 소문자 변환
- `replace(old, new)`: 문자 치환
- `split()`: 문자열 분리
- `os` 모듈: 운영체제 기능 제어
- CWD: Current Working Directory, 현재 작업 위치
- 딕셔너리: key-value 구조
- 단어 빈도 저장: key는 단어, value는 빈도
- `get()`: 안전한 빈도 누적에 사용
- 빈도 순위 표시: 딕셔너리를 리스트로 변환 후 값 기준 정렬
- 시각화: 복잡한 데이터를 직관적 형태로 표현
