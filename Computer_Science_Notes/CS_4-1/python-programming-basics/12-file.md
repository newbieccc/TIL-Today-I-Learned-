## 2. 파일의 이해

### 2-1. 휘발성과 비휘발성

- 폰 노이만 구조에서는 명령과 데이터가 기억장치에 저장되어 처리됨
- 주기억장치의 데이터는 프로그램 실행 중에는 사용되지만,
  프로그램 실행이 종료되거나 전원이 꺼지면 사라질 수 있음
- 데이터를 계속 보존하려면 **영속성(persistence)** 이 필요함

### 영속성

- 데이터가 생성된 프로그램의 실행이 종료되어도
  사라지지 않고 계속 남아 있는 속성
- 데이터의 영속성을 위해 **보조기억장치**를 사용함

### 핵심 정리

- 메모리의 데이터는 휘발성이 있음
- 데이터를 오래 보존하려면 HDD, SSD, USB 같은 보조기억장치에 저장해야 함

---

## 3. 보조기억장치와 파일 시스템

### 3-1. 보조기억장치

- 전원이 소실되어도 저장된 정보가 훼손되지 않는 영구적 기록 장치
- 운영체제는 파일 시스템을 구축하고,
  그 위에 **파일**이라는 논리적 단위를 통해 데이터 저장 기능을 제공함

### 대표 보조기억장치

- HDD
- ODD
- USB
- SSD

### 핵심 정리

- 보조기억장치는 데이터를 장기간 보존하기 위한 장치임
- 운영체제는 파일 시스템을 통해 데이터를 파일 단위로 관리함

## 4. 파일 시스템의 구조

### 4-1. 파일 시스템

- 루트(root, `C:\` 또는 `/`)에서 시작하여
  가지처럼 뻗어나가는 **계층적 트리 구조**임

### 4-2. 디렉터리 또는 폴더

- 파일 및 하위 디렉터리를 포함하는 컨테이너
- 파일을 분류하고 관리하기 위한 논리적 공간

### 4-3. 경로(path)

- 파일 시스템 상에서 특정 파일이나 디렉터리의 위치를 나타내는 주소

### 예시

    /var/etc/python.py

### 핵심 정리

- 파일 시스템은 루트에서 시작하는 트리 구조임
- 디렉터리는 파일과 하위 디렉터리를 담는 컨테이너임
- 경로는 파일이나 디렉터리의 위치를 나타냄

---

## 5. 파일의 경로 지정

### 5-1. 절대 경로

- 최상위 루트 디렉터리부터 목표 파일까지의 전체 경로를 모두 명시함

### 예시

    /var/etc/python.py

### 5-2. 상대 경로

- 현재 작업 디렉터리를 기준으로 목표 파일의 위치를 지정함

### 상대 경로 기호

| 기호   | 의미            |
|------|---------------|
| `.`  | 현재 디렉터리       |
| `..` | 상위 또는 부모 디렉터리 |

### 예시

    ../etc/python.py

### 핵심 정리

- 절대 경로는 루트부터 전체 위치를 작성함
- 상대 경로는 현재 작업 디렉터리를 기준으로 작성함
- `..`는 상위 디렉터리로 이동할 때 사용함

## 6. 파일의 개념

### 6-1. 파일

- 보조기억장치에 저장된 데이터의 논리적 집합

### 6-2. 파일의 구성 요소

| 구성 요소 | 설명                         |
|-------|----------------------------|
| 파일명   | 파일을 식별하는 고유 문자열            |
| 확장자   | 파일의 형식을 명시하는 접미사           |
| 내용    | 컴퓨터에 의해 처리될 또는 처리된 데이터와 정보 |

### 확장자 예시

- `.txt`
- `.py`
- `.csv`

### 핵심 정리

- 파일은 저장된 데이터를 논리적으로 묶은 단위임
- 파일명과 확장자를 통해 파일의 이름과 형식을 구분함

## 7. 파일의 종류

### 7-1. 텍스트 파일

- 사람이 인지 가능한 문자열로 구성됨
- 각 문자는 특정 인코딩 방식에 따라 비트열로 변환되어 저장됨

### 7-2. 바이너리 파일

- 텍스트가 아닌 이진 데이터를 포함함
- 예:
    - 이미지 파일
    - 실행 파일 등
- 전용 소프트웨어를 통해서만 해석 가능함

### 핵심 정리

- 텍스트 파일은 사람이 읽을 수 있는 문자열 중심 파일임
- 바이너리 파일은 전용 프로그램을 통해 해석해야 하는 이진 데이터 파일임

---

## 8. 파일의 표현

### 8-1. 파일의 내부 표현

- 파일은 연속된 바이트로 표현됨
- 파일에는 다음 개념이 있음
    - 파일의 시작
    - 작업 위치 또는 파일 포인터
    - 파일의 끝, `EOF`

### 8-2. 주요 문자 인코딩 방식

| 인코딩            | 설명                                                 |
|----------------|----------------------------------------------------|
| ASCII          | 7비트 기반, 영문 및 기본 특수문자만 표현 가능                        |
| EUC-KR / CP949 | 완성형 한글 표현 방식, Windows 기본값으로 사용되었으나 호환성 문제가 있을 수 있음 |
| UTF-8          | 유니코드 기반 기본 문자 인코딩, 국제 표준으로 널리 사용됨                  |

### 핵심 정리

- 파일은 바이트의 연속으로 저장됨
- 텍스트 파일은 인코딩 방식을 통해 문자를 바이트로 변환하여 저장함
- 파이썬에서 한글 파일을 다룰 때는 `utf-8` 인코딩 지정이 중요함

## 9. 파일의 사용

### 9-1. 파일 입출력의 3단계 메커니즘

파일 입출력은 다음 3단계로 진행됨.

1. `open`
2. `access`
3. `close`

### 파일 열기

- 파일 객체를 생성하고 운영체제로부터 파일 접근 권한을 획득함
- 데이터 이동 통로인 스트림을 형성함

### 접근

- 생성된 파일 객체의 메소드를 통해 데이터를 읽거나 씀

### 파일 닫기

- 파일 객체를 소멸시키고 점유했던 시스템 자원을 반환함
- 데이터 무결성을 유지하기 위해 필요한 절차임

### 핵심 정리

- 파일 작업은 반드시 **열기 → 접근 → 닫기** 순서로 수행함
- 파일을 열었으면 작업 후 반드시 닫아야 함

## 10. 파일 관련 내장 함수와 메소드

### 10-1. 주요 함수와 메소드

| 함수/메소드        | 의미                  |
|---------------|---------------------|
| `open()`      | 파일과 연결된 파일 객체 생성    |
| `read()`      | 특정 개수의 문자를 반환       |
| `readline()`  | 한 라인의 문자열을 반환       |
| `readlines()` | 전체 라인의 문자열을 리스트로 반환 |
| `write()`     | 파일에 문자열 작성          |
| `close()`     | 파일 닫기 및 파일 객체 삭제    |

### 핵심 정리

- 파일 객체를 만든 뒤,
  파일 객체의 메소드를 통해 읽기와 쓰기를 수행함

## 11. 파일 객체 생성

### 11-1. 파일 열기 구문

    파일객체_참조변수 = open(파일이름, 모드, 인코딩)

### 11-2. 파일 모드

| 모드    | 의미                |
|-------|-------------------|
| `"r"` | 읽기 용도             |
| `"w"` | 새로운 파일 쓰기 용도      |
| `"a"` | 파일 끝에 데이터 덧붙이기 용도 |

### 예시

    j_fp = open("Jobs.txt", "r", encoding="utf-8")

### 핵심 정리

- `open()`은 파일 객체를 생성함
- `"r"`은 읽기, `"w"`는 쓰기, `"a"`는 추가 모드임
- 한글 파일은 `encoding="utf-8"`을 명시하는 것이 안전함

## 12. 파일 읽기

### 12-1. `read()` 메소드

- 특정 범위의 데이터를 파일에서 읽고 문자열로 반환함
- 파일 포인터의 이동을 동반함

### 예시

    j_fp = open("Jobs.txt", "r", encoding="utf-8")

    first6 = j_fp.read(6)
    left = j_fp.read()

    j_fp.close()

### 코드 해석

- `read(6)`
    - 현재 파일 포인터 위치에서 문자 6개를 읽음
- `read()`
    - 남은 내용을 모두 읽음
- 읽은 만큼 파일 포인터가 이동함

### 핵심 정리

- `read(n)`은 n개의 문자를 읽음
- `read()`는 남은 전체 내용을 읽음

---

## 13. 파일 분할 읽기

### 13-1. `readline()`

- 호출할 때마다 한 줄씩 읽어 반환함
- 한 줄은 개행 문자 `\n`까지 포함될 수 있음

### 13-2. `readlines()`

- 파일의 모든 줄을 읽어 문자열 리스트 형태로 반환함

### 예시

    j_fp = open("Jobs.txt", "r", encoding="utf-8")

    first_line = j_fp.readline()
    second_line = j_fp.readline()
    lines = j_fp.readlines()

    j_fp.close()

### 핵심 정리

- `readline()`은 한 줄씩 읽음
- `readlines()`는 모든 줄을 리스트로 읽음
- 대용량 파일에서는 `readlines()`보다 한 줄씩 읽는 방식이 더 안전함

---

## 14. 파일 쓰기

### 14-1. `write()` 메소드

- 문자열 타입의 데이터만 인수로 허용함
- 파일 포인터가 위치한 지점에 기록함
- 줄바꿈을 자동으로 수행하지 않음
- 개행 문자는 명시적으로 삽입해야 함

### 예시

    p_fp = open("python.txt", "w", encoding="utf-8")

    p_fp.write("KNOU\n")
    p_fp.write("python programming\n")

    p_fp.close()

### 주의

- `"w"` 모드로 존재하는 파일을 열면 기존 내용이 삭제됨

### 핵심 정리

- `write()`는 문자열만 쓸 수 있음
- 줄바꿈은 `\n`을 직접 작성해야 함
- `"w"` 모드는 기존 파일 내용을 덮어씀

---

## 15. 데이터 추가

### 15-1. append 모드

- 파일 끝에 데이터를 덧붙이는 작업
- 파일을 연 후 파일 포인터를 곧바로 `EOF`로 이동함
- 존재하지 않는 파일의 경우 `"w"` 모드와 동일하게 새 파일을 생성함

### 예시

    p_fp = open("python.txt", "a", encoding="utf-8")

    p_fp.write("by CS\n")

    p_fp.close()

### 핵심 정리

- `"a"` 모드는 기존 파일의 끝에 내용을 추가함
- 기존 내용을 유지하면서 새 내용을 덧붙일 때 사용함

## 16. 파일 객체 종료

### 16-1. `close()` 메소드

- 점유하고 있던 시스템 자원, 메모리, 파일 핸들을 운영체제에 반환함
- `open()` 함수가 호출되었다면 반드시 대응되는 위치에 `close()`가 존재해야 함

### 예시

    p_fp.close()

### 16-2. `close()`의 역할

- 데이터 무결성 보장
    - 임시 버퍼에 남아 있는 데이터가 저장됨
- 자원 누수 방지
    - 운영체제가 동시에 열 수 있는 파일 개수에는 제한이 있음

### 주의

- 프로그램이 비정상 종료되면,
  열린 파일이 사용 중 상태로 잠겨 다른 프로그램에서 접근하지 못할 수 있음

### 핵심 정리

- 파일을 열었으면 반드시 닫아야 함
- `close()`는 데이터 저장과 자원 반환을 위해 중요함

## 17. 효과적 파일 처리

### 17-1. 데이터 파싱의 개요

- 파일에서 읽어 온 데이터는 단순한 문자열 형태로는 곧바로 처리하기 어려울 수 있음
- 데이터 파싱(parsing)은 일련의 문자열을 문법적 규칙에 따라 분해하고,
  의미 있는 단위로 변환하는 과정임

### 17-2. 데이터 파싱 과정

    읽기 → 정제 → 분리 → 변환

### 핵심 정리

- 파일에서 읽은 문자열은 목적에 맞게 정제하고 나누고 변환해야 함
- 대용량 데이터 파일 처리 시 파싱 과정의 효율성을 고려해야 함

## 18. 읽기와 정제

### 18-1. 대용량 파일 처리

- GB 크기의 파일 전체를 한 번에 읽으면 메모리 오버플로 위험이 있음
- 한 번에 한 줄씩만 메모리에 적재하여 처리하는 방식이 안전함

### 예시

    for line in file:
        line.split(",")

### 18-2. 불필요한 문자 제거

- 텍스트 앞뒤에는 개행 문자 `\n`, 공백, 탭 `\t` 등이 포함될 수 있음
- 이러한 불필요한 문자는 비교 연산이나 처리 오류의 원인이 될 수 있음

### 예시

    line = line.strip()

### 핵심 정리

- 대용량 파일은 한 줄씩 처리하는 방식이 효율적임
- `strip()`은 앞뒤 공백, 개행, 탭 등을 제거할 때 사용함

## 19. 정리하기

1. 데이터의 영속성을 위해 비휘발성 보조기억장치가 사용됨
    - 운영체제는 계층적 트리 구조의 파일 시스템을 통해 데이터를 논리적 단위인 파일로 보관함

2. 파일의 위치를 지정할 때는 최상위 루트를 기준으로 하는 절대 경로와,
   현재 작업 중인 디렉터리를 기준으로 하는 상대 경로 방식을 사용함

3. 파일은 데이터의 성격에 따라 텍스트 파일과 바이너리 파일로 구분됨

4. 파일 입출력은 다음 3단계를 가짐

   열기(open) → 접근(read/write) → 닫기(close)

5. 새로운 파일을 생성하거나 덮어쓸 때는 `"w"` 모드,
   기존 데이터 끝에 내용을 덧붙일 때는 `"a"` 모드를 사용함

6. 파일에서 데이터를 가져올 때는
   파일 포인터를 이동하며 특정 개수만큼 읽는 `read()`,
   한 줄만 읽는 `readline()`,
   전체 줄을 리스트로 반환하는 `readlines()` 등을 사용함

7. 대용량 데이터는 한 줄씩 읽어 처리하는 방식으로 메모리 오버플로를 예방해야 함
    - 정확한 비교와 처리를 위해 `strip()`으로 개행 문자와 공백을 제거함

## 20. 핵심 요약

### 20-1. 파일의 핵심 개념

- 파일은 보조기억장치에 저장된 데이터의 논리적 집합임
- 데이터의 영속성을 위해 사용됨
- 파일 시스템은 계층적 트리 구조임
- 파일은 파일명, 확장자, 내용으로 구성됨

### 20-2. 경로

| 구분    | 설명                           |
|-------|------------------------------|
| 절대 경로 | 루트 디렉터리부터 목표 파일까지 전체 경로 명시   |
| 상대 경로 | 현재 작업 디렉터리를 기준으로 목표 파일 위치 지정 |
| `.`   | 현재 디렉터리                      |
| `..`  | 상위 디렉터리                      |

### 20-3. 파일 종류

| 구분      | 설명                        |
|---------|---------------------------|
| 텍스트 파일  | 사람이 읽을 수 있는 문자열 기반 파일     |
| 바이너리 파일 | 이미지, 실행 파일처럼 이진 데이터 기반 파일 |

### 20-4. 파일 모드

| 모드    | 의미                    |
|-------|-----------------------|
| `"r"` | 읽기                    |
| `"w"` | 새 파일 쓰기 또는 기존 파일 덮어쓰기 |
| `"a"` | 기존 파일 끝에 데이터 추가       |

### 20-5. 주요 파일 메소드

| 메소드           | 의미                 |
|---------------|--------------------|
| `read()`      | 특정 개수 또는 전체 문자를 읽음 |
| `readline()`  | 한 줄을 읽음            |
| `readlines()` | 전체 줄을 리스트로 읽음      |
| `write()`     | 문자열을 파일에 씀         |
| `close()`     | 파일을 닫고 자원을 반환함     |

### 20-6. 효과적 파일 처리

- 대용량 파일은 한 번에 전부 읽지 않음
- 한 줄씩 읽어 처리함
- `strip()`으로 개행 문자, 공백, 탭 등을 제거함
- 필요에 따라 `split()`으로 분리하고 타입 변환을 수행함

## 21. 연습문제 정리

### Q1. 현재 작업 디렉터리의 위치가 `/var/home/user/docs`라고 가정하자.

부모 디렉터리에 있는 `lib` 폴더 내의 `config.txt` 파일을 상대 경로로 열고자 할 때 가장 적절한 경로는?

1. `/var/home/lib/config.txt`
2. `./lib/config.txt`
3. `../lib/config.txt`
4. `~/lib/config.txt`

- 정답: **3**

### 해설

- 현재 위치는 `/var/home/user/docs`
- 부모 디렉터리는 `/var/home/user`
- 부모 디렉터리로 이동하려면 `..`를 사용함
- 따라서 `../lib/config.txt`가 가장 적절한 상대 경로임

---

### Q2. 프로그램이 데이터가 누적되도록 파일을 다루고자 할 때,

`open()` 함수에서 지정해야 할 가장 적절한 모드는?

1. `"r"`
2. `"w"`
3. `"a"`
4. `"x"`

- 정답: **3**

### 해설

- `"a"`는 append 모드임
- 기존 파일의 끝에 데이터를 덧붙일 때 사용함
- 기존 데이터를 유지하면서 새로운 데이터를 누적해야 하므로 `"a"`가 적절함

---

### Q3. 대용량의 파일 분석 프로그램 작성 시,

다음 중 데이터를 안전하게 처리할 수 있는 방법은?

1. `read()` 메소드로 파일 전체를 하나의 문자열 변수에 담아 분석한다.
2. `readlines()` 메소드로 파일의 모든 줄을 메모리에 리스트 구조로 한 번에 적재한다.
3. 한 번에 한 줄씩만 메모리에 읽어 들여 전체 처리를 수행한 후 다음 줄로 넘어가는 반복문 `for line in file:`을 구성한다.
4. 데이터를 읽기 전 `close()` 함수를 선제적으로 호출하여 메모리를 비운다.

- 정답: **3**

### 해설

- 대용량 파일을 한 번에 읽으면 메모리 오버플로 위험이 있음
- 따라서 한 줄씩 읽어 처리하는 방식이 안전함

  for line in file:
  line = line.strip()
  data = line.split(",")

## 22. 시험 직전 체크포인트

- 데이터의 영속성을 위해 보조기억장치를 사용한다.
- 파일은 보조기억장치에 저장된 데이터의 논리적 집합이다.
- 파일 시스템은 루트에서 시작하는 계층적 트리 구조이다.
- 절대 경로는 루트부터 전체 경로를 작성한다.
- 상대 경로는 현재 작업 디렉터리를 기준으로 작성한다.
- `.`은 현재 디렉터리, `..`은 상위 디렉터리를 의미한다.
- 파일은 텍스트 파일과 바이너리 파일로 구분된다.
- 텍스트 파일은 문자 인코딩 방식에 따라 비트열로 저장된다.
- 대표 인코딩에는 ASCII, EUC-KR, UTF-8이 있다.
- 파일 입출력은 `open → access → close` 순서로 진행된다.
- `open()`은 파일 객체를 생성한다.
- `"r"`은 읽기 모드이다.
- `"w"`는 쓰기 모드이며 기존 파일이 있으면 내용을 삭제한다.
- `"a"`는 파일 끝에 데이터를 추가하는 모드이다.
- `read()`는 특정 개수 또는 전체 문자를 읽는다.
- `readline()`은 한 줄을 읽는다.
- `readlines()`는 모든 줄을 리스트로 읽는다.
- `write()`는 문자열만 파일에 쓸 수 있다.
- `write()`는 줄바꿈을 자동으로 하지 않으므로 `\n`을 직접 써야 한다.
- 파일을 열었으면 반드시 `close()`로 닫아야 한다.
- 대용량 파일은 한 줄씩 읽어 처리하는 것이 안전하다.
- `strip()`은 문자열 앞뒤의 공백, 개행, 탭 등을 제거한다.
