- 컴퓨터과학과 정재화 교수님

# 1. 데이터와 정보

## 데이터의 개념

- 어떤 현상이나 사실에 대한 설명 또는 설명의 집합
- 관찰이나 측정을 통해 얻은 수치, 문자 형태의 표현할 수 있는 질적(quality) 또는 양적(quantity) 값

### 질적데이터

- 붉다
- 밝다
- 뜨겁다

### 양적데이터

- 3500도

## 정보의 개념

- 문제 또는 질문을 해결하기 위해 사용할 수 있는 데이터와 데이터의 집합
- 관찰과 측정을 통해 얻은 데이터를 처리(정렬, 합산, 군집화 등)하여 얻은 결과물

### 예시 질문

- 저 파란색은 무엇일까?
- 붉고 뜨거운 것과는 무엇이 다를까?

## 정보 처리 과정

- 데이터는 현상에 대한 관찰과 측정으로 생성
    - 관찰: 사물 또는 현상을 파악하는 행위
    - 측정: 절대적인 기준에 맞춰 현상을 수치로 나타내는 과정

- 처리와 가공을 통해 정보로 변환
    - 데이터를 정렬, 군집화, 계산 또는 표현방법 변경 등의 변환을 통해 정보를 생산하는 과정

### 예시

- 온도 데이터의 수치 표현
    - 2000
    - 3000
    - 3500
    - 4000
    - 5000
    - 6000
    - 7000
    - 8000
    - 8900

## 컴퓨터의 정의

- 정의된 방법에 따라 입력된 데이터를 자동으로 처리하여 정보를 생산하는 기계

## 데이터의 기본 단위

### 비트(bit, binary digit)

- 데이터를 표현하는 최소 단위

### 바이트(byte)

- 데이터를 처리하는 기본 단위

## 컴퓨터의 구성요소

### 하드웨어

- 컴퓨터를 구성하고 있는 물리적 부품

### 소프트웨어

- 다양한 장치들을 동작시켜 특정 작업을 해결하는 프로그램

# 2. 하드웨어의 구성

## 하드웨어의 논리적 구성

- 기계를 바꾸지 않고 처리 방법의 교체만으로 무엇이든 할 수 있게 만든 컴퓨터의 기본 설계
- 역할에 따라 입력, 기억·저장, 제어, 연산, 출력 장치로 구분

### 폰 노이만 구조

- 제어
- 연산
- 기억장치
    - 명령과 데이터
- 입력장치
- 출력장치

## 입력장치

- 명령과 데이터를 컴퓨터에 전달하는 장치
- 데이터를 컴퓨터가 처리할 수 있는 2진수 형태로 변환

### 예시

- 키보드 & 마우스
- 스캐너
- 터치스크린

## 출력장치

- 정보 처리 결과를 사람이 인식 가능한 형태의 데이터로 내보내는 장치
- 디지털 데이터를 문자, 숫자, 도형, 음성, 영상 등으로 변환

### 예시

- 모니터
- 스피커
- 프린터
- 3D 프린터

## 특수 출력 장치

### 예시

- HUD
    - Head-Up Display
- HMD
    - Head-Mounted Display

## 기억(저장) 장치

- 명령과 데이터를 저장하는 하드웨어
- 역할에 따라 주기억장치와 보조기억장치로 구분

### 주기억장치와 보조기억장치 비교

- 접근속도
    - 기억장치(주기억장치): 빠름
    - 저장장치(보조기억장치): 느림

- 제조단가
    - 기억장치(주기억장치): 높음
    - 저장장치(보조기억장치): 낮음

- 용량
    - 기억장치(주기억장치): 작음
    - 저장장치(보조기억장치): 큼

- 전원 차단 시
    - 기억장치(주기억장치)
        - 저장내용 보존(ROM)
        - 저장내용 소실(RAM)
    - 저장장치(보조기억장치)
        - 저장내용 보존

## 기억&저장 장치의 종류

### 기억장치(주기억장치)

- ROM
- RAM

### 저장장치(보조기억장치)

- 자기디스크
- 광학디스크
- 플래시 드라이브
- SSD

## 제어와 연산 장치

- 명령어와 데이터를 읽고 데이터를 연산 또는 처리하는 장치

### 구성 요소

- CPU
- 레지스터(register)
- 연산 논리 유닛(ALU)
- 제어 유닛(control unit)

## 메인보드/마더보드

- 각 컴퓨터 장치를 연결해주는 회로를 갖춘 장치

### 정리

- 보조기억장치에 있는 데이터들은 RAM을 거쳐 CPU에 전달됩니다.

# 3. 소프트웨어의 실행

## 프로그램의 이해

- 컴퓨터는 정보를 처리하기 위한 방법과 과정을 자의적으로 결정할 수 없음
- 처리 방법과 작동 단계가 사전에 정의된 프로그램을 사용하여 데이터를 처리
- 컴퓨터가 어떠한 작업을 자동으로 처리할 수 있도록 처리 방법 및 순서를 컴퓨터가 이해할 수 있는 언어의 형태로 기술한 것
- 특정 인스턴스가 아닌 유사한 유형의 여러 문제를 해결할 수 있는 단계를 추상화시킨 알고리즘(algorithm)을 구현한 결과물

## 알고리즘의 정의

- 문제를 풀기 위한 단계별 절차를 체계적 명령의 형태로 기술한 것
- 주어진 명령어를 처리하는 컴퓨터에게 문제를 해결하도록 만드는 정형화된 절차

## 프로그래밍 언어

- 사람과 컴퓨터 사이의 의사소통 도구
- 프로그래머는 컴퓨터가 이해할 수 있는 프로그래밍 언어를 사용하여 프로그램을 작성

## 프로그래밍 언어의 분류

### 저급언어

#### 1세대

- 기계어
- 0과 1

#### 2세대

- 어셈블리어
- 명령어 영어 대체

### 고급언어

#### 3세대

- Python
- C
- Java
- Rust
- 자연어와 유사

#### 4세대

- Visual Basic
- 자동화

## 프로그램의 실행

- 고급 언어로 작성된 프로그램은 기계어밖에 인식할 수 없는 컴퓨터에 의해 실행 불가능
- 컴파일러 또는 인터프리터를 사용

## 소프트웨어

- 각각의 고유한 기능을 수행하는 하드웨어를 동작시켜 특정 작업을 수행하게 하는 명령어, 데이터 등의 일체
- 프로그램, 애플리케이션, 응용 프로그램 등으로 혼용
- 소프트웨어는 '프로그램'이라는 핵심 요소와 그 주변의 모든 것을 포함하는 개념

### 응용 소프트웨어

- 사용자의 업무나 목적에 맞게 문제 해결을 위한 처리 절차를 표현한 명령어 집합

### 시스템 소프트웨어

- 하드웨어를 제어·관리하여 응용 소프트웨어를 실행할 수 있는 환경을 제공

## 시스템 소프트웨어

- 다양한 장치들을 서로 유기적으로 작동시켜 특정 작업을 수행할 수 있는 환경을 조성하는 프로그램

### 구성

- 운영체제(커널)
    - 사용자가 컴퓨터를 효율적으로 운영·관리·사용할 수 있도록 하드웨어를 제어하는 소프트웨어

- 컴파일러
    - 소스코드를 컴퓨터가 이해할 수 있는 기계어로 번역하는 소프트웨어

- 유틸리티
    - 부가적인 기능을 제공하여 사용자가 컴퓨터를 효율적이고 편리하게 관리할 수 있도록 지원해주는 소프트웨어

## 운영체제의 개념

- 응용 소프트웨어가 효과적으로 작동할 수 있는 환경을 조성
- 컴퓨터의 자원(하드웨어)을 효율적으로 관리하고 응용 프로그램에 자원을 할당
