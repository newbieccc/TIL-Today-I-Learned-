## 2. 튜플과 문자열

### 2-1. 튜플(tuple)

- 튜플은 데이터를 순서 있게 저장하는 **불변(immutable) 시퀀스**임
- 리스트와 기본적으로 동일하게 순서를 가짐
- 생성 후 원소가 바뀌지 않도록 설계된 안정적 저장 구조임
- 원소의 순서 또한 고정됨
- 불변성으로 의도치 않은 데이터 변경을 방지함
- 리스트보다 메모리 효율과 접근 속도에서 유리할 수 있음

### 튜플 생성 구문

    tuple([원소 나열])

    (원소 나열)

### 예시

    body = (181, 78, "dark brown", "black")

### 핵심 정리

- 튜플은 리스트처럼 순서가 있지만 수정할 수 없음
- 변경되면 안 되는 데이터를 저장할 때 유용함
- 튜플은 괄호 `()`를 사용하지만, 실제로는 콤마 `,`가 튜플 생성의 핵심임

---

### 2-2. 튜플의 역할

- 튜플의 `()`은 생략 가능함
- 즉, 콤마로 분리된 모든 값들이 튜플로 처리될 수 있음

### 예시

    temp, season = 27, "summer"

    rad, hei = hei, rad

### 함수와 튜플

- 함수는 원칙적으로 단 하나의 값만 반환함
- 두 개 이상의 값을 반환하는 것처럼 보이는 경우,
  실제로는 여러 값을 콤마로 묶어 **하나의 튜플 객체**로 전달하는 것임

### 예시

    vol, surf = rtn_cone_vol_surf(r, h)

### 핵심 정리

- 튜플은 여러 값을 하나로 묶는 역할을 함
- 동시 할당과 함수의 다중 반환처럼 보이는 구조에서 자주 사용됨

---

### 2-3. 튜플 관련 연산자, 내장 함수와 메소드

### 튜플 관련 연산자

- 할당 연산자 `=` 이외의 연산자는 리스트와 거의 동일하게 사용할 수 있음
- 단, 튜플은 불변 자료형이므로 원소를 직접 수정할 수 없음

### 수정 불가 예시

    body = (181, 78, "dark brown", "black")
    body[1] = 81

- 위 코드는 튜플 원소를 수정하려 하므로 오류가 발생함

### 내장 함수

- 튜플에서도 리스트와 같이 다음 내장 함수를 사용할 수 있음

| 함수      | 의미          |
|---------|-------------|
| `len()` | 원소 개수 반환    |
| `min()` | 가장 작은 원소 반환 |
| `max()` | 가장 큰 원소 반환  |
| `sum()` | 모든 원소의 합 반환 |

### 주요 메소드

| 메소드        | 의미                     |
|------------|------------------------|
| `index(x)` | 원소 `x`의 인덱스를 반환        |
| `count(x)` | 튜플 내 원소 `x`의 출현 횟수를 반환 |

### 핵심 정리

- 튜플은 리스트와 비슷하게 조회할 수 있음
- 하지만 원소를 변경하는 메소드는 사용할 수 없음
- `index()`, `count()`처럼 값을 조회하는 메소드는 사용 가능함

---

### 2-4. 튜플의 정렬 연산

- 리스트는 `sort()` 메소드를 통해 스스로 정렬할 수 있음
- 튜플은 불변 자료형이므로 `sort()` 메소드가 없음
- 튜플을 정렬하려면 내장 함수 `sorted()`를 사용함

### 예시

    rad_list = tuple([7, 22, 4, 15, 47])
    srad_list = sorted(rad_list)

### 코드 해석

- `rad_list`
    - 튜플 형태의 데이터
- `sorted(rad_list)`
    - 튜플의 원소를 정렬한 결과를 새 리스트로 반환함

### 핵심 정리

- 튜플은 직접 정렬할 수 없음
- 정렬이 필요하면 `sorted()`를 사용함
- `sorted()` 결과는 리스트로 반환됨

---

### 2-5. 튜플의 원소 추가

- 리스트는 `append()` 메소드를 통해 원소를 추가할 수 있음
- 튜플은 불변 자료형이므로 `append()` 메소드가 없음
- 튜플에 원소를 추가하려면 `+` 연산자로 새로운 튜플을 만들어야 함

### 예시

    rad_list = tuple([7, 22, 4, 22, 47])
    rad_list = rad_list + (30,)

### 코드 해석

- `(30,)`
    - 원소가 하나인 튜플
- 튜플은 원소가 1개일 때도 콤마 `,`가 필요함

### 핵심 정리

- 튜플은 기존 원소를 직접 추가할 수 없음
- 새 튜플을 만들어 다시 변수에 저장하는 방식으로 처리함

---

### 2-6. 튜플 슬라이싱과 컴프리헨션

### 튜플 슬라이싱

- 리스트와 동일하게 `[start:end:step]` 구문을 사용할 수 있음
- 잘라낸 결과는 새로운 튜플로 생성됨
- 슬라이싱을 이용한 부분 수정은 불가능함

### 예시

    hei_list = [1, 5, 14, 26, 31]
    hei_list[1:3] = [32, 19]

    rad_list = tuple([7, 22, 4, 15, 47])
    rad_list[1:3] = [32, 19]

### 코드 해석

- 리스트는 슬라이싱 영역을 수정할 수 있음
- 튜플은 불변 자료형이므로 슬라이싱 영역 수정이 불가능함

### 튜플 컴프리헨션

- 튜플은 리스트처럼 직접적인 컴프리헨션 구문을 제공하지 않음
- 데이터를 요청할 때마다 하나씩 만드는 **제너레이터 표현식**으로 변환됨

### 핵심 정리

- 튜플은 슬라이싱 조회는 가능하지만 부분 수정은 불가능함
- 튜플 컴프리헨션은 리스트 컴프리헨션과 다르게 제너레이터 표현식으로 처리됨

## 3. 문자열

### 3-1. 문자열의 구성

- 문자열은 원소가 문자로만 고정된 **불변 시퀀스**임
- 시퀀스의 성질을 공유함
    - 인덱싱
    - 슬라이싱
    - 시퀀스 내장 함수
    - 반복 기능

### 예시

    fruit = "apple"
    print("apple"[3])

### 코드 해석

- 문자열도 시퀀스이므로 인덱스로 문자에 접근할 수 있음
- `"apple"[3]`은 네 번째 문자에 접근함

### 핵심 정리

- 문자열은 문자들의 시퀀스임
- 리스트처럼 인덱싱과 슬라이싱이 가능함
- 하지만 문자열은 불변 자료형이므로 직접 수정할 수 없음

---

### 3-2. 문자열 메소드

| 메소드                 | 반환 타입  | 의미                        |
|---------------------|--------|---------------------------|
| `split(str)`        | `list` | 특정 문자를 기준으로 쪼개어 리스트로 변환   |
| `join(list)`        | `str`  | 리스트 등의 문자열들을 하나로 합침       |
| `find(str)`         | `int`  | 찾는 문자가 없으면 `-1` 반환        |
| `replace(old, new)` | `str`  | 특정 문자를 다른 문자로 바꾼 새 문자열 반환 |
| `strip()`           | `str`  | 양쪽 공백 또는 특정 문자를 제거        |
| `upper()`           | `str`  | 대문자로 변환                   |
| `lower()`           | `str`  | 소문자로 변환                   |
| `isdigit()`         | `bool` | 숫자로만 구성되어 있는지 확인          |
| `isalpha()`         | `bool` | 알파벳으로만 구성되어 있는지 확인        |
| `isalnum()`         | `bool` | 알파벳과 숫자로만 구성되어 있는지 확인     |

### 핵심 정리

- 문자열 메소드는 문자열을 다양한 방식으로 처리할 수 있게 해 줌
- 문자열은 불변 자료형이므로, 메소드 실행 결과는 보통 새 문자열로 반환됨

---

### 3-3. 문자열과 리스트

### `split()` 사용

    msg = "I do love python"
    msg_list = msg.split(" ")

    msg_list2 = "I do love python".split(" ")

### 코드 해석

- `" "` 공백을 기준으로 문자열을 나눔
- 결과는 다음과 같은 리스트가 됨

  ["I", "do", "love", "python"]

### `join()` 사용

    hei_list = [1, 5, 14, 26, 31]
    hei_list = [str(i) for i in hei_list]
    hei_csv = ",".join(hei_list)

### 코드 해석

- 숫자 리스트를 문자열 리스트로 변환한 뒤
  `join()`으로 하나의 문자열로 합침
- 결과는 CSV 형태의 문자열이 될 수 있음

### 핵심 정리

- `split()`은 문자열을 리스트로 바꿈
- `join()`은 리스트의 문자열 원소들을 하나의 문자열로 합침
- 문자열과 리스트는 데이터 전처리에서 자주 함께 사용됨

---

### 3-4. 문자열 실습 예시: 팰린드롬 검사

### 팰린드롬

- 앞에서 읽어도 뒤에서 읽어도 같은 문자열을 의미함
- 예:
    - `토마토`
    - `level`
    - `madam`

### 처리 흐름

1. 문자열을 소문자로 변환
2. 알파벳과 숫자만 남김
3. 문자열을 앞뒤로 비교
4. 같으면 팰린드롬, 다르면 팰린드롬 아님

### 예시 코드

    def is_palindrome(text):
        cleaned = [ch.lower() for ch in text if ch.isalnum()]

        return cleaned == cleaned[::-1]

    text = input("문장을 입력하세요: ")

    if is_palindrome(text):
        print("팰린드롬입니다.")
    else:
        print("팰린드롬이 아닙니다.")

### 핵심 정리

- `lower()`로 대소문자를 통일함
- `isalnum()`으로 문자와 숫자만 남김
- `[::-1]`로 역순을 만들어 비교함

---

### 3-5. 문자열과 리스트 실습 예시: 점수 처리

### 처리 흐름

1. 입력받은 문자열을 콤마 기준으로 분리함
2. 각 값을 정수로 변환함
3. 정렬함
4. 평균과 중간값을 계산함

### 예시 코드

    score = input("점수를 입력하세요.(,로 분리): ").split(",")
    score = [int(s) for s in score]

    score.sort()
    score = score[::-1]

    avg = sum(score) / len(score)
    print("평균 점수는", str(format(avg, ".2f")) + "입니다.")

    mid_idx = len(score) // 2
    print("중간값은", str(score[mid_idx]) + "입니다.")

### 핵심 정리

- `split(",")`으로 입력 문자열을 리스트로 변환함
- 리스트 컴프리헨션으로 각 원소를 정수형으로 변환함
- `sum()`, `len()`을 이용해 평균을 구할 수 있음

## 4. 세트

### 4-1. 세트(set)

- 세트는 중복을 허용하지 않고 순서를 유지하지 않는 **변경 가능 컬렉션**임
- 수학적 집합 개념을 구현한 데이터 구조임
- 원소 간의 고유성(uniqueness)을 보장함
- 중복된 값을 자동으로 제거하여 데이터 정제에 유용함

### 세트 생성 구문

    set([원소 나열])

    set("원소 나열")

    {원소 나열}

### 예시

    rad_list = set([7, 22, 4, 22, 47])

### 핵심 정리

- 세트는 중복을 제거함
- 순서 개념이 없으므로 인덱싱이나 슬라이싱이 불가능함
- 고유한 값만 남기고 싶을 때 유용함

---

### 4-2. 세트 관련 연산자

### 기본 연산

- 할당 연산자 `=` 사용 가능
- 멤버십 연산자 `in`, `not in` 사용 가능
- 산술 연산자 `+`, `*`는 사용 불가

### 비교 연산자

| 연산자        | 의미                       |
|------------|--------------------------|
| `<`, `<=`  | 왼쪽 세트가 오른쪽 세트의 부분집합인지 확인 |
| `>`, `>=`  | 왼쪽 세트가 오른쪽 세트의 상위집합인지 확인 |
| `==`, `!=` | 순서와 관계없이 원소 구성이 같은지 비교   |

### 예시

    rad_list = set([7, 22, 4, 22, 47])
    rad_list2 = set([22, 4, 7, 47])

### 핵심 정리

- 세트는 순서가 없으므로 리스트처럼 연결하거나 반복할 수 없음
- 세트 비교 연산은 크기 비교가 아니라 집합 관계 비교임
- `==`는 순서가 아니라 원소 구성이 같은지를 비교함

---

### 4-3. 세트 관련 내장 함수

- 세트는 중복을 허용하지 않으므로 다른 컬렉션 객체와 일부 실행 결과가 다를 수 있음

| 함수       | 의미                   |
|----------|----------------------|
| `len(s)` | 중복이 제거된 후의 원소 개수     |
| `min(s)` | 세트에서 가장 작은 원소        |
| `max(s)` | 세트에서 가장 큰 원소         |
| `sum(s)` | 모든 원소의 합, 중복은 1회만 반영 |

### 예시

    rad_list = {7, 22, 4, 22, 47}

    len(rad_list)
    sum(rad_list)

### 핵심 정리

- 세트는 중복을 제거한 후 계산함
- `sum()`도 중복된 값은 한 번만 반영함

---

### 4-4. 세트 주요 메소드

| 메소드               | 의미                |
|-------------------|-------------------|
| `issuperset(s)`   | 자신의 상위집합 여부 검사    |
| `issubset(s)`     | 자신의 부분집합 여부 검사    |
| `add(x)`          | 원소 `x` 추가         |
| `remove(x)`       | 원소 `x` 제거         |
| `intersection(s)` | 세트 `s`와 교집합 결과 반환 |
| `union(s)`        | 세트 `s`와 합집합 결과 반환 |
| `difference(s)`   | 세트 `s`와 차집합 결과 반환 |

### 핵심 정리

- 세트는 집합 연산에 유용함
- 교집합, 합집합, 차집합 등을 메소드로 처리할 수 있음

---

### 4-5. 세트의 슬라이싱과 컴프리헨션

### 세트 슬라이싱

- 세트는 순서 개념이 없는 비순서 컬렉션임
- 따라서 슬라이싱이 불가능함

### 세트 컴프리헨션

- 컬렉션으로부터 조건 필터링, 변환, 중복 제거가 가능함

### 구문 형식

    {표현식 for 변수 in 컬렉션 if 조건식}

### 예시

    rad_list = [7, 22, 4, 22, 47]
    even_rad = {x for x in rad_list if x % 2 == 0}

### 핵심 정리

- 세트 컴프리헨션은 중복 제거와 조건 필터링에 유용함
- 결과는 순서를 보장하지 않음

---

### 4-6. 세트 실습 예시: 스팸 제목 판별

### 처리 흐름

1. 메일 제목을 대소문자 중 하나로 통일함
2. `replace()`로 특수 기호를 제거함
3. 정제된 텍스트를 공백 단위로 쪼갬
4. 리스트를 세트로 변환함
5. 금칙어 세트와 비교함

### 예시 코드

    banned_set = {"free", "money", "lucky", "행운", "바보", "성인"}

    def is_spam(title):
        title = title.lower()
        title = title.replace("!", " ")
        title = title.replace("?", " ")

        words = title.split(" ")
        word_set = set(words)

        if word_set & banned_set:
            return True
        else:
            return False

    title = input("제목을 입력하세요: ")

    if is_spam(title):
        print("스팸입니다.")
    else:
        print("스팸이 아닙니다.")

### 핵심 정리

- 문자열 정제 후 세트로 변환하면 중복을 제거하고 비교하기 쉬움
- 금칙어 세트와 교집합이 있으면 스팸으로 판단할 수 있음

---

## 5. 딕셔너리

### 5-1. 딕셔너리(dictionary)

- 딕셔너리는 `key-value` 쌍으로 데이터를 저장하는 **매핑(mapping) 자료형**임
- 키에 대응되는 값을 빠르게 검색할 수 있도록 설계된 구조임
- 키는 중복되지 않아야 함
- 값은 어떤 타입의 데이터도 저장할 수 있음

### 딕셔너리 생성 구문

    {"키1": "값1", "키2": "값2", ..., "키n": "값n"}

### 예시

    body = {
        "키": 181,
        "몸무게": 78,
        "눈": "dark brown",
        "머리": "black"
    }

### 핵심 정리

- 딕셔너리는 key를 통해 value에 접근함
- 인덱스가 아니라 의미 있는 키로 데이터를 관리함
- 검색 속도가 빠르고 데이터 구조화에 유용함

---

### 5-2. 딕셔너리 관련 연산자

### 멤버십 연산자

- `in`, `not in`은 **키의 존재 여부**를 확인함

### 접근 연산자

- 딕셔너리 값에는 대괄호 `[]`를 통해 접근함

### 예시

    d1 = {"a": 10, "b": 20, "c": 30, "d": 40}

    print(d1["a"])

### 사용할 수 없는 연산

- 산술 연산자 `+`, `*`는 사용 불가
- 비교 연산자 `<`, `>`, `<=`, `>=`는 사용 불가
- `==`, `!=`는 key-value 쌍이 같은지 비교함
- 순서는 무관함

### 주의

    print(d1[40])

- 위 코드는 키 `40`이 없으므로 오류가 발생할 수 있음

### 핵심 정리

- 딕셔너리는 key를 기준으로 접근함
- `in`은 값이 아니라 key 존재 여부를 확인함
- 없는 key에 접근하면 오류가 발생할 수 있음

---

### 5-3. 딕셔너리 관련 내장 함수

| 함수                | 의미              |
|-------------------|-----------------|
| `len(s)`          | key-value 쌍의 개수 |
| `min(s.values())` | 값 중 가장 작은 값     |
| `min(s.keys())`   | 키 중 가장 작은 값     |
| `max(s.values())` | 값 중 가장 큰 값      |
| `max(s.keys())`   | 키 중 가장 큰 값      |

### 예시

    basic_parts = {
        "CPU": 450000,
        "RAM": 120000
    }

    len(basic_parts)
    min(basic_parts.values())

### 핵심 정리

- 딕셔너리는 key와 value를 구분해서 다루어야 함
- `keys()`, `values()`를 활용하면 키나 값만 대상으로 연산할 수 있음

---

### 5-4. 딕셔너리의 생성 및 조작

### 예시

    pc_parts = {
        "CPU": 950000,
        "RAM": 420000,
        "SSD": 195000,
        "GPU": 1850000
    }

    pc_parts["MainBoard"] = 230000
    pc_parts["CPU"] = 920000

### 코드 해석

- `pc_parts["MainBoard"] = 230000`
    - 새로운 key-value 쌍을 추가함
- `pc_parts["CPU"] = 920000`
    - 기존 key `"CPU"`에 해당하는 값을 변경함

### 핵심 정리

- 존재하지 않는 key에 값을 할당하면 새 항목이 추가됨
- 이미 존재하는 key에 값을 할당하면 기존 값이 수정됨

---

### 5-5. 딕셔너리의 슬라이싱과 컴프리헨션

### 딕셔너리 슬라이싱

- 딕셔너리는 순서 개념이 없는 비순서 컬렉션으로 다루므로 슬라이싱이 불가능함

### 딕셔너리 컴프리헨션

- key와 value에 대한 조작을 동시에 할 수 있음

### 구문 형식

    {키 표현식: 값 표현식 for 키변수, 값변수 in 딕셔너리.items() if 조건식}

### 예시

    pc_parts = {
        "CPU": 950000,
        "RAM": 420000,
        "SSD": 195000,
        "GPU": 1850000,
        "Mainboard": 330000,
        "Case": 75000,
        "Power": 90000
    }

    a_parts = {k: v for k, v in pc_parts.items() if v <= 200000}

    usd_parts = {k.lower(): round(v / 1450)
                 for k, v in pc_parts.items()}

    dc_parts = {p: pc_parts[p] * 0.9
                for p in pc_parts}

### 코드 해석

- `a_parts`
    - 가격이 200000 이하인 부품만 추출
- `usd_parts`
    - key를 소문자로 바꾸고, 값을 달러 기준으로 변환
- `dc_parts`
    - 모든 부품 가격에 0.9를 곱해 할인 가격 계산

### 핵심 정리

- 딕셔너리 컴프리헨션은 key와 value를 동시에 변환하거나 필터링할 수 있음
- `.items()`를 사용하면 key와 value를 함께 꺼낼 수 있음

---

### 5-6. 딕셔너리 주요 메소드

| 메소드         | 의미                          |
|-------------|-----------------------------|
| `keys()`    | 포함된 모든 키를 반환                |
| `values()`  | 포함된 모든 값을 반환                |
| `items()`   | `(키, 값)` 형태의 항목들을 반환        |
| `clear()`   | 모든 항목 삭제                    |
| `get(key)`  | key에 해당하는 값을 반환             |
| `pop(key)`  | key에 해당하는 값을 반환하고 해당 항목 삭제  |
| `popitem()` | 무작위로 한 `(키, 값)` 항목을 반환하고 삭제 |

### 핵심 정리

- `keys()`, `values()`, `items()`는 딕셔너리 순회와 변환에 자주 사용됨
- `get()`은 key에 해당하는 값을 안전하게 가져올 때 유용함
- `pop()`은 값을 가져오면서 항목을 삭제함

---

### 5-7. 딕셔너리 실습 예시: 모스 부호 변환

### 처리 흐름

1. 문자와 모스 부호를 딕셔너리로 저장함
2. 사용자가 입력한 문장을 대문자로 변환함
3. 각 문자를 딕셔너리 key로 조회함
4. 해당 모스 부호를 리스트에 추가함
5. `join()`으로 하나의 문자열로 출력함

### 예시 코드

    MORSE_T = {
        "A": ".-",
        "B": "-...",
        "C": "-.-.",
        "D": "-.."
    }

    in_text = input("문장을 영어로 입력하세요: ")

    out_text = []

    for ch in in_text.upper():
        if ch in MORSE_T:
            out_text.append(MORSE_T[ch])

    out_text = " ".join(out_text)

    print(out_text)

### 핵심 정리

- 딕셔너리는 문자 → 모스 부호처럼 대응 관계를 저장하기에 적합함
- key로 문자를 조회하여 value인 모스 부호를 얻을 수 있음
- 리스트에 결과를 모은 뒤 `join()`으로 출력하면 깔끔함

## 6. 정리하기

1. 튜플은 데이터를 순서 있게 저장하는 불변 시퀀스임
    - 리스트와 유사하지만 생성 후 원소를 수정할 수 없음

2. 튜플은 괄호보다 콤마가 본질적인 생성 요소임
    - 여러 값을 하나로 묶어 반환하거나 여러 변수에 나누어 대입할 때 활용됨

3. 튜플은 인덱싱과 슬라이싱이 가능하지만 `sort()`나 `append()`처럼 값을 바꾸는 메소드는 사용할 수 없음
    - 정렬은 `sorted()` 사용
    - 원소 추가는 `+` 연산자 사용

4. 문자열은 문자로만 이루어진 불변 시퀀스임
    - 인덱싱, 슬라이싱, 반복, 시퀀스 관련 기능을 사용할 수 있음

5. 문자열 메소드에는 `split()`, `join()`, `find()`, `replace()`, `strip()`, `upper()`, `lower()`, `isdigit()`, `isalpha()`,
   `isalnum()` 등이 있음

6. 세트는 중복을 허용하지 않고 순서를 유지하지 않는 변경 가능 컬렉션임
    - 데이터 중복 제거, 집합 연산에 유용함

7. 세트 컴프리헨션은 다른 콜렉션에서 조건에 맞는 값을 추출하거나 변환해 중복을 제거하는 데 활용됨

8. 딕셔너리는 key-value 쌍으로 데이터를 저장하는 매핑 자료형임
    - key를 이용해 value를 빠르게 접근할 수 있음
    - 새로운 항목 추가나 기존 값 수정이 가능함

9. 딕셔너리는 순서와 상관없이 key-value 쌍을 관리함
    - `keys()`, `values()`, `items()`, `get()`, `pop()`, `popitem()` 등의 메소드를 활용해 데이터를 효율적으로 처리할 수 있음

## 7. 핵심 요약

### 7-1. 튜플

- 순서 있음
- 불변 자료형
- 괄호 `()` 또는 콤마로 생성
- 인덱싱, 슬라이싱 가능
- 수정, 추가, 삭제 불가
- `index()`, `count()` 사용 가능
- 정렬은 `sorted()` 사용
- 원소 추가는 새 튜플을 만들어야 함

### 7-2. 문자열

- 문자로 이루어진 불변 시퀀스
- 인덱싱, 슬라이싱 가능
- 주요 메소드:
    - `split()`
    - `join()`
    - `find()`
    - `replace()`
    - `strip()`
    - `upper()`
    - `lower()`
    - `isdigit()`
    - `isalpha()`
    - `isalnum()`

### 7-3. 세트

- 중복 허용 안 함
- 순서 없음
- 변경 가능
- 인덱싱, 슬라이싱 불가
- 집합 연산에 적합
- 주요 메소드:
    - `add()`
    - `remove()`
    - `intersection()`
    - `union()`
    - `difference()`
    - `issuperset()`
    - `issubset()`

### 7-4. 딕셔너리

- key-value 쌍으로 저장
- key는 중복 불가
- value는 어떤 타입도 가능
- key를 이용해 값에 접근
- 주요 메소드:
    - `keys()`
    - `values()`
    - `items()`
    - `clear()`
    - `get()`
    - `pop()`
    - `popitem()`

### 7-5. 자료형 비교

| 튜플   | 있음     | 불가능   | 가능        | 인덱스   |
| 문자열  | 있음     | 불가능   | 가능        | 인덱스   |
| 세트   | 없음     | 가능    | 불가능       | 원소    |
| 딕셔너리 | key 기준 | 가능    | key 중복 불가 | key   |

## 8. 연습문제 정리

### Q1. 다음 중 튜플에 대한 설명으로 옳은 것은?

1. 원소를 자유롭게 수정할 수 있는 가변 시퀀스
2. 중복을 허용하지 않는 비순서 컬렉션
3. key-value의 쌍으로 데이터를 저장하는 자료형
4. 원소의 순서를 유지하는 불변 시퀀스

- 정답: **4**

### 해설

- 튜플은 순서를 유지하는 시퀀스 자료형임
- 하지만 생성 후 원소를 수정할 수 없는 불변 자료형임
- 따라서 정답은 **원소의 순서를 유지하는 불변 시퀀스**임

### Q2. 다음 코드의 출력값으로 옳은 것은?

    msg = "I do love python"
    print(msg.split(" "))

1. `"I do love python"`
2. `["I", "do", "love", "python"]`
3. `("I", "do", "love", "python")`
4. `[I do love python]`

- 정답: **2**

### 해설

- `split(" ")`은 공백을 기준으로 문자열을 나누어 리스트로 반환함
- 따라서 출력 결과는 다음과 같음

  ["I", "do", "love", "python"]

---

### Q3. 다음 중 세트에 사용할 수 없는 연산자는 무엇인가?

1. `in`
2. `not in`
3. `+`
4. `==`

- 정답: **3**

### 해설

- 세트는 멤버십 연산자인 `in`, `not in`을 사용할 수 있음
- 같은 원소 구성을 비교하는 `==`도 사용할 수 있음
- 하지만 세트는 순서가 없는 컬렉션이므로 `+`를 이용한 연결 연산은 사용할 수 없음

## 9. 시험 직전 체크포인트

- 튜플은 순서가 있는 불변 시퀀스이다.
- 튜플은 리스트와 비슷하지만 원소 수정이 불가능하다.
- 튜플은 괄호보다 콤마가 생성의 핵심이다.
- `sorted(튜플)`은 정렬된 리스트를 반환한다.
- 원소 1개짜리 튜플은 `(30,)`처럼 콤마가 필요하다.
- 문자열은 문자로만 구성된 불변 시퀀스이다.
- `split()`은 문자열을 리스트로 나눈다.
- `join()`은 문자열 리스트를 하나의 문자열로 합친다.
- `find()`는 찾는 문자가 없으면 `-1`을 반환한다.
- 세트는 중복을 허용하지 않고 순서가 없다.
- 세트는 인덱싱과 슬라이싱이 불가능하다.
- 세트에서 `+`, `*` 연산자는 사용할 수 없다.
- 세트의 `<`, `<=`, `>`, `>=`는 부분집합·상위집합 관계를 의미한다.
- 딕셔너리는 key-value 쌍으로 데이터를 저장한다.
- 딕셔너리에서 `in`은 key의 존재 여부를 확인한다.
- 딕셔너리는 `+`, `*`, `<`, `>` 연산을 사용할 수 없다.
- `keys()`, `values()`, `items()`는 딕셔너리 처리에서 중요하다.
- 딕셔너리 컴프리헨션은 `{키 표현식: 값 표현식 for ...}` 형태로 작성한다.
