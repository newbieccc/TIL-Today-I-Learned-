## 2. 선택 구조의 이해

### 2-1. 선택 구조의 정의

- 선택 구조는 특정 영역 내의 명령문에 대한 실행 여부를
  프로그램 실행 과정 중 **조건의 참·거짓**에 따라 결정하는 구조임
- 동일한 입력에 대해 항상 같은 행동을 하는 것이 아니라,
  상황에 따라 다르게 행동하게 만드는 지능의 시작이라고 볼 수 있음
- 오류 처리, 예외 상황 대응 등 프로그램의 안정성을 책임짐

### 2-2. 선택 구조의 구문 형식

- 파이썬에서 선택 구조는 `if`문으로 표현함

  if 불리언식:
  명령문1
  ...
  명령문n

### 2-3. 코드 블록

- 동일한 깊이의 들여쓰기는 코드 블록을 표현함
- 코드 블록은 특정 동작을 수행하는 한 라인 이상의 명령문 집합임
- 파이썬에서는 `{}`가 아니라 **들여쓰기**로 블록을 구분함
- 들여쓰기는 스페이스 4칸을 권고함(PEP-8)

### 2-4. 선택 구조의 핵심 흐름

- `if` 뒤의 조건식이 `True`이면 들여쓰기 된 명령문 집합을 실행함
- 조건식이 `False`이면 해당 명령문 집합을 실행하지 않고 다음 코드로 넘어감

### 핵심 정리

- 선택 구조는 조건에 따라 실행 여부를 결정하는 구조임
- 파이썬에서는 `if 조건식:` 형태로 작성함
- `if`문 안에서 실행할 명령문은 반드시 들여쓰기 해야 함

## 3. 불리언식과 비교 연산자

### 3-1. 불리언식

- 비교 연산자를 사용하여 결과가 **불리언(Boolean) 타입**으로 생성되는 표현식
- 불리언 타입은 `True` 또는 `False` 값만 가질 수 있음
- 선택 구조, 반복 구조의 조건을 판단할 때 사용됨

### 3-2. 비교 연산자의 종류

| 연산자  | 수학적 표현 | 의미     |
|------|--------|--------|
| `<`  | `<`    | 작다     |
| `<=` | `≤`    | 작거나 같다 |
| `>`  | `>`    | 크다     |
| `>=` | `≥`    | 크거나 같다 |
| `==` | `=`    | 같다     |
| `!=` | `≠`    | 같지 않다  |

### 3-3. 비교 연산 예시

    3 > 6
    suf == vol

    light_on = 3 > 6
    isStop = suf == vol

### 코드 해석

- `3 > 6`
    - 3이 6보다 큰지 비교함
    - 결과는 `False`
- `suf == vol`
    - `suf`와 `vol` 값이 같은지 비교함
- `light_on = 3 > 6`
    - 비교 결과인 `False`가 `light_on` 변수에 저장됨
- `isStop = suf == vol`
    - 비교 결과인 `True` 또는 `False`가 `isStop`에 저장됨

### 핵심 정리

- 비교 연산자의 결과는 항상 `True` 또는 `False`임
- `=`는 대입 연산자이고, `==`는 비교 연산자임
- 조건문에서는 보통 비교 연산을 통해 조건을 만듦

## 4. 연산자 우선순위

### 4-1. 연산자 우선순위의 의미

- 표현식에 여러 연산자가 사용될 때
  어떤 연산을 먼저 수행할지 결정하는 기준임

### 4-2. 연산자 우선순위 기본

1. 괄호 내부의 수식
2. 거듭제곱(`**`) 연산자
3. 곱셈, 실수 나눗셈, 정수 나눗셈, 모듈로 연산자
    - `*`, `/`, `//`, `%`
    - 왼쪽에서 오른쪽 순서로 적용
4. 덧셈, 뺄셈 연산자
    - `+`, `-`
    - 왼쪽에서 오른쪽 순서로 적용
5. 비교 연산자
    - `<`, `<=`, `>`, `>=`, `==`, `!=`
6. 할당 연산자
    - `=`

### 4-3. 논리 연산자 포함 우선순위

1. 괄호 내부의 수식
2. 거듭제곱(`**`) 연산자
3. 곱셈, 나눗셈, 정수 나눗셈, 모듈로 연산자
4. 덧셈, 뺄셈 연산자
5. 비교 연산자
6. 논리 부정(`not`)
7. 논리곱(`and`)
8. 논리합(`or`)
9. 할당 연산자

### 핵심 정리

- 괄호가 가장 먼저 계산됨
- 비교 연산은 산술 연산보다 뒤에 수행됨
- 논리 연산자 사이에서는 `not` → `and` → `or` 순서임
- 복잡한 조건식은 괄호를 사용하면 더 명확함

## 5. 논리 연산자

### 5-1. 논리 연산자의 정의

- 참, 거짓을 구별할 수 있는 명제를 대상으로
  명제의 집합을 위해 고안한 연산자임
- 명제는 참과 거짓 중 하나의 진리값만을 갖는 서술문임
- 논리 연산자는 불리언식에 적용하여 또 하나의 불리언 값을 산출하는 연산 체계임
- 두 개의 논리값을 연산하여 `True` 또는 `False`를 결과로 생성함

### 5-2. 논리 연산자의 종류

| 연산자   | 의미   | 형태     |
|-------|------|--------|
| `and` | 논리곱  | 이항 연산자 |
| `or`  | 논리합  | 이항 연산자 |
| `not` | 논리부정 | 단항 연산자 |

### 핵심 정리

- `and`, `or`는 두 개의 조건을 연결함
- `not`은 하나의 조건 결과를 반대로 바꿈
- 논리 연산자는 왼쪽에서 오른쪽 방향으로 결합함

---

## 6. `and` 연산자

### 6-1. 정의

- 두 논리값이 모두 `True`일 때만 `True`를 반환함
- 둘 중 하나라도 `False`이면 `False`를 반환함

### 6-2. 진리표

| b1    | b2    | b1 and b2 |
|-------|-------|-----------|
| False | False | False     |
| False | True  | False     |
| True  | False | False     |
| True  | True  | True      |

### 6-3. 예시

    temp >= 27 and fruit == "apple"
    rad > 0 and hei > 0

### 코드 해석

- `temp >= 27 and fruit == "apple"`
    - 온도가 27 이상이고 과일이 `"apple"`일 때만 `True`
- `rad > 0 and hei > 0`
    - 반지름과 높이가 모두 0보다 커야 `True`

### 핵심 정리

- `and`는 모든 조건이 참이어야 함
- 하나라도 거짓이면 전체 결과는 거짓임

---

## 7. `or` 연산자

### 7-1. 정의

- 두 논리값이 모두 `False`일 때만 `False`를 반환함
- 둘 중 하나라도 `True`이면 `True`를 반환함

### 7-2. 진리표

| b1    | b2    | b1 or b2 |
|-------|-------|----------|
| False | False | False    |
| False | True  | True     |
| True  | False | True     |
| True  | True  | True     |

### 7-3. 예시

    temp >= 27 or fruit == "apple"
    rad > 0 or hei > 0

### 코드 해석

- `temp >= 27 or fruit == "apple"`
    - 온도가 27 이상이거나 과일이 `"apple"`이면 `True`
- `rad > 0 or hei > 0`
    - 반지름 또는 높이 중 하나라도 0보다 크면 `True`

### 핵심 정리

- `or`는 조건 중 하나만 참이어도 전체 결과가 참임
- 모든 조건이 거짓일 때만 거짓임

---

## 8. `not` 연산자

### 8-1. 정의

- 단항 연산자로 논리값을 반전함
- `False`는 `True`로, `True`는 `False`로 반환함

### 8-2. 진리표

| b1    | not(b1) |
|-------|---------|
| False | True    |
| True  | False   |

### 8-3. 예시

    not(temp >= 27)

### 코드 해석

- `temp >= 27`의 결과가 `True`이면 `not` 적용 후 `False`
- `temp >= 27`의 결과가 `False`이면 `not` 적용 후 `True`

### 핵심 정리

- `not`은 조건 결과를 반대로 뒤집음
- 조건을 부정할 때 사용함

---

## 9. 단락 평가

### 9-1. 단락 평가의 정의

- 단락 평가(short-circuit evaluation)는
  불리언식의 논리 연산 결과가 이미 결정되면
  전체 연산 결과를 도출하기 전 나머지 피연산자를 평가하지 않는 기법임

### 9-2. 대상 연산자

- `and`
- `or`

### 9-3. `and`에서의 단락 평가

- `A and B`에서 `A`가 `False`이면
  전체 결과는 이미 `False`로 결정됨
- 따라서 `B`는 평가하지 않음

  temp >= 27 and season == "summer"

### 9-4. `or`에서의 단락 평가

- `A or B`에서 `A`가 `True`이면
  전체 결과는 이미 `True`로 결정됨
- 따라서 `B`는 평가하지 않음

  temp >= 27 or season == "summer"

### 핵심 정리

- `and`
    - 앞 조건이 `False`이면 뒤 조건을 평가하지 않음
- `or`
    - 앞 조건이 `True`이면 뒤 조건을 평가하지 않음
- 불필요한 연산을 줄이고 오류 발생 가능성을 낮출 수 있음

## 10. 선택 구조의 확장

### 10-1. 이분 선택 구조

- 이분 선택 구조는 조건이 참일 때와 거짓일 때
  각각 다른 명령문 집합을 실행하는 구조임
- `if ~ else` 구문을 사용함

### 10-2. 구문 형식

    if 불리언식:
        명령문 집합1
    else:
        명령문 집합2

### 10-3. 제어 흐름

- 불리언식이 `True`이면 `명령문 집합1` 실행
- 불리언식이 `False`이면 `명령문 집합2` 실행
- 두 명령문 집합 중 하나만 실행됨

### 10-4. 예시

    temp = int(input("온도를 입력하세요: "))

    if temp <= 0:
        print("겨울입니다.")
    else:
        print("겨울이 아닙니다.")

### 핵심 정리

- `if`만 사용하면 조건이 참일 때만 실행함
- `if ~ else`는 참과 거짓 각각의 경우를 모두 처리함
- `else` 뒤에도 콜론(`:`)을 붙임
- `if`와 `else` 아래 명령문은 들여쓰기 해야 함

## 11. 핵심 요약

### 11-1. 선택 구조

- 조건의 참·거짓에 따라 명령문의 실행 여부를 결정하는 구조임
- 프로그램이 상황에 따라 다르게 동작하도록 만들어 줌
- 오류 처리와 예외 상황 대응에 활용됨

### 11-2. `if`문

- 기본 형식

  if 불리언식:
  명령문

- 조건이 `True`일 때만 명령문을 실행함

### 11-3. 들여쓰기

- 파이썬은 들여쓰기로 코드 블록을 구분함
- 동일한 깊이의 들여쓰기가 같은 블록을 의미함
- 스페이스 4칸을 권고함

### 11-4. 비교 연산자

- 비교 결과는 `True` 또는 `False`임
- 대표 비교 연산자:
    - `<`
    - `<=`
    - `>`
    - `>=`
    - `==`
    - `!=`

### 11-5. 논리 연산자

- `and`
    - 모두 참일 때만 참
- `or`
    - 하나라도 참이면 참
- `not`
    - 참과 거짓을 반전

### 11-6. 이분 선택 구조

- `if ~ else` 구조를 사용함
- 조건이 참일 때와 거짓일 때를 나누어 처리함

## 12. 연습문제 정리

### Q1. 파이썬에서 명령어의 논리적 집합인 명령 블록을 표현하기 위한 방법은?

1. 불리언식
2. 들여쓰기
3. 콜론(`:`)
4. 중괄호(`{}`)

- 정답: **2**

### 해설

- 파이썬은 명령 블록을 중괄호가 아니라 **들여쓰기**로 표현함
- 동일한 깊이의 들여쓰기가 하나의 코드 블록을 의미함

---

### Q2. 다음은 온도(`temp`)가 0 이하일 경우 `"겨울입니다."`를 출력하는 프로그램의 일부이다.

밑줄 친 빈칸에 들어갈 명령문은?

    temp = int(input("온도를 입력하세요: "))

    (____________________)
        print("겨울입니다.")

1. `temp > 0`
2. `temp <= 0`
3. `if temp > 0`
4. `if temp <= 0 :`

- 정답: **4**

### 해설

- 조건문은 `if 조건식:` 형태로 작성해야 함
- 0 이하일 경우 출력해야 하므로 조건은 `temp <= 0`이어야 함
- 따라서 올바른 명령문은 다음과 같음

  if temp <= 0:
  print("겨울입니다.")

---

### Q3. 사용자가 입력한 수가 3과 5의 공배수인 경우

`"3과 5의 공배수입니다."`, 아닐 경우  
`"3과 5의 공배수가 아닙니다."`를 출력하려고 한다.  
밑줄 친 빈칸에 들어갈 연산자는?

    guess = int(input("온도를 입력하세요: "))

    if guess % 3 == 0 (____) guess % 5 == 0:
        print("3과 5의 공배수입니다.")
    else:
        print("3과 5의 공배수가 아닙니다.")

1. `and`
2. `or`
3. `True`
4. `False`

- 정답: **1**

### 해설

- 3과 5의 공배수는 3으로도 나누어떨어지고,
  5로도 나누어떨어지는 수임
- 두 조건이 모두 참이어야 하므로 `and`를 사용함

  if guess % 3 == 0 and guess % 5 == 0:
  print("3과 5의 공배수입니다.")
  else:
  print("3과 5의 공배수가 아닙니다.")

## 13. 시험 직전 체크포인트

- 선택 구조는 조건의 참·거짓에 따라 실행 여부를 결정한다.
- 파이썬의 코드 블록은 들여쓰기로 표현한다.
- `if 조건식:` 뒤에는 반드시 콜론(`:`)을 붙인다.
- `True`, `False`는 불리언 타입의 값이다.
- 비교 연산자의 결과는 불리언 값이다.
- `==`는 같다, `!=`는 같지 않다를 의미한다.
- `and`는 모든 조건이 참이어야 참이다.
- `or`는 하나 이상의 조건이 참이면 참이다.
- `not`은 논리값을 반대로 바꾼다.
- `and`와 `or`는 단락 평가가 적용된다.
- `if ~ else`는 참과 거짓 두 경우를 나누어 처리한다.
