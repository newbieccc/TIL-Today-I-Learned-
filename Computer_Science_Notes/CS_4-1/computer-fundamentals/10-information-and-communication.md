## 1. 정보통신 개요

### 1.1 정보통신 개념

정보통신은 디지털 데이터와 디지털 정보를 통신하는 것을 의미한다.

| 구분    | 의미                          | 예              |
|-------|-----------------------------|----------------|
| 통신    | 정보를 주고받는 행위                 | 서신, 봉화, 전보, 전화 |
| 정보통신  | 디지털 데이터·디지털 정보의 통신          | 컴퓨터 간 데이터 전송   |
| 정보통신망 | 정보기술 + 통신기술 + 네트워크기술이 결합된 망 | 컴퓨터통신망         |

정보통신망은 컴퓨터통신망이라고도 부른다.  
이때 컴퓨터통신망은 컴퓨터의 역할을 강조한 용어이다.

---

### 1.2 정보통신의 역사

정보통신은 전기통신 이전의 넓은 의미의 통신에서 출발하여, 데이터 통신과 컴퓨터 통신망으로 발전하였다.

| 구분     | 내용                |
|--------|-------------------|
| 광의의 통신 | 전기통신 이전의 통신       |
| 예      | 서신, 봉화, 기타 다양한 기호 |
| 협의의 통신 | 데이터 통신 중심         |

---

### 1.3 주요 정보통신 역사

| 연도     | 사건              | 핵심 내용                                |
|--------|-----------------|--------------------------------------|
| 1837년  | 모스 부호           | 사무엘 모스가 개발                           |
| 1844년  | 최초의 전보          | 전보 통신 시작                             |
| 1876년  | 최초의 음성통화        | 알렉산더 벨                               |
| 1940년  | 원격 계산기와 데이터 통신  | 조지 스티비츠가 전화기를 이용하여 원격 계산기에 데이터 통신 성공 |
| 1958년  | SAGE 시스템        | 컴퓨터와 통신을 결합한 최초의 컴퓨터 통신 시스템          |
| 1964년  | SABRE 시스템       | American Airline사의 여객기 좌석 예약 업무 처리   |
| 1968년  | ALOHA 시스템       | 하와이 대학의 실험적 무선 패킷 교환 네트워크            |
| 1969년  | ARPANET 구축      | 인터넷의 전신, 패킷 교환 네트워크                  |
| 1970년대 | TELENET, TYMNET | 상용 패킷 교환 네트워크와 부가가치 통신망              |

---

### 1.4 주요 시스템 정리

| 시스템     | 설명                          |
|---------|-----------------------------|
| SAGE    | 컴퓨터와 통신을 결합한 최초의 컴퓨터 통신 시스템 |
| SABRE   | 항공기 좌석 예약 업무 처리 시스템         |
| ARPANET | 인터넷의 전신, TCP/IP 개발과 관련      |
| ALOHA   | 실험적 무선 패킷 교환 네트워크           |
| TELENET | 최초의 상용 패킷 교환 네트워크           |
| TYMNET  | 부가가치 통신망                    |

---

### 1.5 정보통신망의 목적

정보통신망의 대표적인 목적은 다음 세 가지이다.

| 목적       | 설명                                  |
|----------|-------------------------------------|
| 자원의 공유   | 여러 지역의 장치, 데이터, 프로그램 등을 공동으로 사용     |
| 신뢰도 향상   | 일부 장치에 장애가 발생해도 다른 장치를 통해 서비스 유지 가능 |
| 처리기능의 분산 | 여러 컴퓨터가 작업을 나누어 처리 가능               |

시험에서는 정보통신망의 목적을 묻는 문제에서 “정보를 집중시켜 관리한다”는 식의 표현이 나오면 부적절한 설명으로 볼 수 있다.

## 2. 정보통신망 구성

### 2.1 데이터통신 시스템의 구성요소

데이터통신 시스템은 데이터를 보내고 받기 위한 장치와 통신선로, 제어장치, 컴퓨터 등으로 구성된다.

| 구성요소   | 설명                      |
|--------|-------------------------|
| 단말장치   | 사용자가 데이터를 입력하거나 출력하는 장치 |
| 통신선로   | 데이터가 전달되는 물리적 경로        |
| 신호변환장치 | 신호를 통신선로에 맞게 변환         |
| 통신제어장치 | 데이터 전송을 제어              |
| 컴퓨터    | 데이터를 처리하는 장치            |

데이터통신 시스템은 크게 데이터 전송 시스템과 데이터 처리 시스템으로 나눌 수 있다.

    단말장치 → 신호변환장치 → 통신선로 → 신호변환장치 → 통신제어장치 → 컴퓨터

---

### 2.2 통신선로의 구성

통신선로는 단말기와 컴퓨터 또는 장치들을 연결하는 방식에 따라 구분된다.

| 구분      | 의미                            |
|---------|-------------------------------|
| 점대점 선로  | 두 단말기를 1대 1로 연결하는 통신선로        |
| 멀티드롭 선로 | 하나의 회선에 2개 이상의 단말기를 연결하는 통신선로 |

---

### 2.3 점대점 선로

점대점(point-to-point) 선로는 두 장치를 직접 1대 1로 연결하는 방식이다.

| 특징    | 설명                     |
|-------|------------------------|
| 연결 형태 | 두 단말기를 직접 연결           |
| 장점    | 구조가 단순하고 전용 회선처럼 사용 가능 |
| 단점    | 연결할 장치가 많아지면 회선 수가 증가  |
| 활용    | 단말기와 컴퓨터를 직접 연결하는 경우   |

---

### 2.4 멀티드롭 선로

멀티드롭(multi-drop) 선로는 하나의 회선에 여러 단말기를 연결하는 방식이다.

| 특징    | 설명                       |
|-------|--------------------------|
| 연결 형태 | 하나의 회선에 여러 단말기를 연결       |
| 장점    | 통신 회선을 공유하므로 회선 비용 절감 가능 |
| 단점    | 점대점 선로보다 신뢰도가 낮을 수 있음    |
| 필요 요소 | 회선을 공유하기 위한 매체제어 프로토콜 필요 |

시험에서는 “멀티드롭 선로가 점대점 선로에 비해 신뢰도가 높다”는 설명은 부적절한 설명으로 볼 수 있다.

---

### 2.5 네트워크 연결장치

네트워크 연결장치는 네트워크 세그먼트를 서로 연결하거나 네트워크를 인터넷에 연결하기 위한 통신 장치이다.

| 장치            | 원어                          | 역할                          |
|---------------|-----------------------------|-----------------------------|
| 모뎀            | Modem                       | 디지털 신호와 아날로그 신호를 변환         |
| 네트워크 인터페이스 카드 | NIC, Network Interface Card | 컴퓨터를 네트워크에 연결               |
| 허브            | Hub                         | 하나의 노드에서 수신한 신호를 다른 노드로 전송  |
| 리피터           | Repeater                    | 신호를 재생하거나 증폭하여 장거리 전송 지원    |
| 브리지           | Bridge                      | 여러 LAN을 연결                  |
| 라우터           | Router                      | IP 네트워크 간 연결 및 인터넷 연결       |
| 게이트웨이         | Gateway                     | 서로 다른 통신 프로토콜을 사용하는 네트워크 연결 |

---

### 2.6 허브

허브는 하나의 노드에서 수신한 신호를 정확히 재생하여 다른 노드로 전송하는 장치이다.

| 종류      | 설명                |
|---------|-------------------|
| 더미 허브   | 단순히 신호를 모든 포트로 전달 |
| 스위칭 허브  | 목적지에 맞게 신호를 전달    |
| 스태커블 허브 | 여러 허브를 쌓아 확장 가능   |

---

### 2.7 리피터

리피터는 장거리 전송을 위해 신호를 새로 재생하거나 출력 전압을 높여 주는 장치이다.

- 약해진 신호를 다시 강하게 만들어 전송
- 전송 거리를 늘리는 데 사용
- 데이터의 내용보다는 신호 자체를 재생하는 역할

---

### 2.8 브리지, 라우터, 게이트웨이

| 장치    | 핵심 역할                             |
|-------|-----------------------------------|
| 브리지   | 복수의 LAN을 연결하는 장치                  |
| 라우터   | IP 네트워크 간 연결 또는 IP 네트워크와 인터넷 간 연결 |
| 게이트웨이 | 서로 다른 통신 프로토콜을 사용하는 네트워크 연결       |

시험에서는 “서로 다른 통신 프로토콜을 사용하는 두 개의 이질적 네트워크를 연결하는 장치”라고 나오면 게이트웨이를 떠올리면 된다.

## 3. 정보통신 방식

## 3.1 데이터 전송방식

데이터 전송방식은 전송방향과 전송모드에 따라 구분할 수 있다.

---

### 3.1.1 전송방향에 따른 전송방식

| 방식     | 영어          | 설명                       | 예             |
|--------|-------------|--------------------------|---------------|
| 단방향 전송 | simplex     | 한 방향으로만 데이터 전송           | TV 방송, 라디오 방송 |
| 반이중 전송 | half duplex | 양방향 전송이 가능하지만 동시에 송수신 불가 | 무전기           |
| 전이중 전송 | full duplex | 양방향으로 동시에 송수신 가능         | 전화            |

---

### 3.1.2 단방향 전송

단방향 전송은 정보가 한 방향으로만 전달되는 방식이다.

- 송신자와 수신자의 역할이 고정됨
- 수신자는 데이터를 받을 수만 있음
- 예: 라디오, TV 방송

---

### 3.1.3 반이중 전송

반이중 전송은 양방향 통신이 가능하지만, 한 순간에는 한쪽 방향으로만 전송되는 방식이다.

- 송신과 수신 방향이 바뀔 수 있음
- 동시에 양방향 전송은 불가능
- 예: 무전기

---

### 3.1.4 전이중 전송

전이중 전송은 동시에 양방향으로 데이터를 송수신할 수 있는 방식이다.

- 송신과 수신이 동시에 가능
- 예: 전화 통화

---

### 3.1.5 전송모드에 따른 전송방식

| 방식   | 영어                    | 설명                              |
|------|-----------------------|---------------------------------|
| 직렬전송 | serial transmission   | 하나의 통신선로를 통해 한 번에 한 비트씩 순서대로 전송 |
| 병렬전송 | parallel transmission | 여러 통신선로를 통해 여러 비트를 동시에 전송       |

---

### 3.1.6 직렬전송

직렬전송은 하나의 통신회선을 통해 한 번에 한 비트씩 순서대로 전송하는 방식이다.

- 하나의 선로를 사용
- 먼 거리 전송에 유리
- 전송 속도는 병렬전송보다 느릴 수 있음
- 구조가 단순하고 비용이 적게 듦

---

### 3.1.7 병렬전송

병렬전송은 여러 개의 통신회선을 통해 여러 비트를 동시에 전송하는 방식이다.

- 여러 비트를 동시에 전송
- 가까운 거리에서 빠른 전송 가능
- 선로가 많이 필요함
- 장거리 전송에는 불리할 수 있음

---

## 3.2 데이터 교환방식

데이터 교환방식은 네트워크 자원을 공유하여 데이터를 전달하는 방식이다.

대표적인 방식은 회선교환과 축적교환이다.

| 구분   | 종류                |
|------|-------------------|
| 회선교환 | 회선을 설정한 뒤 통신      |
| 축적교환 | 메시지교환, 패킷교환       |
| 패킷교환 | 가상회선 방식, 데이터그램 방식 |

---

### 3.2.1 회선교환 방식

회선교환(circuit switching) 방식은 통신 전에 송신자와 수신자 사이의 전용 통신 경로를 설정한 후 데이터를 전송하는 방식이다.

| 특징    | 설명                   |
|-------|----------------------|
| 전송 방식 | 연결지향형 전송             |
| 대표 예  | 전화망                  |
| 장점    | 연결 후 안정적인 전송 가능      |
| 단점    | 통신하지 않는 시간에도 회선이 점유됨 |

---

### 3.2.2 축적교환 방식

축적교환(store-and-forward switching) 방식은 데이터를 중간 노드에 저장한 뒤 다음 노드로 전달하는 방식이다.

축적교환 방식은 다음과 같이 나뉜다.

| 방식    | 설명                      |
|-------|-------------------------|
| 메시지교환 | 메시지 전체를 하나의 단위로 저장 후 전달 |
| 패킷교환  | 메시지를 작은 패킷으로 나누어 전송     |

---

### 3.2.3 메시지교환 방식

메시지교환(message switching) 방식은 송신할 데이터를 메시지 단위로 전달하는 방식이다.

- 메시지 전체를 중간 노드에 저장
- 다음 경로로 전달
- 회선교환처럼 전용 회선을 미리 설정하지 않음
- 메시지 크기가 크면 지연이 발생할 수 있음

---

### 3.2.4 패킷교환 방식

패킷교환(packet switching) 방식은 메시지를 패킷이라는 작은 단위로 나누어 전송하는 방식이다.

- 메시지를 여러 패킷으로 분할
- 각 패킷에는 목적지 주소 등 제어 정보가 포함됨
- 네트워크에서 적절한 경로를 찾아 목적지까지 전달
- 인터넷에서 사용하는 핵심 방식

---

### 3.2.5 가상회선 방식

가상회선(virtual circuit) 방식은 패킷교환 방식 중 연결지향형 전송 방식이다.

| 특징    | 설명                         |
|-------|----------------------------|
| 전송 방식 | 연결지향형                      |
| 경로    | 통신 전에 논리적 경로 설정            |
| 패킷 순서 | 대체로 같은 경로를 따라가므로 순서 유지에 유리 |
| 핵심    | 회선교환처럼 보이지만 실제로는 패킷교환 방식   |

---

### 3.2.6 데이터그램 방식

데이터그램(datagram) 방식은 패킷교환 방식 중 비연결형 전송 방식이다.

| 특징    | 설명                    |
|-------|-----------------------|
| 전송 방식 | 비연결형                  |
| 경로    | 패킷마다 독립적으로 경로 선택      |
| 패킷 순서 | 도착 순서가 송신 순서와 다를 수 있음 |
| 핵심    | 각 패킷이 독립적으로 전달됨       |

시험에서는 “데이터그램 방식은 송신하는 패킷의 순서와 수신하는 패킷의 순서가 항상 같다”는 설명은 부적절한 설명으로 볼 수 있다.

---

## 4. 정보통신망 유형

정보통신망은 위상, 규모, 활용 목적에 따라 분류할 수 있다.

---

## 4.1 위상에 따른 유형

위상은 네트워크 장치들이 연결된 형태를 의미한다.

| 유형  | 영어   | 설명                        |
|-----|------|---------------------------|
| 성형  | star | 중앙 컴퓨터나 허브를 중심으로 각 노드가 연결 |
| 환형  | ring | 노드들이 고리 모양으로 연결           |
| 버스형 | bus  | 하나의 공통 통신선로에 여러 노드가 연결    |
| 그물형 | mesh | 여러 노드가 서로 복잡하게 연결         |
| 트리형 | tree | 계층 구조로 연결                 |

---

### 4.1.1 성형 네트워크

성형(star) 네트워크는 중앙컴퓨터가 네트워크 중앙에 위치하고, 다른 노드들이 중앙컴퓨터에 점대점 링크로 연결된 형태이다.

- 중앙 장치 중심 구조
- 관리가 쉬움
- 중앙 장치에 장애가 발생하면 전체 네트워크에 영향
- 시험에서 “중앙컴퓨터가 네트워크 중앙에 위치”하면 성형 네트워크

---

### 4.1.2 환형 네트워크

환형(ring) 네트워크는 각 노드가 양쪽의 이웃 노드와 연결되어 전체가 고리 형태를 이루는 네트워크이다.

- 데이터가 일정 방향으로 순환 가능
- 한 노드나 링크의 장애가 전체 통신에 영향을 줄 수 있음

---

### 4.1.3 버스형 네트워크

버스형(bus) 네트워크는 하나의 공통 통신선로에 여러 노드가 연결된 형태이다.

- 구조가 단순함
- 설치 비용이 비교적 적음
- 공통 선로에 문제가 발생하면 전체 통신에 영향

---

### 4.1.4 그물형 네트워크

그물형(mesh) 네트워크는 여러 노드가 서로 다양한 경로로 연결된 형태이다.

- 장애에 강함
- 우회 경로를 찾을 수 있음
- 연결선이 많아져 비용과 관리 복잡성이 증가

---

### 4.1.5 트리형 네트워크

트리형(tree) 네트워크는 계층 구조로 노드가 연결된 형태이다.

- 상위 노드와 하위 노드가 계층적으로 연결
- 조직 구조나 네트워크 확장에 적합

---

## 4.2 규모에 따른 유형

정보통신망은 네트워크가 연결되는 규모에 따라 구분된다.

| 유형  | 원어                        | 설명         |
|-----|---------------------------|------------|
| PAN | Personal Area Network     | 개인 영역 통신망  |
| LAN | Local Area Network        | 근거리 통신망    |
| CAN | Campus Area Network       | 캠퍼스 영역 통신망 |
| MAN | Metropolitan Area Network | 도시권 통신망    |
| WAN | Wide Area Network         | 광역 통신망     |

---

### 4.2.1 PAN

PAN은 개인 주변의 장치들을 연결하는 소규모 네트워크이다.

- 개인 영역 통신망
- 예: 스마트폰, 웨어러블 기기, 블루투스 장치 연결

---

### 4.2.2 LAN

LAN은 학교, 사무실, 건물 등 비교적 가까운 지역의 컴퓨터를 연결하는 근거리 통신망이다.

- 근거리 통신망
- 비교적 좁은 범위
- 가정, 사무실, 학교 등에서 사용

---

### 4.2.3 CAN

CAN은 캠퍼스나 기관 내부처럼 여러 건물이나 구역을 연결하는 네트워크이다.

- 캠퍼스 영역 통신망
- 대학, 회사 단지 등에서 사용

---

### 4.2.4 MAN

MAN은 도시 규모의 네트워크이다.

- 도시권 통신망
- LAN보다 넓고 WAN보다 좁은 범위

---

### 4.2.5 WAN

WAN은 넓은 지역을 연결하는 광역 통신망이다.

- 국가, 대륙, 전 세계 규모의 네트워크
- 보기 중 VAN, ISDN, WAN, LAN이 있으면 가장 큰 규모는 WAN

---

## 4.3 활용 목적에 따른 유형

| 유형     | 원어                                  | 설명          |
|--------|-------------------------------------|-------------|
| VAN    | Value Added Network                 | 부가가치 통신망    |
| ISDN   | Integrated Services Digital Network | 종합정보 통신망    |
| B-ISDN | Broadband ISDN                      | 광대역 종합정보통신망 |

---

### 4.3.1 VAN

VAN은 통신망사업자에게 전용선을 임차하여 구성하는 부가가치 통신망이다.

- 홈뱅킹, 홈쇼핑 등의 서비스 제공
- 기존 통신망에 부가가치를 더한 서비스 제공

---

### 4.3.2 ISDN

ISDN은 Integrated Services Digital Network의 약자이다.

- 여러 서비스를 통합하여 제공
- 전화망과 컴퓨터망의 통합
- 하나의 망에서 음성 통신과 영상 통신 가능

---

### 4.3.3 B-ISDN

B-ISDN은 Broadband ISDN의 약자이다.

- 광대역 서비스가 가능한 ISDN
- 광케이블을 이용한 통신
- 수백 Mbps급 통신 가능

---

## 5. 통신 프로토콜

## 5.1 통신 프로토콜의 필요성

통신 프로토콜은 통신을 원하는 두 개체 간에 무엇을, 어떻게, 언제 통신하도록 할 것인지를 서로 약속한 규약이다.

즉, 통신이 원하는 대로 구현되기 위해 통신 주체들 간에 미리 정해 놓은 규칙이다.

---

### 5.2 통신 프로토콜의 핵심

| 구분  | 의미                  |
|-----|---------------------|
| 무엇을 | 어떤 데이터를 주고받을 것인가    |
| 어떻게 | 어떤 형식과 절차로 주고받을 것인가 |
| 언제  | 어떤 시점과 순서로 통신할 것인가  |

---

### 5.3 통신 프로토콜의 종류

| 프로토콜 | 원어                            | 주요 역할            |
|------|-------------------------------|------------------|
| TCP  | Transmission Control Protocol | 신뢰성 있는 데이터 전송 제어 |
| IP   | Internet Protocol             | 인터넷 주소 지정과 패킷 전달 |
| UDP  | User Datagram Protocol        | 빠른 비연결형 데이터 전송   |
| SMTP | Simple Mail Transfer Protocol | 전자우편 송신          |
| POP3 | Post Office Protocol 3        | 전자우편 수신          |
| HTTP | Hyper-Text Transfer Protocol  | 웹 문서 전송          |

---

## 정리하기

1. ARPANET은 1969년 미국 국방성에서 개발한 분산형 컴퓨터 통신망으로, TCP/IP 프로토콜을 적용하여 패킷교환 네트워크로 완성되었으며 인터넷의 기원이라고 불린다.

2. 정보통신망의 대표적인 활용 목적은 자원의 공유, 신뢰도 향상, 처리기능의 분산이다.

3. 데이터통신 시스템의 4대 구성요소는 단말장치, 데이터 전송회선, 통신제어장치, 컴퓨터이다.

4. 점대점 선로는 두 단말기를 1대 1로 연결하는 통신선로이고, 멀티드롭 선로는 한 회선에 2개 이상의 단말기를 연결하는 통신선로이다.

5. 네트워크 연결장치는 네트워크 세그먼트를 서로 연결하거나 인터넷을 만들기 위해 네트워크를 연결하는 데 사용된다. 대표적으로 모뎀, NIC, 허브, 리피터, 브리지, 라우터, 게이트웨이가 있다.

6. 단방향 전송은 한 방향으로만 정보를 전달하는 방식이고, 반이중 전송은 정보 전달 방향이 교대로 바뀔 수 있는 방식이며, 전이중 전송은 동시에 데이터를 양방향으로 송수신할 수 있는 방식이다.

7. 통신방식은 전송모드에 따라 직렬전송과 병렬전송으로 구분할 수 있다.

8. 데이터 교환방식에는 회선교환과 축적교환이 있다. 축적교환에는 메시지교환과 패킷교환이 있으며, 패킷교환은 가상회선 방식과 데이터그램 방식으로 구분된다.

9. 정보통신망은 위상에 따라 성형, 환형, 버스형, 그물형, 트리형 네트워크 등으로 구분된다.

10. 정보통신망은 규모에 따라 PAN, LAN, CAN, MAN, WAN 등으로 구분된다.

11. 정보통신망은 활용 목적에 따라 VAN, ISDN, B-ISDN 등으로 구분된다.

12. 통신 프로토콜은 통신을 원하는 두 개체 간에 무엇을, 어떻게, 언제 통신할 것인지를 서로 약속한 규약이다. 대표적인 통신 프로토콜에는 TCP, IP, UDP, SMTP, POP3, HTTP 등이 있다.

### Q6. 하나의 통신회선을 통해 한 번에 한 비트씩 순서대로 전송하는 데이터 전송 방식은?

1. 직렬 전송 방식
2. 병렬 전송 방식
3. 반이중 전송 방식
4. 전이중 전송 방식

정답: 1

해설:
직렬전송은 하나의 통신회선을 통해 데이터를 한 비트씩 순서대로 전송하는 방식이다.

---

### Q7. 다음 중 데이터 교환 방식에 관한 설명으로 부적절한 것은?

1. 기존의 음성전화 서비스는 회선 교환 방식을 사용한다.
2. 패킷 교환 방식은 송신할 데이터를 패킷 단위로 나누어 전송한다.
3. 패킷 교환 방식은 다시 가상회선 방식과 데이터그램 방식으로 나뉜다.
4. 데이터그램 방식은 송신하는 패킷의 순서와 수신하는 패킷의 순서가 항상 같다.

정답: 4

해설:
데이터그램 방식은 비연결형 전송 방식이므로 패킷마다 서로 다른 경로로 전송될 수 있다. 따라서 송신한 패킷 순서와 수신한 패킷 순서가 항상 같다고 할 수 없다.

### Q8. 중앙컴퓨터가 네트워크 중앙에 위치하고 다른 노드들이 이 중앙컴퓨터에 점대점 링크에 의해 연결된 형태를 갖는 정보통신망 형태는?

1. 성형 네트워크
2. 버스형 네트워크
3. 환형 네트워크
4. 계층형 네트워크

정답: 1

해설:
성형 네트워크는 중앙의 컴퓨터나 허브를 중심으로 각 노드가 점대점 링크로 연결되는 형태이다.

---

### Q9. 다음 중 가장 큰 규모의 컴퓨터 통신망은?

1. VAN
2. ISDN
3. WAN
4. LAN

정답: 3

해설:
WAN은 Wide Area Network로 광역 통신망을 의미한다. LAN은 근거리 통신망이고, VAN과 ISDN은 활용 목적에 따른 통신망 유형이다. 보기 중 가장 큰 규모는 WAN이다.

### Q10. 통신을 원하는 두 개체 간에 무엇을, 언제, 어떻게 통신할 것인지를 서로 약속한 규약은?

1. 구문(syntax)
2. 통신 프로토콜
3. 전자 서명
4. 네트워크 연결

정답: 2

해설:
통신 프로토콜은 통신을 원하는 두 개체 사이에서 무엇을, 어떻게, 언제 통신할 것인지를 정해 놓은 규약이다.
