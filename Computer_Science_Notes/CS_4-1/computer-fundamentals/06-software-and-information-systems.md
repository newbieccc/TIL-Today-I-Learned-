## 2. 소프트웨어의 종류

### 2-1. 소프트웨어의 개념

- 소프트웨어란 컴퓨터의 하드웨어가 다양한 목적의 작업을 수행할 수 있도록 지시하는 일련의 명령어로 구성된 다양한 종류의 프로그램 및 관련 데이터임
- 사용자는 소프트웨어를 통하여 컴퓨터를 사용함

### 2-2. 소프트웨어의 큰 분류

| 구분        | 설명                                 |
|-----------|------------------------------------|
| 시스템 소프트웨어 | 하드웨어를 운영하고 응용 소프트웨어가 동작할 기반을 제공    |
| 응용 소프트웨어  | 사용자가 특정 목적의 작업을 수행할 수 있도록 개발된 프로그램 |

### 핵심 정리

- 소프트웨어는 하드웨어를 실제로 사용할 수 있게 만드는 명령어와 데이터의 집합임
- 시스템 소프트웨어는 컴퓨터 운영의 기반이고, 응용 소프트웨어는 사용자의 목적 달성을 위한 프로그램임

---

## 3. 시스템 소프트웨어

### 3-1. 시스템 소프트웨어의 개념

- 컴퓨터 하드웨어를 운영하고, 응용 소프트웨어가 동작하기 위한 기반 구조를 형성하는 소프트웨어임
- 컴퓨터 사용자가 손쉽게 컴퓨터를 쓸 수 있도록 도와주고,
  동시에 컴퓨터 시스템을 효율적으로 운영해 주는 기능을 갖춘 프로그램의 집단임

### 시스템 소프트웨어의 예

- 운영체제
- 프로그래밍 언어 번역기
    - 컴파일러
    - 인터프리터
- 유틸리티 소프트웨어 등

## 4. 운영체제

### 4-1. 운영체제의 역할

- 운영체제는 컴퓨터가 동작하는 동안 항상 실행됨
- 사용자가 컴퓨터와 대화할 수 있도록 인터페이스를 제공함
- 컴퓨터의 하드웨어 장치를 관리함
- 파일 시스템을 관리하고 보존함
- 프로그램들이 원활하게 실행될 수 있도록 관리하고 지원함

### 4-2. 부팅 과정

- ROM에 들어 있는 BIOS 실행
- POST 수행
    - Power-On Self-Test
- Booting
    - 운영체제를 RAM에 적재

### 4-3. 사용자 인터페이스

| 구분  | 설명                                      |
|-----|-----------------------------------------|
| CLI | Command-Line Interface, 명령어 기반 인터페이스    |
| GUI | Graphical User Interface, 그래픽 사용자 인터페이스 |

### 4-4. 하드웨어 자원 관리

- 주기억장치 관리
- 디스플레이 관리
- 키보드, 마우스 관리
- 오디오 장치 관리
- 프린터 관리
- 네트워크 관리
- 보조기억장치 관리

### 4-5. 파일 시스템 관리

- 폴더 관리
- 파일 생성
- 파일 복사
- 파일 삭제
- 파일 검색
- 백업 및 복구

### 핵심 정리

- 운영체제는 사용자와 하드웨어 사이의 중간 역할을 함
- 운영체제는 하드웨어 자원, 파일 시스템, 프로그램 실행을 관리함
- CLI에서 GUI로 발전하며 사용성이 향상됨

## 5. 대표 운영체제

### 5-1. 마이크로소프트 Windows

- 1995년 개인용 컴퓨터를 위한 GUI 환경 운영체제인 Windows 95 출시
- 이후 Windows 98, Windows XP, Windows Vista, Windows 7 등을 거쳐 발전
- 2015년 Windows 10 출시
- 2021년 Windows 11 공개
- 개인용 데스크톱, 노트북 외에도 서버 및 모바일용 버전 출시

### Windows 10 특징

- UWP 도입
    - Universal Windows Platform
    - PC, 태블릿, 스마트폰 등 다양한 장치에서 거의 동일한 코드로 앱 실행 가능
- Microsoft Edge 웹 브라우저 제공
- 가상 데스크톱 기능
- 작업보기(Task View) 기능
- Windows Hello
    - 다요소 인증

### Windows 11 특징

- Teams를 통한 협업 기능
    - 채팅
    - 화상통화 등
- 여러 작업 창을 편리하게 배치할 수 있도록 사용자 인터페이스 개선
- 위젯을 통해 뉴스, 할 일, 날씨 등 필요한 콘텐츠 제공

- 이후 다양한 서버급 컴퓨터에서도 사용될 수 있도록 이식됨

## 6. 언어 번역기

### 6-1. 언어 번역기의 개념

- 프로그래밍 언어로 작성된 프로그램을 컴퓨터가 실행할 수 있는 기계어 코드로 변환하는 프로그램임
- 원시 프로그램을 목적 프로그램으로 변환함

### 예시

    int countM3(int *arr, int len)
    {
        int count = 0;

        for (int i = 0; i < len; i++) {
            if (arr[i] % 3 == 0)
                count++;
        }

        return count;
    }

### 6-2. 고급언어 번역기의 종류

| 구분    | 설명                                    |
|-------|---------------------------------------|
| 컴파일러  | 원시 프로그램 전체를 기계어 명령으로 구성된 목적 프로그램으로 번역 |
| 인터프리터 | 프로그램의 문장 단위로 명령을 해석하여 실행              |

### 컴파일러

- 원시 프로그램 전체를 한 번에 번역함
- 인터프리터 방식에 비해 프로그램이 빠르게 실행됨

### 인터프리터

- 프로그램의 문장 단위로 명령을 해석하여 실행함
- 프로그램 번역을 기다리지 않고 즉시 실행할 수 있음

### 핵심 정리

- 컴파일러는 전체 번역 후 실행하는 방식임
- 인터프리터는 문장 단위로 해석하며 실행하는 방식임
- 실제로는 두 방식의 장점을 절충한 다양한 변형이 활용됨

## 9. 소프트웨어의 개발

## 9-1. 프로그래밍 언어

### 저급언어

- 기계 중심의 언어
- 컴퓨터가 직접 이해하고 실행할 수 있는 2진 코드 형태의 언어

### 예시

### 예시

    ADD AX, 0010H

---

### 고급언어

#### 제3세대 언어

- 문제 중심의 언어
- 사람에게 친숙한 문장 구조나 수식 표현을 사용함

### 예시

    if a < 0:
        print('음수')

### 제3세대 언어의 분류

- 절차적 언어:
    - FORTRAN
    - COBOL
    - PASCAL
    - C 등
- 객체지향 언어:
    - C++
    - Java
    - C#
    - Objective-C
    - Swift 등

---

### 제4세대 언어

- 기존 순차적 고급언어에 비해 더 높은 수준의 기능을 제공함
- 프로그래밍 노력이나 시간을 줄이기 위한 언어임

### 활용 분야

- 데이터베이스 검색
- 리포트 생성
- 데이터 조작 및 분석 등

---

### 제5세대 언어

- 1980년대 인공지능 구현이나 전문가 시스템 제작을 목적으로 개발됨
- 주어진 문제에 대해 제공된 규칙, 제약사항 등을 이용하여 문제를 해결하는 방식의 언어임
- 함수형 언어, 논리형 언어 등의 선언형 언어가 포함됨

### 예

- LISP
- Prolog
- OPS5 등

### 특징

- 실용적 응용에서 기대에 부응할 만한 결과를 만들어 내는 데는 실패함

---

### 9-2. 프로그래밍 언어의 추상화 단계

- 제1세대 언어:
    - 기계어
- 제2세대 언어:
    - 어셈블리어
- 제3세대 언어:
    - 고급언어
- 제4세대 언어:
    - 더 높은 수준의 기능 제공
- 제5세대 언어:
    - 인공지능, 전문가 시스템 지향

### 핵심 정리

- 1세대와 2세대 언어는 기계 중심 언어임
- 3세대 이후는 사람의 문제 해결 방식에 가까운 언어임
- 세대가 올라갈수록 추상화 수준이 높아지고 프로그래밍 부담은 줄어듦

## 10. 알고리즘

### 10-1. 프로그램이란?

- 원하는 결과를 얻을 수 있도록 컴퓨터에게 일련의 일을 시키는 명령어의 집합체임
- 필요한 업무를 처리할 수 있도록 알고리즘을 설계하고,
  프로그래밍 언어로 구현함

### 10-2. 알고리즘이란?

- 주어진 문제를 해결하기 위하여 유한한 개수의 잘 정의된 명령으로 표현된 처리 절차임

### 10-3. 알고리즘의 표현

- 프로그램을 작성하기 전 알고리즘을 먼저 구성해야 함
- 알고리즘을 명확하게 나타내기 위한 표현 방법이 필요함

### 표현 방법

- 순서도(flow chart)
- 의사코드(pseudocode)

---

### 10-4. 윤년 판단 알고리즘

### 의사코드 예시

    Procedure 윤년검사
        input year;

        IF year mod 4 = 0 THEN
            IF year mod 100 <> 0 THEN
                print '윤년';
            ELSE IF year mod 400 = 0 THEN
                print '윤년';
            ELSE
                print '평년';
            END-IF
        ELSE
            print '평년';
        END-IF;
    END

### 윤년 판단 조건

- 4로 나누어 떨어지지 않으면 평년
- 4로 나누어 떨어지고 100으로 나누어 떨어지지 않으면 윤년
- 100으로 나누어 떨어지지만 400으로 나누어 떨어지면 윤년
- 400으로 나누어 떨어지지 않으면 평년

### 핵심 정리

- 알고리즘은 문제 해결 절차를 명확하게 표현한 것임
- 순서도와 의사코드는 알고리즘을 표현하는 대표적인 방법임

## 11. 소프트웨어 공학과 개발 방법론

### 11-1. 소프트웨어 공학

- 소프트웨어의 개발, 운영, 유지보수에 체계적이고 숙달되고 정량화된 접근 방법을 적용하는 것

### 11-2. 소프트웨어 개발 방법론

### 개발 방법론 예

- 폭포수 모델

2. 요구분석
3. 설계
4. 구현
5. 시험
6. 유지보수

## 13. 소프트웨어 사용권의 유형

### 13-1. 사유 소프트웨어

- 저작권 소유자가 허가한 법적 권한의 범위에서 사용할 수 있는 컴퓨터 소프트웨어임
- 소프트웨어를 분석, 수정, 공유하는 행위를 제한함
- 상용 소프트웨어의 경우 이에 상응하는 비용을 소프트웨어 소유자에게 지불해야 함

### 셰어웨어

- 판매를 위해 제작된 사유 소프트웨어임
- 사용자에게 일정 기간 또는 정품에 비해 제한된 기능만 사용할 수 있도록 무료 제공됨

### 프리웨어

- 사용권에 대한 비용 없이 사용할 수 있도록 허용된 소프트웨어임
- 단, 사용권은 개인이나 기관·기업에 다르게 적용될 수 있음
- 개인에게 무료로 제공되는 프리웨어를 직장의 컴퓨터에 설치하여 사용할 경우 저작권 침해가 될 수 있음

### 핵심 정리

- 사유 소프트웨어는 사용, 수정, 공유가 제한됨
- 셰어웨어는 일정 기간 또는 제한된 기능을 무료로 제공함
- 프리웨어는 무료로 사용할 수 있지만 사용 조건을 반드시 확인해야 함

## 14. 자유 소프트웨어 운동

### 14-1. 자유 소프트웨어 운동의 개념

- 소프트웨어의 자유로운 활용을 장려해야 한다는 움직임
- 1983년 MIT의 리처드 스톨먼이 주도한 GNU 프로젝트에서 출발함
- 1985년 자유 소프트웨어 재단을 설립함

### 14-2. GNU 시스템과 리눅스

- 소프트웨어 공유를 위해 운영체제 및 관련 유틸리티 소프트웨어가 필요했음
- GNU 시스템이라는 UNIX 호환 소프트웨어 시스템을 개발함
- 리눅스는 GNU 시스템의 핵심이 되는 운영체제 커널임
- GNU GPL을 따름

### 14-3. GNU GPL의 네 가지 자유

1. 프로그램을 어떠한 목적으로든 사용할 수 있는 자유
2. 프로그램의 소스 코드를 분석하고 수정할 수 있는 자유
3. 소프트웨어를 타인에게 재배포할 수 있는 자유
4. 사용자가 수정한 소프트웨어를 배포할 수 있는 자유

### 14-4. 자유 소프트웨어 사용권

- GNU GPL
- BSD 사용권
- Mozilla Public License
- MIT 사용권
- Apache 사용권

### 핵심 정리

- 자유 소프트웨어는 무료라는 의미만이 아니라,
  사용, 분석, 수정, 재배포의 자유를 강조함
- GNU GPL은 자유 소프트웨어 운동을 대표하는 사용권임

## 15. 기업활동과 정보 시스템의 발전

### 15-1. 경영정보 시스템(MIS)

- 기업 경영에 관한 정보를 제공해 주는 컴퓨터 시스템임
- 단순 전산처리 중심에서
  기업 경영에 도움을 주는 정보 제공 중심으로 발전함

### 경영정보 시스템의 범주

| 구분  | 의미         |
|-----|------------|
| TPS | 거래처리 시스템   |
| DSS | 의사결정지원 시스템 |
| ERP | 전사적 자원관리   |
| CRM | 고객관계 관리    |
| SCM | 공급망 관리     |

### 15-2. ERP

- 생산, 판매, 자재, 인사, 회계 등 기업 전반적인 업무 프로세스를 하나의 체계로 통합한 시스템임
- 개별 업무가 별개의 시스템으로 운영되면 상호 연계 정보가 단절될 수 있음
- ERP는 전 부문에 걸쳐 있는 경영자원을 하나의 통합적 시스템으로 구축함

### 핵심 정리

- MIS는 기업 경영을 지원하는 정보 시스템임
- ERP는 기업 전반의 업무 프로세스를 하나의 체계로 통합하는 시스템임

---

## 16. 정리하기

1. 소프트웨어란 컴퓨터의 하드웨어가 다양한 목적의 작업을 수행할 수 있도록 지시하는 일련의 명령어로 구성된 다양한 종류의 프로그램 및 관련 데이터를 의미함

2. 시스템 소프트웨어는 하드웨어를 운영하고 응용 소프트웨어가 동작하기 위한 기반 구조를 형성함
    - 운영체제, 프로그래밍 언어 번역기, 유틸리티 프로그램 등이 포함됨

3. 운영체제는 사용자 인터페이스 제공, 하드웨어와 파일 시스템 관리, 프로그램 실행 관리와 지원을 수행함

4. 응용 소프트웨어는 컴퓨터 사용자들이 특정 분야의 응용을 목적으로 사용할 수 있도록 개발되는 프로그램임

5. 프로그래밍 언어는 기계어나 어셈블리어 같은 기계 중심의 저급언어와,
   사람의 언어와 유사한 문장을 사용하는 고급언어가 있음

6. 제4세대 언어는 기존 순차적 고급언어보다 높은 수준의 기능을 제공하여 프로그래밍 노력이나 시간을 줄이기 위한 언어임
    - 제5세대 언어는 인공지능 분야에 사용되는 언어임

7. 알고리즘은 주어진 문제를 해결하기 위하여 유한한 개수의 잘 정의된 명령으로 표현된 처리 절차임
    - 순서도나 의사코드로 표현할 수 있음

8. 사유 소프트웨어는 저작권 소유자의 예외적 법적 권한 하에 허가된 소프트웨어로,
   소프트웨어를 분석·수정하거나 공유하는 행위를 제한함

9. GNU 프로젝트는 소프트웨어의 사용, 공유, 분석, 수정 등의 자유를 누릴 수 있도록 하는 자유 소프트웨어 운동임
    - 리눅스는 GNU 시스템의 핵심이 되는 운영체제 커널임

10. 경영정보 시스템은 기업 경영에 관한 정보를 제공해 주는 컴퓨터 시스템임

## 19. 시험 직전 체크포인트

- 소프트웨어는 프로그램과 관련 데이터의 집합이다.
- 시스템 소프트웨어는 하드웨어를 운영하고 응용 소프트웨어의 기반이 된다.
- 운영체제는 사용자 인터페이스, 하드웨어 자원, 파일 시스템, 프로그램 실행을 관리한다.
- BIOS 실행 → POST → Booting 순서로 운영체제가 적재된다.
- CLI는 명령어 기반, GUI는 그래픽 기반 사용자 인터페이스이다.
- Windows, UNIX, Linux, macOS는 대표적인 운영체제이다.
- Linux는 자유 소프트웨어 및 오픈 소스 운영체제이다.
- 언어 번역기는 원시 프로그램을 목적 프로그램으로 변환한다.
- 컴파일러는 전체 번역 후 실행하고, 인터프리터는 문장 단위로 해석한다.
- 유틸리티 소프트웨어는 컴퓨터 운영과 관리를 돕는다.
- 응용 소프트웨어는 사용자의 특정 목적을 위해 개발된다.
- 기계어는 1세대, 어셈블리어는 2세대, 고급언어는 3세대 언어이다.
- 제4세대 언어는 데이터베이스 검색, 리포트 생성 등에 활용된다.
- 제5세대 언어는 인공지능과 전문가 시스템 구현을 목적으로 개발되었다.
- 알고리즘은 유한한 개수의 잘 정의된 명령으로 표현된 처리 절차이다.
- 알고리즘은 순서도와 의사코드로 표현할 수 있다.
- 소프트웨어 공학은 신뢰성 있고 효율적인 소프트웨어를 경제적으로 생산하기 위한 공학적 접근이다.
- 폭포수 모델은 계획 → 요구분석 → 설계 → 구현 → 시험 → 유지보수 단계로 진행된다.
- EULA는 최종사용자 사용권 동의서이다.
- 사유 소프트웨어는 분석, 수정, 공유를 제한한다.
- 셰어웨어는 일정 기간 또는 제한 기능을 무료로 제공한다.
- 프리웨어는 무료 사용이 가능하지만 사용 조건을 확인해야 한다.
- GNU 프로젝트는 자유 소프트웨어 운동과 관련된다.
- GNU GPL은 사용, 분석, 수정, 재배포의 자유를 강조한다.
- MIS는 기업 경영 정보를 제공하는 컴퓨터 시스템이다.
- ERP는 기업 전반의 업무 프로세스를 통합하는 전사적 자원관리 시스템이다.
