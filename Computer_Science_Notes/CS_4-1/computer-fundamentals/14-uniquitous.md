## 1. 유비쿼터스의 개념과 발전과정

### 1.1 유비쿼터스의 의미

유비쿼터스는 라틴어에서 유래한 말로, 물이나 공기처럼 “언제 어디서나 존재한다”는 의미를 가진다.

컴퓨터 분야에서 유비쿼터스는 언제, 어디서나, 누구라도 컴퓨터와 네트워크를 통해 손쉽고 편리하며 안전하게 서비스를 이용할 수 있는 환경을 의미한다.

| 구분    | 내용                                       |
|-------|------------------------------------------|
| 어원    | 라틴어, 언제 어디서나 존재한다                        |
| 핵심 의미 | 언제, 어디서나, 누구나 컴퓨터와 네트워크 이용               |
| 제시자   | 마크 와이저                                   |
| 시기    | 1988년                                    |
| 기관    | 제록스 팰러앨토연구소, PARC                        |
| 관련 용어 | Ubiquitous computing, Ubiquitous network |

---

### 1.2 유비쿼터스 환경의 모습

유비쿼터스 환경에서는 가전제품, 가구, 의류, 식품, 도로, 다리, 터널, 빌딩, 벽 등 모든 물리공간에 작은 마이크로프로세서나 RFID가 들어간다.

이를 통해 모든 사물과 대상이 지능화되고 연결되어 서로 정보를 주고받는다.

예:

- 냉장고가 식품 정보를 인식
- 도로가 교통상황을 감지
- 건물이 에너지 사용량을 자동 조절
- 사람이 의식하지 않아도 주변 기기가 정보를 처리

---

### 1.3 유비쿼터스 발전과정

컴퓨터 이용 환경은 다음과 같이 발전하였다.

| 단계   | 시대         | 특징                         | 관계                         |
|------|------------|----------------------------|----------------------------|
| 제1단계 | 메인프레임 시대   | 1960년대 비즈니스 분야에서 컴퓨터 활용    | many person one computer   |
| 제2단계 | 개인용 컴퓨터 시대 | 1980년대 클라이언트와 서버 등장        | fewer persons one computer |
| 제3단계 | 인터넷 시대     | 1990년대 중반 개인용 컴퓨터가 인터넷에 연결 | one person one computer    |
| 제4단계 | 유비쿼터스 시대   | 다양한 컴퓨터가 도처에 편재            | one person many computer   |

---

### 1.4 각 단계의 핵심

#### 제1단계: 메인프레임 시대

- 1960년대 비즈니스 분야에서 본격적으로 컴퓨터 활용
- 많은 사람이 하나의 컴퓨터를 함께 사용
- 주역은 컴퓨터

  many person one computer

---

#### 제2단계: 개인용 컴퓨터 시대

- 1980년대 클라이언트와 서버 등장
- 개인용 컴퓨터가 전용선으로 네트워크화되어 사용
- 한 컴퓨터를 적은 수의 사람이 사용

  fewer persons one computer

---

#### 제3단계: 인터넷 시대

- 1990년대 중반 개인용 컴퓨터가 인터넷 네트워크에 연결
- 사람과 컴퓨터는 대등한 관계
- 한 사람이 하나의 컴퓨터를 사용하는 형태

  one person one computer

---

#### 제4단계: 유비쿼터스 시대

- 다양한 컴퓨터가 도처에 존재
- 이용자는 컴퓨터의 존재를 의식하지 않음
- 한 사람이 많은 컴퓨터를 사용하는 형태
- 사람과 컴퓨터의 주종관계가 역전

  one person many computer

유비쿼터스는 정보통신 관점에서 사회 모든 분야에 대한 새로운 패러다임을 창조한다.

---

## 2. 유비쿼터스의 지향점

### 2.1 유비쿼터스 컴퓨팅의 특징

마크 와이저는 유비쿼터스 컴퓨팅의 기준을 다음과 같이 제시하였다.

| 기준    | 설명                       |
|-------|--------------------------|
| 연결성   | 모든 장치는 서로 연결되어 있어야 함     |
| 비가시성  | 사용자 인터페이스는 눈에 보이지 않아야 함  |
| 편재성   | 어디서나 컴퓨터 사용이 가능해야 함      |
| 생활 통합 | 컴퓨터가 생활 속에 자연스럽게 통합되어야 함 |

---

### 2.2 Invisible Interface

유비쿼터스 컴퓨팅에서는 사용자가 컴퓨터를 의식하지 않아도 자연스럽게 서비스를 이용할 수 있어야 한다.

즉, 누구나 쉽게 사용할 수 있는 사용자 중심의 인터페이스가 필요하다.

서비스는 사용자의 상황에 따라 달라져야 한다.

| 사용자 상황 | 예                 |
|--------|-------------------|
| 장소     | 집, 회사, 거리, 차량 안   |
| ID     | 사용자 식별            |
| 장치     | 스마트폰, 태블릿, 센서, 가전 |
| 시간     | 아침, 낮, 밤          |
| 온도     | 실내외 온도            |
| 명암     | 주변 밝기             |
| 날씨     | 비, 눈, 맑음 등        |

---

유비쿼터스의 기본 개념은 다음과 같이 정리된다.

| 안심하고 사용 가능한 네트워크 | 안전한 정보이용 환경               |

## 3. 유비쿼터스 관련기술

## 3.1 미세전자기계시스템

### 3.1.1 MEMS의 의미

MEMS는 Micro Electro Mechanical Systems의 약자이다.

전자, 반도체, 기계, 광 기술을 융합하여 초소형 부품과 시스템을 설계, 제작, 응용하는 기술이다.

| 구분 | 내용                               |
|----|----------------------------------|
| 원어 | Micro Electro Mechanical Systems |
| 약어 | MEMS                             |
| 의미 | 전자·기계·광 기술을 결합한 초소형 시스템          |
| 크기 | 마이크로 단위                          |
| 활용 | 센서, 초소형 부품, 정밀 제어 장치             |

---

### 3.1.2 MEMS의 특징

- 웨이퍼 상에 일괄 제조할 수 있어 소형화 가능
- 하나의 칩에 많은 기능 소자와 신호 처리부를 집적화 가능
- 고성능과 고신뢰성 확보 가능
- 대량 생산이 가능하여 가격을 낮출 수 있음

시험에서는 “전자, 기계, 광 기술을 융합하여 마이크로 단위의 초소형 부품과 시스템을 설계·제작·응용하는 기술”이라고 나오면 MEMS를 고른다.

---

## 3.2 IPv6

### 3.2.1 IPv6의 의미

IPv6는 IPv4의 문제점을 보완한 차세대 인터넷 프로토콜이다.

IPNG, 즉 IP Next Generation이라고도 한다.

| 구분    | 내용                                  |
|-------|-------------------------------------|
| IPv6  | Internet Protocol version 6         |
| IPNG  | IP Next Generation                  |
| 주소 길이 | 128비트                               |
| 목적    | IPv4의 주소 부족, 보안성 취약, 멀티미디어 성능 부족 보완 |
| 관련성   | 유비쿼터스 네트워킹 구현 기반                    |

---

### 3.2.2 IP 주소 비교

| 구분   | 주소 길이 | 특징                         |
|------|------:|----------------------------|
| IPv4 |  32비트 | 약 43억 개 주소                 |
| IPv5 |  64비트 | 강의에서는 비교용으로 제시             |
| IPv6 | 128비트 | 매우 많은 주소 제공, 유비쿼터스 네트워킹 기반 |

유비쿼터스 환경에서는 수많은 사물과 장치가 네트워크에 연결되어야 하므로 IPv6처럼 방대한 주소 공간이 필요하다.

---

## 3.3 RFID

### 3.3.1 RFID의 의미

RFID는 Radio Frequency Identification의 약자로, 일정한 주파수 대역을 이용해 무선으로 각종 데이터를 주고받을 수 있는 시스템이다.

| 구분 | 내용                             |
|----|--------------------------------|
| 원어 | Radio Frequency Identification |
| 의미 | 무선인식 시스템                       |
| 구성 | 초소형 마이크로 칩, 안테나                |
| 역할 | 사물 식별, 데이터 송수신                 |

---

### 3.3.2 RFID의 특징

RFID는 다음과 같은 특징을 가진다.

- 원거리에서도 인식 가능
- 이동 중에도 인식 가능
- 눈, 비, 바람, 먼지, 자석 등 환경의 영향을 적게 받음
- 제조과정에서 유일한 ID를 부여할 수 있어 위조가 어려움

---

### 3.3.3 RFID 구성요소

| 구성요소     | 설명                       |
|----------|--------------------------|
| RFID 태그  | 안테나와 집적회로로 구성, 작은 크기도 가능 |
| RFID 판독기 | 태그 안의 암호화된 데이터를 읽고 해석    |
| 안테나      | 무선 신호 송수신                |
| 호스트 컴퓨터  | 판독된 데이터를 처리              |

예:

- 무인마트
- 물류 관리
- 출입 관리
- 제품 식별

시험에서는 “모든 사물에 아주 작은 마이크로프로세서를 넣어 정보를 주고받게 하는 기술” 또는 “유일한 ID 부여로 위조가 어려운 무선인식 기술”이라고 나오면 RFID를 떠올린다.

---

## 3.4 유비쿼터스 센서 네트워크

### 3.4.1 USN의 의미

USN은 Ubiquitous Sensor Network의 약자이다.

모든 사물에 전자태그나 센서를 부착하여 사물과 환경을 인식하고, 네트워크를 통해 실시간 정보를 전달하고 활용하는 기술이다.

| 구분    | 내용                              |
|-------|---------------------------------|
| 원어    | Ubiquitous Sensor Network       |
| 다른 표현 | Wireless Sensor Network         |
| 핵심    | 사물과 환경 인식, 실시간 정보 전달            |
| 구성    | 전자태그, 리더, 미들웨어, 응용서비스 플랫폼, 유무선망 |

---

### 3.4.2 USN 활용 분야

USN은 다양한 분야에서 활용될 수 있다.

| 분야 | 예                                |
|----|----------------------------------|
| 의료 | u-Healthcare, 환자 상태 감시, 응급 상황 알림 |
| 교통 | 이동 차량 위치 추적, 버스 정보, 교통정보 제공      |
| 물류 | 컨테이너 위치 확인, 창고 재고 관리             |
| 환경 | 대기, 수질, 온도, 습도 감시                |
| 산업 | 설비 상태 감시, 자동 제어                  |

---

## 3.5 임베디드 시스템

### 3.5.1 임베디드 시스템의 의미

임베디드 시스템은 기계 또는 전자 장치에 두뇌 역할을 하는 마이크로프로세서를 장착하여 효과적으로 제어할 수 있도록 하는 시스템이다.

즉, 기기를 동작시키는 소프트웨어를 칩에 담아 기기에 내장한 형태의 장치이다.

| 구분 | 내용                      |
|----|-------------------------|
| 원어 | Embedded System         |
| 핵심 | 마이크로프로세서와 소프트웨어를 기기에 내장 |
| 목적 | 기기 제어                   |
| 예  | 세탁기, 밥솥, 냉장고            |

---

### 3.5.2 임베디드 시스템 예

- 세탁물 종류에 따라 물, 세제, 세탁 강도를 제어하는 세탁기
- 재료에 따라 작동 방식이 달라지는 밥솥
- 냉장실 또는 냉동실의 재료에 따라 자동 반응하는 냉장고

---

## 3.6 단일칩시스템

### 3.6.1 SoC의 의미

SoC는 System on Chip의 약자이다.

하나의 칩에 명령어 처리를 위해 필요한 컴퓨터의 모든 하드웨어와 부수적인 전자부품을 포함하는 기술이다.

| 구분 | 내용                 |
|----|--------------------|
| 원어 | System on Chip     |
| 약어 | SoC                |
| 핵심 | 하나의 칩에 컴퓨터 하드웨어 통합 |
| 효과 | 소형화, 저전력, 고집적화     |

---

|-------|------------------------------|

## 4. 사물인터넷

## 4.1 사물인터넷의 의미

사물인터넷(IoT, Internet of Things)은 인터넷을 기반으로 모든 사물을 연결하여 사람과 사물, 사물과 사물 사이의 정보를 상호 소통하는 지능형 기술과 서비스를 의미한다.

1999년 MIT Auto-ID 센터의 Kevin Ashton이 RFID와 기타 센서를 일상생활 속 사물에 탑재하면 사물인터넷이 구축될 것이라고 언급하였다.

---

### 4.2 사물인터넷의 특징

## 5. 스마트 시티

## 5.1 스마트 시티의 의미

스마트 시티는 첨단 정보통신기술, IoT 등을 이용하여 도시 생활 속에서 발생하는 교통 문제, 환경 문제, 주거 문제, 시설 비효율 등을 해결하는 도시이다.

목표는 시민들이 편리하고 쾌적한 삶을 누릴 수 있도록 하는 효율적이고 똑똑한 도시를 만드는 것이다.

| 구분    | 내용                    |
|-------|-----------------------|
| 원어    | Smart City            |
| 활용 기술 | ICT, IoT              |
| 해결 문제 | 교통, 환경, 주거, 시설 비효율    |
| 목적    | 편리하고 쾌적한 도시생활         |
| 국내 사례 | 세종, 부산 스마트 시티 국가 시범도시 |

| 영역    | 예                          |

| 생활·복지 | 시민 편의 서비스, 건강 관리           |

### 5.3 세계 동향

세계 여러 도시가 스마트 시티 구축을 추진하고 있다.

스마트 조명, 스마트 교통 관리, 스마트 에너지 관리처럼 도시의 기반시설을 데이터와 네트워크로 연결하는 방식이 확대되고 있다.

세종 스마트 시티는 국가 시범도시 중 하나로, 자율주행과 공유 기반 이동 서비스, 스마트 에너지, 스마트 헬스케어, 생활 편의 서비스를 중심으로 한 미래 도시 모델을 지향한다.

핵심 방향은 다음과 같다.

- 자동차가 줄어드는 도시
- 개인 맞춤형 헬스케어
- 데이터 기반 도시 운영
- 친환경 에너지 관리
- 시민 중심의 스마트 서비스

---

### 5.5 부산 스마트 시티

부산 스마트 시티는 에코델타시티를 중심으로 추진되는 국가 시범도시이다.

핵심 방향은 다음과 같다.

- 수변도시와 스마트 기술의 결합
- 물 관리와 친환경 도시 운영
- 로봇, 물류, 헬스케어 등 도시 서비스 적용
- 에너지, 교통, 환경 데이터를 활용한 도시 관리

## 정리하기

1. 유비쿼터스는 라틴어에서 유래한 말로, 물이나 공기처럼 언제 어디서나 존재한다는 의미이다.

2. 유비쿼터스 컴퓨팅은 언제, 어디서나, 누구나 컴퓨터와 네트워크를 손쉽고 편리하며 안전하게 이용할 수 있는 환경을 뜻한다.

3. 유비쿼터스 개념은 1988년 제록스 PARC의 마크 와이저가 처음 제시하였다.

4. 컴퓨터 이용 환경은 메인프레임 시대, 개인용 컴퓨터 시대, 인터넷 시대, 유비쿼터스 시대로 발전하였다.

5. 유비쿼터스 시대는 one person many computer 환경이며, 다양한 컴퓨터가 도처에 존재하지만 사용자는 그 존재를 의식하지 않는다.

6. 마크 와이저는 유비쿼터스 컴퓨팅의 기준으로 연결성, 비가시성, 어디서나 사용 가능성, 생활 통합성을 제시하였다.

7. 유비쿼터스 환경은 5C와 5ANY를 지향한다.

8. 5C는 Computing, Communication, Connectivity, Contents, Calm이다.

9. 5ANY는 Anytime, Anywhere, Any network, Any device, Any service이다.

10. 유비쿼터스 관련기술에는 MEMS, IPv6, RFID, USN, 임베디드 시스템, SoC, Bluetooth, ZigBee 등이 있다.

11. MEMS는 전자, 기계, 광 기술을 융합하여 마이크로 단위의 초소형 부품과 시스템을 설계·제작·응용하는 기술이다.

12. IPv6는 128비트 주소를 사용하는 차세대 인터넷 프로토콜로 유비쿼터스 네트워킹에 중요하다.

13. RFID는 일정한 주파수 대역을 이용하여 무선으로 데이터를 주고받는 무선인식 시스템이다.

14. USN은 모든 사물에 전자태그나 센서를 부착하여 사물과 환경을 인식하고 실시간 정보를 전달·활용하는 네트워크이다.

15. 임베디드 시스템은 기기를 동작시키는 소프트웨어를 칩에 담아 기기에 내장한 시스템이다.

16. 블루투스는 10m 이내의 초단거리 저전력 무선 연결 기술이다.

17. 지그비는 블루투스의 고가격, 고전력 소비의 단점을 보완한 기술이다.

18. 사물인터넷은 인터넷을 기반으로 모든 사물을 연결하여 사람과 사물, 사물과 사물 간 정보를 상호 소통하는 지능형 기술 및 서비스이다.

19. 사물인터넷 응용 분야에는 스마트팜, 자율주행 무인자동차, 헬스케어, 스마트 홈, 스마트 시티 등이 있다.

20. 스마트 시티는 ICT와 IoT를 이용하여 교통, 환경, 주거, 시설 비효율 등을 해결하고 시민에게 편리하고 쾌적한 삶을 제공하는 도시이다.

- Bluetooth = 10m 이내 초단거리 저전력 무선연결
- ZigBee = 블루투스의 고가격·고전력 단점 보완
- IoT = 사물인터넷
- 스마트 시티 = ICT와 IoT로 도시 문제 해결
