## 1. 정보보안의 개요

### 1.1 정보보안의 등장 배경

인터넷이 보편화되면서 많은 활동이 오프라인에서 온라인으로 이동하였다.

- 오프라인 중심 생활 → 온라인 중심 생활
- 정보 활용 증가
- 개인정보, 금융정보, 시스템 정보의 가치 증가
- 정보의 역기능 발생
- 정보보안의 필요성 증가

정보를 많이 활용할수록 정보 유출, 시스템 침입, 악성 프로그램 감염, 개인정보 남용 등의 문제가 발생할 수 있으므로 정보보안이 중요해진다.

---

### 1.2 정보보안과 정보보호

| 구분   | 의미                                             |
|------|------------------------------------------------|
| 정보보안 | 정보를 사용하는 과정에서 발생할 수 있는 부작용에 대처하기 위한 모든 정보보호 활동 |
| 정보보호 | 논리적 장치와 물리적 장치를 이용하여 정보 활용의 부작용을 미연에 방지하는 활동   |

정보보호는 공급자와 수요자 입장에서 다음과 같이 이해할 수 있다.

| 구분  | 내용                   |
|-----|----------------------|
| 공급자 | 정보 자산을 안전하게 보호하고 운영함 |
| 수요자 | 개인정보의 유출이나 남용을 방지함   |

---

### 1.3 정보보안의 3대 목표

정보보안의 주요 목표는 기밀성, 무결성, 가용성이다.

| 목표  | 영어              | 의미                                       |
|-----|-----------------|------------------------------------------|
| 기밀성 | confidentiality | 허가되지 않은 사용자에게 정보가 노출되지 않도록 하는 것          |
| 무결성 | integrity       | 허가되지 않은 사용자에 의해 정보가 변경되지 않도록 하는 것        |
| 가용성 | availability    | 허가된 사용자가 원하는 경우 지체 없이 정보에 접근할 수 있도록 하는 것 |

시험에서는 정보보안의 목표로 `기밀성`, `무결성`, `가용성`을 반드시 기억한다.

---

### 1.4 정보보안의 종류

정보보안은 크게 시스템 보안과 네트워크 보안으로 나눌 수 있다.

| 종류      | 설명                                                    |
|---------|-------------------------------------------------------|
| 시스템 보안  | 권한이 없는 사용자가 파일이나 장치 등을 사용하는 것을 제한하여 시스템을 보호하는 방법      |
| 네트워크 보안 | 권한이 없는 사용자의 네트워크 접근이나 우연적·의도적 방해와 파괴로부터 네트워크를 보호하는 방법 |

---

### 1.5 네트워크 보안 요구사항

| 요구사항    | 의미                            |
|---------|-------------------------------|
| 실체 인증   | 정보통신에 참여한 실체에 대한 진위를 검증       |
| 데이터 무결성 | 데이터가 파괴되거나 변경되지 않도록 함         |
| 데이터 보안성 | 데이터의 비밀이 보장되고 사용자 접근이 제어됨     |
| 데이터 인증  | 데이터가 신뢰할 수 있는 자로부터 전송된 것인지 확인 |
| 부인 방지   | 데이터 송신이나 수신 사실을 부인할 수 없도록 함   |

---

## 2. 해킹

### 2.1 해킹의 정의

해킹(hacking)은 1950년대 말 미국 MIT 동아리 모임에서 처음 사용된 `해크(hack)`에서 유래하였다.

초기에는 작업 과정 자체에서 느껴지는 순수한 즐거움이라는 의미가 있었으나, 일부 사람들의 악의적인 행동이 늘어나면서 부정적인 의미를 갖게 되었다.

| 구분    | 의미                                                   |
|-------|------------------------------------------------------|
| 해킹    | 의도와 상관없이 다른 컴퓨터에 침입하는 모든 행위                          |
| 크래킹   | 다른 컴퓨터 시스템에 파괴적인 계획을 가지고 침입하는 행위                     |
| 법적 의미 | 정보통신망을 통해 타인의 컴퓨터 시스템에 접근권한 없이 무단 침입하여 법적 행위를 저지르는 것 |

시험에서는 “파괴적인 계획을 가지고 침입”이라는 표현이 나오면 `크래킹(cracking)`을 떠올린다.

---

### 2.2 해킹 방법

대표적인 해킹 방법은 다음과 같다.

| 방법        | 영어       | 핵심                 |
|-----------|----------|--------------------|
| 백도어       | backdoor | 몰래 들어갈 수 있는 비밀 통로  |
| 스니핑       | sniffing | 네트워크 패킷을 엿봄        |
| 스푸핑       | spoofing | 주소를 속여 권한이나 정보를 탈취 |
| 서비스 거부 공격 | DoS      | 과도한 요청으로 서비스를 마비시킴 |

---

### 2.3 백도어

백도어(backdoor)는 허가받지 않은 사용자가 인터넷과 같은 네트워크를 통해 다른 시스템에 들어갈 수 있을 만큼 허술한 부분을 의미한다.

- 시스템 관리자 등이 점검 목적으로 만든 통로가 악용될 수 있음
- 보안이 제거된 비밀통로처럼 동작할 수 있음
- 침입 사실을 은폐하기 쉽고 흔적이 남지 않을 수 있음

---

### 2.4 스니핑

스니핑(sniffing)은 네트워크상에서 전달되는 다른 사람들의 패킷을 엿보며 계정 ID와 패스워드 등을 알아내려는 행위이다.

| 구분      | 설명                               |
|---------|----------------------------------|
| sniff   | 코를 킁킁거리다, 냄새를 맡다                 |
| sniffer | 통신 중인 패킷을 가져와 네트워크 상태를 확인하는 프로그램 |
| 위험      | 계정 정보, 비밀번호, 통신 내용 유출 가능         |
| 예방      | 데이터 암호화                          |

---

### 2.5 스푸핑

스푸핑(spoofing)은 해커가 어떤 호스트의 IP 주소나 이메일 주소를 바꾸어 일반 사용자의 권한을 획득한 뒤 정보를 빼내는 해킹 방법이다.

- spoof: 속이다, 패러디하다, 이메일을 도용하다
- IP 주소나 이메일 주소를 위조
- 가짜 웹사이트 개설 가능
- 사용자를 속여 개인정보나 권한을 탈취

---

### 2.6 서비스 거부 공격

서비스 거부 공격(DoS, Denial of Service)은 서버가 처리할 수 있는 능력 이상의 서비스를 요구하여 다른 서비스를 정지시키거나 시스템을 다운시키는 공격이다.

| 구분   | 설명                               |
|------|----------------------------------|
| DoS  | 하나의 공격자가 과도한 요청을 보내 시스템 기능을 마비시킴 |
| DDoS | 다수의 컴퓨터를 이용하여 목표 시스템을 집중 공격      |

DDoS는 Distributed DoS attack의 약자로, 많은 수의 컴퓨터에 공격 프로그램을 설치한 뒤 목표 사이트를 집중적으로 공격하여 성능 저하나 기능 마비를 일으킨다.

---

## 3. 악성 프로그램

### 3.1 악성 프로그램의 의미

악성 프로그램은 사용자 시스템에 침투하여 정보를 유출하거나 정상적인 동작을 지연·방해하는 프로그램이다.

대표적인 종류는 다음과 같다.

| 종류       | 영어             |
|----------|----------------|
| 컴퓨터 바이러스 | computer virus |
| 웜        | worm           |
| 트로이 목마   | Trojan horse   |

---

### 3.2 컴퓨터 바이러스

컴퓨터 바이러스는 컴퓨터에서 실행되는 일종의 명령어들이 결합하면서 컴퓨터 프로그램이나 데이터 파일을 감염시키는 악성 프로그램이다.

#### 감염 방식

- 정상 파일에 바이러스 코드를 붙임
- 본래 목적 이외의 작업을 처리함
- 자신을 복제함

#### 주요 증상

| 증상      | 설명                     |
|---------|------------------------|
| 부팅 문제   | 부팅 시간이 증가하거나 부팅이 되지 않음 |
| 프로그램 문제 | 프로그램이 오동작하거나 실행되지 않음   |
| 데이터 문제  | 저장된 데이터가 변조되거나 삭제됨     |

#### 방어 방법

- 최신 바이러스 백신 프로그램 사용
- 주기적인 검사
- 백신 프로그램 업데이트

---

### 3.3 웜

웜(worm)은 네트워크를 통해 자신을 복제하고 전파할 수 있는 프로그램이다.

| 구분       | 설명                      |
|----------|-------------------------|
| 특징       | 감염시키지 않고도 자신을 복제할 수 있음  |
| 바이러스와 차이 | 바이러스는 감염시키지만 복제에 한계가 있음 |
| 주요 감염 경로 | email, P2P, 메신저 등       |
| 주요 피해    | 시스템 접근 통로 제공, DDoS 공격   |

웜은 네트워크를 통해 빠르게 전파될 수 있으므로 피해 범위가 커질 수 있다.

---

### 3.4 트로이 목마

트로이 목마(Trojan horse)는 컴퓨터 사용자의 정보를 빼내기 위한 목적으로 제작된 악성 프로그램이다.

- 악성 코드를 유틸리티 프로그램에 내장하여 배포
- 정상 프로그램처럼 위장하여 배포
- 사용자가 스스로 실행하도록 유도
- 자기 복제능력은 없음

#### 바이러스, 웜, 트로이 목마 비교

| 구분    | 바이러스          | 웜           | 트로이 목마       |
|-------|---------------|-------------|--------------|
| 감염 대상 | 파일이나 프로그램     | 네트워크와 시스템   | 사용자 실행 프로그램  |
| 자기 복제 | 가능            | 가능          | 없음           |
| 전파 방식 | 감염된 파일 실행     | 네트워크 전파     | 정상 프로그램처럼 위장 |
| 핵심 피해 | 파일 감염, 데이터 손상 | 빠른 전파, DDoS | 정보 유출, 권한 탈취 |

---

## 3.5 악성 프로그램의 감염 유형

대표적인 감염 유형은 피싱과 파밍이다.

| 유형 | 영어       | 핵심                        |
|----|----------|---------------------------|
| 피싱 | phishing | 가짜 메시지나 사이트로 개인정보 입력 유도   |
| 파밍 | pharming | 도메인 또는 DNS를 속여 가짜 사이트로 유도 |

---

### 3.6 피싱

피싱(phishing)은 Private information과 Fishing의 합성어이다.

인터넷에서 송신자를 알리지 않는 스팸 메일 등을 이용하여 수신자의 개인정보를 빼내 불법적으로 이용하는 범죄이다.

#### 피싱 사례

- 이메일로 특정 웹사이트에 접속하도록 유도
- 접속한 웹사이트에서 신상 정보 요구
- 이벤트나 설문조사로 위장
- 전화 피싱도 존재

시험에서는 “스팸 메일을 이용하여 개인정보를 빼내는 것”이라는 표현이 나오면 피싱을 떠올린다.

---

### 3.7 파밍

파밍(pharming)은 Phishing과 Farming의 합성어이다.

정당한 웹사이트의 도메인을 탈취하거나 DNS 이름을 속여 미리 정해 놓은 웹사이트로 유인한 뒤 개인정보를 빼내 불법적으로 이용하는 범죄이다.

#### 파밍의 특징

- 사용자가 정상 주소를 입력해도 가짜 사이트로 이동할 수 있음
- 유인한 웹사이트가 합법적인 웹사이트와 거의 동일하게 보일 수 있음
- 피싱보다 더 위협적일 수 있음

#### 방어 방법

- 웹 브라우저 보안성 강화
- 전자서명 이용
- 의심스러운 보안 경고 확인
- 주소창과 인증서 확인

---

## 4. 보안 강화 방법

대표적인 보안 강화 방법은 다음과 같다.

1. 암호화
2. 전자서명
3. 백신 프로그램
4. 방화벽 사용

---

## 4.1 암호화

암호화(encryption)는 인터넷에서 누구나 읽을 수 있는 평문을 제3자가 읽을 수 없는 형태인 암호문으로 변환하는 과정이다.

| 구분  | 설명               |
|-----|------------------|
| 평문  | 암호화 전의 원래 문장     |
| 암호문 | 암호화 후의 문장        |
| 암호화 | 평문을 암호문으로 바꾸는 과정 |
| 복호화 | 암호문을 평문으로 바꾸는 과정 |

흐름은 다음과 같다.

    평문 → 암호화 알고리즘 → 암호문 → 복호화 알고리즘 → 평문

---

### 4.2 암호화 방식

암호화 방식은 크게 대칭키 암호화 방식과 공개키 암호화 방식으로 나뉜다.

| 방식         | 다른 이름            | 핵심               |
|------------|------------------|------------------|
| 대칭키 암호화 방식 | 비밀키 암호화, 관용키 암호화 | 암호화 키와 복호화 키가 같음 |
| 공개키 암호화 방식 | 비대칭키 암호화         | 암호화 키와 복호화 키가 다름 |

---

### 4.3 대칭키 암호화 방식

대칭키 암호화 방식은 암호화 키와 복호화 키가 같은 방식이다.

- 암호화 키 = 복호화 키
- 비밀키 암호화 방식이라고도 함
- 관용키 암호화 방식이라고도 함

#### 장점

- 구현이 용이함
- 실행 속도가 빠름

#### 단점

- 키 분배와 관리가 어려움
- 인증이 보장되지 않음
- 송수신 부인 방지가 보장되지 않음

#### 예

- RC2
- RC4
- RC5
- IDEA
- DES
- Triple DES

---

### 4.4 공개키 암호화 방식

공개키 암호화 방식은 암호화 키와 복호화 키가 다른 방식이다.

- 암호화 키 ≠ 복호화 키
- 비대칭키 암호화 방식이라고도 함
- 암호화 키는 공개키
- 복호화 키는 비밀키 또는 개인키

#### 장점

- 대칭키 암호화 방식의 키 분배 문제를 해결
- 디지털 서명 기능 제공
- 부인 방지 가능

#### 단점

- 구현이 어려움
- 처리 속도가 느림

#### 예

- RSA

---

### 4.5 대칭키와 공개키 비교

| 구분    | 대칭키 암호화          | 공개키 암호화          |
|-------|------------------|------------------|
| 키 관계  | 암호화 키와 복호화 키가 같음 | 암호화 키와 복호화 키가 다름 |
| 다른 이름 | 비밀키, 관용키         | 비대칭키             |
| 속도    | 빠름               | 느림               |
| 구현    | 비교적 쉬움           | 어려움              |
| 키 관리  | 어려움              | 상대적으로 유리         |
| 전자서명  | 어려움              | 가능               |
| 예     | DES, Triple DES  | RSA              |

---

## 4.6 전자서명

전자서명은 공개키 암호화 방식에서 개인키를 이용하여 메시지를 암호화함으로써 메시지 작성자 본인을 확인할 수 있게 하는 기술이다.

공개키 암호화 방식에서 개인키를 이용한 메시지 암호화는 메시지 작성자만 할 수 있으므로, 이를 이용하여 메시지 작성자 본인을 알리는 서명을 작성할 수 있다.

| 구분  | 설명                                  |
|-----|-------------------------------------|
| 개인키 | 작성자가 서명할 때 사용                       |
| 공개키 | 수신자가 서명을 검증할 때 사용                   |
| PKI | Public Key Infrastructure, 공개키 기반구조 |

---

### 4.7 전자서명의 기본 조건

| 조건     | 영어                  | 설명                              |
|--------|---------------------|---------------------------------|
| 서명자 인증 | user authentication | 전자서명의 서명자를 불특정 다수가 검증할 수 있어야 함  |
| 부인 불가  | non-repudiation     | 서명자는 서명 사실을 부인할 수 없어야 함         |
| 변경 불가  | unalterable         | 서명한 문서의 내용이 변경되지 않아야 함          |
| 재사용 불가 | not reusable        | 한 문서의 전자서명을 다른 문서에 재사용할 수 없어야 함 |
| 위조 불가  | unforgeable         | 적법한 서명자만 전자서명을 할 수 있어야 함        |

---

## 4.8 백신 프로그램

백신 프로그램은 악성 프로그램을 검사하고 치료하는 보안 프로그램이다.

- 사용자가 원하는 시점에 악성 프로그램 검사 및 치료 가능
- 실시간 감시 지원
- 항상 활성화해 두는 것이 중요함
- 프로그램을 항상 최신 상태로 업데이트해야 함

---

## 4.9 방화벽

방화벽(firewall)은 인터넷에 내재된 취약점에 대처하기 위한 네트워크 보안 해결 방법이다.

네트워크와 네트워크 사이에서 송수신되는 패킷을 검사하여 조건에 맞는 패킷만 통과시키는 소프트웨어나 하드웨어를 총칭한다.

| 구분 | 설명              |
|----|-----------------|
| 역할 | 네트워크 사이의 패킷 검사  |
| 목적 | 외부의 불법 접근 차단    |
| 형태 | 소프트웨어 또는 하드웨어   |
| 핵심 | 조건에 맞는 패킷만 통과시킴 |

---

## 정리하기

1. 정보보안의 목표는 기밀성, 무결성, 가용성이다.

2. 정보보안은 시스템 보안과 네트워크 보안으로 구분할 수 있다.

3. 네트워크 보안 요구사항에는 실체 인증, 데이터 무결성, 데이터 보안성, 데이터 인증, 부인 방지가 있다.

4. 해킹은 컴퓨터 네트워크의 보안 취약점을 찾아 시스템에 침입하는 행위이며, 파괴적인 계획을 가지고 침입하는 행위는 크래킹이라고 한다.

5. 대표적인 해킹 방법에는 백도어, 스니핑, 스푸핑, 서비스 거부 공격이 있다.

6. 악성 프로그램은 사용자 시스템에 침투하여 정보를 유출하거나 정상적인 동작을 지연·방해하는 프로그램이다.

7. 악성 프로그램의 종류에는 컴퓨터 바이러스, 웜, 트로이 목마가 있다.

8. 악성 프로그램의 감염 유형에는 피싱과 파밍이 있다.

9. 암호화는 평문을 암호문으로 변환하는 과정이고, 복호화는 암호문을 평문으로 변환하는 과정이다.

10. 암호화 방식에는 대칭키 암호화 방식과 공개키 암호화 방식이 있다.

11. 대칭키 암호화 방식은 암호화 키와 복호화 키가 같고, 공개키 암호화 방식은 암호화 키와 복호화 키가 다르다.

12. 전자서명은 공개키 암호화 방식을 활용하여 메시지 작성자 인증, 문서의 부인 방지, 송신 부인 방지 등의 기능을 제공한다.

13. 백신 프로그램은 악성 프로그램을 검사하고 치료하며, 항상 최신 상태로 유지해야 한다.

14. 방화벽은 네트워크 사이에서 송수신되는 데이터를 검사하여 조건에 맞는 데이터만 통과시키는 보안 장치이다.

### 7. 암호화 방식 비교

| 단점 | 키 관리 어려움        | 느리고 구현 어려움          |
| 예  | DES, Triple DES | RSA                 |

## 연습문제 정리

### Q1. 정보보안의 목표로서 부적절한 것은?

1. 기밀성
2. 확장성
3. 가용성
4. 무결성

정답: 2

해설:
정보보안의 3대 목표는 기밀성, 무결성, 가용성이다. 확장성은 정보보안의 주요 목표가 아니다.

---

### Q2. “파괴적인 계획을 가지고 다른 컴퓨터에 침입하는 행위”를 가리키는 용어는?

1. 백도어
2. 스니핑
3. 스푸핑
4. 크래킹

정답: 4

해설:
크래킹은 파괴적인 계획을 가지고 다른 컴퓨터 시스템에 침입하는 행위를 의미한다.

---

### Q3. 공격자가 컴퓨터가 감당할 수 없을 정도로 많은 양의 데이터를 전송하여 과도한 부하로 정보 시스템의 기능을 마비시키는 해킹 방법은?

1. 피싱
2. 백도어
3. 트로이 목마
4. 서비스 거부

정답: 4

해설:
서비스 거부 공격은 서버가 처리할 수 있는 능력 이상의 서비스를 요구하여 다른 서비스를 정지시키거나 시스템을 다운시키는 공격이다.

---

### Q4. 인터넷에서 도메인을 탈취하거나 DNS 이름을 속여 사용자를 미끼 웹사이트로 유인한 다음, 유인된 사용자의 개인정보를 빼내 불법적으로 사용하는 것은?

1. 파밍
2. 컴퓨터 바이러스
3. 스니핑
4. 트로이 목마

정답: 1

해설:
파밍은 정당한 웹사이트의 도메인을 탈취하거나 DNS 이름을 속여 사용자를 가짜 사이트로 유도한 뒤 개인정보를 탈취하는 방식이다.

---

### Q5. 다음 중 암호화에 관련된 설명으로 옳은 것은?

1. 암호화는 암호화된 문장을 평문으로 변환하는 과정이다.
2. 복호화는 평문을 암호문으로 변환하는 과정이다.
3. 대칭키 암호화 방식은 암호화 키와 복호화 키가 서로 같은 암호 방식이다.
4. 공개키 암호화 방식은 암호화 키와 복호화 키가 서로 같은 암호 방식이다.

정답: 3

해설:
대칭키 암호화 방식은 암호화 키와 복호화 키가 같다. 암호화는 평문을 암호문으로 변환하는 과정이고, 복호화는 암호문을 평문으로 변환하는 과정이다. 공개키 암호화 방식은 암호화 키와 복호화 키가 서로 다르다.

---

### Q6. 다음 중 정보보안 강화 방법으로 부적절한 것은?

1. 전자서명
2. 분산 서비스 거부
3. 방화벽
4. 백신 프로그램

정답: 2

해설:
전자서명, 방화벽, 백신 프로그램은 정보보안 강화 방법이다. 분산 서비스 거부 공격(DDoS)은 보안 강화 방법이 아니라 공격 방법이다.

## 시험 직전 체크포인트

### 반드시 암기하기

- 정보보안 목표 = 기밀성 + 무결성 + 가용성
- 정보보안 종류 = 시스템 보안 + 네트워크 보안
- 네트워크 보안 요구사항 = 실체 인증 + 데이터 무결성 + 데이터 보안성 + 데이터 인증 + 부인 방지

- 악성 프로그램 = 바이러스 + 웜 + 트로이 목마
- 감염 유형 = 피싱 + 파밍
- 보안 강화 방법 = 암호화 + 전자서명 + 백신 + 방화벽
- 대칭키 = 암호화 키와 복호화 키가 같음
- 공개키 = 암호화 키와 복호화 키가 다름
