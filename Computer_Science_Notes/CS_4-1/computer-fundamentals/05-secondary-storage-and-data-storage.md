## 2. 보조기억장치 개요

### 2-1. 보조기억장치의 필요성

- 컴퓨터 시스템은 크게 다음으로 구성됨
    - 입력장치
    - CPU
    - 주기억장치
    - 출력장치
    - 보조기억장치

### 주기억장치와 보조기억장치

| 구분     | 설명                        |
|--------|---------------------------|
| 주기억장치  | CPU가 직접 읽거나 기록할 수 있는 기억장치 |
| 보조기억장치 | 주기억장치를 보조하는 2차 기억장치       |

### 보조기억장치가 필요한 이유

1. 대용량의 기억장치가 필요함

- 주기억장치는 프로그램과 데이터를 모두 수용하기에는 충분하지 않음
- 정보화 확장, 멀티미디어 데이터, SNS 활동 등으로 데이터가 큰 폭으로 증가함

2. 비휘발성 기억장치가 필요함

- 전원 공급 없이 데이터를 보관할 수 있어야 함

3. 주기억장치의 효율적 사용

- 지역성의 원리를 활용한 기억장치 계층구조 구성
- CPU 레지스터 → 캐시 기억장치 → 주기억장치 → 보조기억장치 순으로 계층화됨

### 기억장치 계층구조의 특징

| 위치          | 접근 속도 | 가격 | 용량  |
|-------------|-------|----|-----|
| CPU에 가까움    | 빠름    | 고가 | 소량  |
| 보조기억장치에 가까움 | 느림    | 저가 | 대용량 |

### 핵심 정리

- 보조기억장치는 대용량 데이터를 비휘발성으로 저장하기 위해 필요함
- 주기억장치보다 느리지만 용량이 크고 가격이 낮음
- 기억장치 계층구조는 성능, 비용, 저장 용량의 균형을 맞추기 위한 구조임

## 3. 보조기억장치의 종류

### 3-1. 접근 방식에 따른 분류

| 접근 방식 | 매체 형태 | 매체 종류                                              |
|-------|-------|----------------------------------------------------|
| 순차 접근 | 자기테이프 | 릴 테이프, 카트리지 테이프(DAT, DLT, LTO 등)                   |
| 직접 접근 | 자기디스크 | 플로피디스크, 하드디스크                                      |
| 직접 접근 | 광디스크  | CD-ROM, CD-R, CD-RW, DVD, DVD-R, HD-DVD, Blu-ray 등 |
| 직접 접근 | 기타 매체 | 플래시 메모리, SSD                                       |

### 핵심 정리

- 자기테이프는 순차 접근 방식임
- 자기디스크, 광디스크, SSD는 직접 접근 방식임
- 대화식 처리에는 직접 접근 저장장치가 적합함

---

## 4. 자기테이프 기억장치

### 4-1. 자기테이프 기억장치의 특성

- 순차접근 기억장치임
- 물리적 순서에 따라 데이터를 저장함
- 기록된 순서에 따라 데이터를 읽을 수 있음
- 원하는 위치에 신속하게 접근하기 어려움
- 대화식 처리에 직접 사용하기에는 적합하지 않음
- 주로 데이터 백업용으로 사용함

### 핵심 정리

- 자기테이프는 순서대로 저장하고 순서대로 읽는 저장장치임
- 속도는 느리지만 대용량 백업 용도로 적합함

---

### 4-2. 초기의 자기테이프 드라이브

- 초기에는 릴 형태의 자기테이프 드라이브가 사용됨
- 대표 예:
    - IBM System 360 테이프 드라이브
    - DEC TE 16 9-트랙 테이프 드라이브

### 핵심 정리

- 초기 자기테이프는 대형 장비 형태로 사용되었음
- 컴퓨터 초기 시대의 대표적인 대용량 저장 매체였음

---

### 4-3. 카트리지 테이프

- 최근의 자기테이프는 카트리지 내부에 장착됨
- 테이프를 보호하고 다루기 편리하게 함

### 대표 종류

- DAT
    - Digital Audio Tape
- DLT
    - Digital Linear Tape
- LTO
    - Linear Tape-Open

### 저장 용량 예시

- LTO-9:
    - 18TB
    - 압축 시 45TB

### 핵심 정리

- 카트리지 테이프는 기존 릴 테이프보다 보관과 관리가 편리함
- LTO 계열은 대용량 백업에 많이 사용됨

---

### 4-4. 테이프 라이브러리

- 대용량 저장장치임
- 최대 수 엑사바이트 수준의 대용량 기억장치 구성 가능
- 구성 요소:
    - 1개 이상의 테이프 드라이브
    - 여러 개의 테이프 카트리지 슬롯
    - 테이프 식별용 바코드 입력기
    - 테이프를 자동으로 드라이브에 장착하는 로봇

### 활용

- 백업
- 자주 사용되지 않는 데이터 저장
- 장기 보관 데이터 관리

### 핵심 정리

- 테이프 라이브러리는 많은 테이프를 자동으로 관리하는 대용량 저장 시스템임
- 대규모 백업과 아카이브에 적합함

---

## 5. 자기디스크 기억장치

### 5-1. 직접접근 저장장치

- 데이터의 위치를 지정하여 데이터를 저장하거나 읽어 낼 수 있는 저장장치
- 대화식 처리를 위한 빠른 데이터 처리를 위해 필요함

### 대표 직접접근 저장장치

- 자기디스크
- 광디스크
- 반도체 저장장치

### 핵심 정리

- 직접접근 저장장치는 원하는 위치의 데이터를 빠르게 읽거나 쓸 수 있음
- 대화식 처리와 실시간 데이터 접근에 적합함

---

### 5-2. 자기디스크 기억장치 개요

- 자성체가 코팅된 원판 형태의 기록 매체를 사용함
- 판독/기록 헤드를 움직여 원하는 위치에 접근함
- 대표적인 직접접근 저장장치임

### 초기 사례

- IBM 350 디스크 저장장치
    - 최초의 디스크 드라이브
    - IBM 305 RAMAC 컴퓨터 시스템에 사용
    - 1956년

### 핵심 정리

- 자기디스크는 회전하는 디스크와 이동하는 헤드를 이용해 데이터를 읽고 씀
- 직접접근이 가능하다는 점에서 자기테이프와 다름

---

### 5-3. 플로피디스크

- 산화철로 코팅된 편평하고 둥근 플라스틱으로 구성됨
- 크기:
    - 8인치
    - 5.25인치
    - 3.5인치
- 소형 컴퓨터에서 휴대용 저장 매체로 사용되었음
- 최근에는 USB 플래시 드라이브 등으로 대체되어 거의 사용되지 않음

### 핵심 정리

- 플로피디스크는 과거 휴대용 저장매체였으나 현재는 거의 사용되지 않음

---

### 5-4. 하드디스크

- 1개의 회전축에 자성체가 코팅된 여러 개의 금속 디스크를 결합하여 고속 회전함
- 액추에이터 암에 연결된 읽기·쓰기 헤드가 기록면 위에서 이동함
- 원하는 위치에서 데이터를 읽거나 씀
- 구성품을 밀봉하여 먼지나 이물질 오염을 막음
- 디스크 기록면과 헤드를 가깝게 유지하여 고속·정밀 동작 가능

### 하드디스크 주요 구성

- 회전축
- 디스크
- 읽기·쓰기 헤드
- 액추에이터 암

### 핵심 정리

- 하드디스크는 회전하는 디스크와 움직이는 헤드를 이용한 자기디스크 저장장치임
- 밀봉 구조를 통해 정밀한 고속 동작이 가능함

---

### 5-5. 하드디스크의 데이터 저장 구조

| 용어  | 설명                   |
|-----|----------------------|
| 트랙  | 디스크 표면에 배열된 원형 기록 경로 |
| 섹터  | 각 트랙을 나눈 데이터 저장 구획   |
| 실린더 | 동일 반경에 위치한 트랙의 집합    |

### 핵심 정리

- 하드디스크는 트랙과 섹터 단위로 데이터를 저장함
- 여러 디스크 면의 같은 반경 트랙 묶음을 실린더라고 함

---

### 5-6. 하드디스크의 성능

| 항목     | 설명                             |
|--------|--------------------------------|
| 탐색시간   | 헤드가 지정된 트랙으로 이동하는 시간           |
| 회전지연시간 | 디스크가 회전하여 데이터 위치가 헤드에 도달하는 시간  |
| 전송시간   | 주기억장치와 하드디스크 사이에서 데이터를 전송하는 시간 |

### 참고 수치

- 탐색시간:
    - 수 ms
- 디스크 회전 속도:
    - 4,200 ~ 15,000 RPM
- 평균 회전지연시간:
    - 약 7.1 ~ 2 ms

### 핵심 정리

- 하드디스크 성능은 탐색시간, 회전지연시간, 전송시간의 영향을 받음
- 디스크 회전 속도는 접근 속도와 밀접한 관련이 있음

---

### 5-7. 고성능 디스크의 구성: 디스크 어레이

- 여러 개의 하드디스크에 데이터를 분산하여 저장함

### 목적

- 데이터 저장의 신뢰성 향상
- 접근 성능 향상

### RAID

- RAID:
    - Redundant Array of Inexpensive Disks
    - 또는 Redundant Array of Independent Disks로도 설명됨
- 여러 디스크를 하나의 논리적인 단위로 구성하는 기술

### 핵심 정리

- RAID는 여러 하드디스크를 묶어 신뢰성과 성능을 높이는 기술임
- 서버나 대용량 저장 시스템에서 자주 사용됨

## 6. 광 디스크

### 6-1. 광 디스크란?

- 레이저를 이용하여 기록면에서 데이터를 읽는 저장매체임
- 한 장의 마스터로부터 대용량 데이터를 인쇄하듯 찍어낼 수 있어 대량생산이 가능함

### 활용

- 음악
- 영화
- 프로그램 설치 디스크
- 데이터 배포

### 핵심 정리

- 광 디스크는 레이저를 이용해 데이터를 읽음
- 대량생산이 가능하여 콘텐츠 배포 매체로 많이 사용되었음

---

### 6-2. 광 디스크의 기록 방법에 따른 분류

| 종류   | 특성                                   |
|------|--------------------------------------|
| ROM  | 제작 단계에서 한 번만 기록 가능, 내용 추가·수정 불가      |
| WORM | 공백 상태로 제작된 디스크에 1회만 기록 가능, 영구적 기록용   |
| RW   | 반복적으로 쓰고 지울 수 있음, 약 1,000회 정도 재기록 가능 |

### 핵심 정리

- ROM은 읽기 전용
- WORM은 한 번 쓰고 여러 번 읽음
- RW는 반복 기록 가능

---

### 6-3. CD

- CD:
    - Compact Disc
- 오디오용으로 개발되었으나 이후 데이터 저장용으로 활용됨
- 직경 120mm 표준 CD는 74분의 스테레오 음악 저장을 목표로 개발됨

### 용량

- 약 650MB ~ 700MB

### 종류

- CD-ROM
- CD-R
- CD-RW

### 핵심 정리

- CD는 오디오와 데이터 저장에 사용된 대표적 광 디스크임
- 용량은 약 650~700MB 수준임

---

### 6-4. DVD

- DVD:
    - Digital Versatile Disc
- 고화질 영화를 저장하기 위한 기록 매체로 개발됨
- 해상도 720×480 지원
- 5.1채널 돌비 AC3 입체음향
- 다국어 음성 및 자막 제공 가능
- CD보다 파장이 짧은 적색 레이저를 사용하여 용량을 높임

### 용량

| 구분     | 용량    |
|--------|-------|
| 싱글 레이어 | 4.7GB |
| 듀얼 레이어 | 8.5GB |

### 핵심 정리

- DVD는 CD보다 저장 용량이 크며 영상 저장에 적합함
- 적색 레이저를 사용해 기록 밀도를 높임

---

### 6-5. Blu-ray Disc

- 고선명 HD 비디오 저장을 위한 디지털 기록 매체
- HD 화질의 2시간 분량 비디오 저장 가능
- DVD보다 파장이 더 짧은 청색 반도체 레이저를 사용함
- DVD보다 더 큰 저장 용량 제공

### 용량

| 구분     | 용량           |
|--------|--------------|
| 싱글 레이어 | 25GB         |
| 듀얼 레이어 | 50GB         |
| BDXL   | 100GB, 128GB |

### 핵심 정리

- Blu-ray는 청색 레이저를 사용해 DVD보다 큰 용량을 제공함
- HD 영상 저장에 적합함

---

### 6-6. 광 디스크의 데이터 저장 방식

### CLV 방식

- CLV:
    - Constant Linear Velocity
- 광 디스크에서 주로 사용되는 방식
- 일정한 선속도로 회전하여 기록 밀도와 전송 속도를 일정하게 유지함
- 디스크 중앙에서 바깥쪽으로 향하는 하나의 나선형 트랙을 사용함
- CAV 방식에 비해 직접접근 속도는 느림

### CAV 방식

- CAV:
    - Constant Angular Velocity
- 자기디스크에서 주로 사용되는 방식
- 일정한 각속도로 회전함
- 직접접근을 빠르게 할 수 있음
- 일반 데이터 저장용으로 광디스크 드라이브에서 지원하기도 함
- 동심원 트랙 구조를 사용함

### 핵심 정리

| 방식  | 특징                            |
|-----|-------------------------------|
| CLV | 일정한 선속도, 나선형 트랙, 광디스크에서 주로 사용 |
| CAV | 일정한 각속도, 동심원 트랙, 직접접근이 빠름     |

---

## 7. 반도체 기억장치

### 7-1. USB 플래시 드라이브

- 플래시 메모리와 USB를 결합한 저장장치

### 플래시 메모리

- 전기적으로 데이터를 지우고 다시 기록할 수 있는 비휘발성 반도체 메모리

### USB

- Universal Serial Bus
- 컴퓨터와 주변기기 사이의 데이터 교환 버스 규격

### USB 전송 속도 예

| 규격      | 최대 전송 속도   |
|---------|------------|
| USB 2.0 | 최대 480Mbps |
| USB 3.2 | 최대 20Gbps  |

### 특징

- 컴퓨터가 켜진 상태에서 자유롭게 연결과 분리가 가능함
- 크기가 작고 가벼워 휴대용 저장장치로 적합함

### 핵심 정리

- USB 플래시 드라이브는 휴대성이 뛰어난 비휘발성 반도체 저장장치임

---

### 7-2. SLC 방식과 MLC 방식

| 구분    | SLC 방식            | MLC 방식               |
|-------|-------------------|----------------------|
| 뜻     | Single Level Cell | Multi Level Cell     |
| 저장 방식 | 1개의 기억 소자에 1비트 저장 | 1개의 기억 소자에 2비트 이상 저장 |
| 장점    | 안정성이 높고 처리 속도가 빠름 | 가격이 저렴함              |
| 단점    | 가격이 높음            | 안정성과 속도가 낮음          |
| 관련 종류 | -                 | DLC, TLC, QLC        |

### 핵심 정리

- SLC는 빠르고 안정적이지만 비쌈
- MLC는 저렴하지만 상대적으로 안정성과 속도가 낮음

---

### 7-3. SSD

- SSD:
    - Solid-State Drive
- 자기디스크와 동일한 인터페이스를 가지지만,
  디스크 대신 반도체 메모리를 사용하여 데이터를 저장함
- 플래시 메모리를 사용하는 방식과 DRAM·배터리를 사용하는 방식이 있음
- 주로 플래시 메모리를 사용함

### SSD의 장점

- 움직이는 기계 부품이 없음
- 내구성이 강함
- 전력 소모가 적음
- 기계적 움직임이 없어 데이터를 읽는 시간이 매우 빠름
- 무게가 가벼움

### SSD의 단점

- 기록과 삭제 횟수가 한정되어 수명이 짧을 수 있음
- 데이터 기록 시 먼저 블록 삭제 후 쓰기를 수행해야 함
- 읽기와 쓰기의 속도가 비대칭임
- 가격이 비쌈

### 핵심 정리

- SSD는 하드디스크보다 빠르고 가볍고 전력 소모가 적음
- 다만 기록·삭제 수명, 가격, 쓰기 구조 측면의 한계가 있음

---

## 8. 클라우드 저장장치의 활용

### 8-1. 클라우드 컴퓨팅

- 인터넷을 기반으로 컴퓨터의 자원을 언제 어디서나 필요한 만큼 이용하고 비용을 지불하는 컴퓨팅 방식

### 제공 자원

- 계산능력
- 저장공간
- 서비스 등

### 8-2. 클라우드 저장장치

- 서비스를 제공하는 사업자가 다수의 분산된 저장장치로 거대한 저장공간을 구성함
- 사용자 또는 기관은 서비스 제공자와 계약하여 사용량에 따른 비용을 지불하고 사용함

### 핵심 정리

- 클라우드 저장장치는 인터넷을 통해 접근 가능한 원격 저장공간임
- 사용자는 직접 저장장치를 보유하지 않아도 필요한 만큼 저장공간을 사용할 수 있음

---

### 8-3. 파일 호스팅 서비스

- 사용자에게 파일 저장공간을 제공하는 인터넷 호스팅 서비스

### 대표 서비스

- Google Drive
- Microsoft OneDrive
- DropBox
- Apple iCloud Drive
- NAVER MYBOX 등

### 활용

- 개인 파일 저장
- 파일 동기화
- 파일 공유

### 핵심 정리

- 파일 호스팅 서비스는 인터넷이 연결되어 있으면 언제 어디서든 파일 저장·동기화·공유가 가능함

---

## 9. 정리하기

1. 보조기억장치는 주기억장치를 보조하는 2차 기억장치로,
   주기억장치에 비해 접근속도는 느리지만
   낮은 비용으로 대량의 데이터를 저장할 수 있고
   비휘발성 기억장치이므로 데이터를 안정적으로 보관할 수 있음

2. 보조기억장치에는 자기테이프와 같은 순차접근 저장장치,
   자기디스크, 광디스크, SSD 등의 직접접근 저장장치가 있음

3. 자기테이프는 순차적으로 데이터를 저장하고 읽을 수 있으며,
   대화식 처리보다는 주로 데이터 백업용으로 사용됨

4. 테이프 라이브러리는 슬롯에 있는 여러 개의 테이프 카트리지를
   자동으로 테이프 드라이브에 넣고 꺼낼 수 있는 로봇을 이용하여
   대용량 기억장치를 구성함

5. 자기디스크는 회전하는 원판 위에서 헤드를 움직여
   원하는 위치에서 데이터를 읽고 쓸 수 있는 직접접근 저장장치임

6. 디스크의 회전속도와 헤드가 지정된 트랙으로 움직이는 속도는
   디스크의 접근 시간과 밀접한 관계가 있음

7. 광디스크는 빛의 반사를 이용하여 데이터를 저장하는 디스크 장치로
   CD, DVD, Blu-ray disc 등이 이에 해당함

8. SSD는 플래시메모리와 같은 반도체 기억장치를 이용하여
   자기디스크와 동일한 인터페이스에 연결할 수 있도록 만든 장치임

9. 파일 호스팅 서비스를 이용하면 클라우드 저장장치에 파일을 저장하거나 공유할 수 있으며,
   인터넷에 연결되어 있다면 언제 어디서든 저장된 파일을 이용할 수 있음

---

## 11. 연습문제 정리

### Q1. 기억장치 구조를 계층적으로 설계하는 목적을 적절히 설명한 것은?

1. 성능과 비용, 저장 용량의 균형을 맞춘 컴퓨터 시스템을 구성한다.
2. 컴퓨터의 기억장치를 고속의 메모리만으로 구성하여 성능을 최적화한다.
3. 컴퓨터의 기억장치를 주기억장치만으로 구성하여 보조기억장치의 필요성을 없앤다.
4. 컴퓨터 시스템에서 고가의 기억장치를 제거하여 저렴한 컴퓨터를 만들기 위한 것이다.

- 정답: **1**

### 해설

- 기억장치 계층구조는 빠르지만 비싼 기억장치와,
  느리지만 저렴하고 대용량인 기억장치를 조합하여
  성능과 비용, 저장 용량의 균형을 맞추기 위한 구조임

---

### Q2. 하드디스크 장치에 대한 올바른 설명은?

1. 디스크는 한 장씩 교체할 수 있다.
2. 순차접근 기억장치이다.
3. 디스크의 회전 속도는 디스크의 접근 속도와 밀접한 관계가 있다.
4. 디스켓이라고도 부른다.

- 정답: **3**

### 해설

- 하드디스크는 직접접근 저장장치임
- 디스크의 회전 속도와 헤드 이동 속도는 접근 시간에 영향을 줌
- 플로피디스크는 디스켓이라고도 불리지만 하드디스크와는 다름

---

### Q3. 여러 개의 하드디스크에 데이터를 분산 저장하여 데이터 저장의 신뢰성을 높이거나 성능을 향상하기 위한 것은?

1. RAID
2. Blu-ray
3. DAT
4. USB

- 정답: **1**

### 해설

- RAID는 여러 개의 디스크를 하나의 논리적 단위로 구성하여
  신뢰성과 성능을 높이는 기술임

---

### Q4. 플래시 메모리를 사용하는 SSD가 자기디스크와 비교했을 때 갖는 장점에 해당하는 것은?

1. 전력 소모가 적다.
2. 기록 및 삭제 횟수가 무한히 가능하며 수명이 길다.
3. 가격이 저렴하다.
4. 읽기와 쓰기의 속도가 동일하다.

- 정답: **1**

### 해설

- SSD는 기계적으로 움직이는 부품이 없어 전력 소모가 적고 데이터 접근이 빠름
- 하지만 기록·삭제 횟수가 한정되어 수명에 제한이 있고, 가격이 비교적 비싸며, 읽기와 쓰기의 속도 차이가 있음

---

### Q5. 파일 호스팅 서비스에 대한 설명으로 옳은 것은?

1. 개인 컴퓨터의 보조기억장치를 구성해 주는 서비스이다.
2. 파일을 저장하기 위한 저장장치는 사용자의 사무실 또는 가정에 설치한다.
3. 인터넷을 통해 언제 어디서든 클라우드 저장장치를 사용할 수 있다.
4. Google 스프레드시트는 파일 호스팅 서비스에 속한다.

- 정답: **3**

### 해설

- 파일 호스팅 서비스는 인터넷을 통해 파일 저장공간을 제공하는 서비스임
- 사용자는 인터넷이 연결되어 있으면 언제 어디서든 파일 저장, 동기화, 공유 기능을 사용할 수 있음

## 12. 시험 직전 체크포인트

- 보조기억장치는 주기억장치를 보조하는 2차 기억장치이다.
- 보조기억장치는 주기억장치보다 느리지만 대용량 저장이 가능하다.
- 보조기억장치는 비휘발성 저장장치이다.
- 기억장치 계층구조는 성능, 비용, 용량의 균형을 맞추기 위한 구조이다.
- 자기테이프는 순차접근 저장장치이다.
- 자기테이프는 주로 백업과 아카이브에 사용된다.
- DAT, DLT, LTO는 카트리지 테이프의 종류이다.
- 테이프 라이브러리는 여러 테이프와 로봇을 이용한 대용량 저장장치이다.
- 직접접근 저장장치는 데이터 위치를 지정해 빠르게 접근할 수 있다.
- 하드디스크는 자기디스크 기억장치이다.
- 하드디스크는 트랙, 섹터, 실린더 구조로 데이터를 저장한다.
- 하드디스크 성능은 탐색시간, 회전지연시간, 전송시간의 영향을 받는다.
- RAID는 여러 하드디스크를 묶어 신뢰성과 성능을 높인다.
- 광디스크는 레이저를 이용해 데이터를 읽는다.
- CD는 약 650~700MB를 저장한다.
- DVD는 싱글 레이어 4.7GB, 듀얼 레이어 8.5GB를 저장한다.
- Blu-ray는 싱글 레이어 25GB, 듀얼 레이어 50GB 이상을 저장한다.
- CLV는 일정한 선속도 방식이고, 광디스크에서 주로 사용된다.
- CAV는 일정한 각속도 방식이며 직접접근이 빠르다.
- USB 플래시 드라이브는 플래시 메모리와 USB를 결합한 저장장치이다.
- SLC는 빠르고 안정적이지만 비싸다.
- MLC는 저렴하지만 안정성과 속도가 낮다.
- SSD는 반도체 메모리를 사용하므로 빠르고 가볍고 전력 소모가 적다.
- SSD는 기록·삭제 횟수 제한과 가격이 단점이다.
- 클라우드 저장장치는 인터넷 기반 원격 저장공간이다.
- 파일 호스팅 서비스는 파일 저장, 동기화, 공유에 활용된다.
