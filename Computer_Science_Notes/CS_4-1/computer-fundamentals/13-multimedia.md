## 1. 멀티미디어 개요

### 1.1 정보 전달 매체의 진화

초기의 정보 전달은 책, 종이 문서, 라디오, 텔레비전 등 전통적 매체를 중심으로 이루어졌다.

컴퓨터와 정보통신기술이 발전하면서 정보 전달 매체는 IT 기기 중심으로 변화하였다.

| 구분          | 특징                                   |
|-------------|--------------------------------------|
| 전통적 매체      | 책, 문서, 라디오, TV 등                     |
| IT 기기 기반 매체 | 컴퓨터, 스마트폰, 태블릿, 디지털 카메라, 디지털 디스플레이 등 |
| 변화 방향       | 단일 매체 중심 → 여러 매체의 통합 표현              |

---

### 1.2 컴퓨터 정보 표현 방식의 발전

초기의 컴퓨터는 숫자와 문자 중심으로 정보를 표현하였다.

표, 그래프, 그림도 문자를 이용하여 표현하는 경우가 많았다.

예를 들어 그림을 문자나 기호로 표현하는 방식은 다음과 같은 한계가 있었다.

- 표현이 단순함
- 실제 이미지처럼 자연스럽지 않음
- 소리나 동영상 표현이 어려움
- 사용자가 상호작용하기 어려움

---

### 1.3 개인용 컴퓨터와 표현 방식의 확대

1970년대 후반부터 1990년대 초반까지 개인용 컴퓨터가 보급되면서 정보 표현 방식이 넓어졌다.

| 시기                  | 특징                 |
|---------------------|--------------------|
| 1970년대 후반~1990년대 초반 | 간단한 그래픽 기능 포함      |
| 초기 개인용 컴퓨터          | 초보적인 수준의 오디오 기능 제공 |
| 대표 사례               | Apple II, IBM PC   |

---

### 1.4 1990년대 이후 다양한 매체 사용

1990년대 초반 이후에는 컴퓨터의 그래픽 처리 성능이 급격히 발전하였다.

또한 오디오 카드, 주기억장치, 보조기억장치, CPU, 주변장치 성능이 향상되면서 오디오와 비디오 데이터를 저장하고 처리할 수 있게 되었다.

| 발전 요소      | 내용                    |
|------------|-----------------------|
| 그래픽 처리     | 고해상도 이미지와 동영상 처리 가능   |
| 오디오 카드     | 입체음향 기능 제공            |
| 저장장치       | 방대한 오디오·비디오 데이터 저장 가능 |
| CPU 및 주변장치 | 멀티미디어 데이터 처리 성능 향상    |

---

## 2. 멀티미디어란?

### 2.1 멀티미디어의 의미

멀티미디어(multimedia)는 두 종류 이상의 매체가 통합되어 구성된 응용 또는 저작물을 의미한다.

여기서 매체에는 다음이 포함된다.

- 텍스트
- 그림
- 음향
- 동영상
- 애니메이션

즉, 멀티미디어는 여러 형태의 정보를 컴퓨터를 이용하여 통합적으로 표현하고 처리하는 방식이다.

---

### 2.2 멀티미디어의 특징

멀티미디어의 주요 특징은 세 가지로 정리할 수 있다.

| 특징        | 설명                                  |
|-----------|-------------------------------------|
| 다수의 매체 통합 | 텍스트, 그림, 음향, 동영상, 애니메이션 등이 유기적으로 통합 |
| 디지털 방식 표현 | 컴퓨터를 이용하여 저장·처리 가능                  |
| 상호작용성     | 사용자가 콘텐츠와 상호작용하며 의도에 따라 이용 가능       |

---

### 2.3 디지털 방식 표현의 장점

디지털 방식으로 표현된 멀티미디어는 반복적으로 재생, 전송, 복제되어도 품질이 저하되지 않는다.

| 구분 | 내용                 |
|----|--------------------|
| 저장 | 컴퓨터 저장장치에 저장 가능    |
| 처리 | 프로그램을 이용해 편집·가공 가능 |
| 복제 | 반복 복제해도 품질 저하가 없음  |
| 전송 | 네트워크를 통해 빠르게 전달 가능 |

---

### 2.4 상호작용성과 비선형 콘텐츠

멀티미디어는 사용자가 수동적으로 콘텐츠를 보는 것에 그치지 않고, 콘텐츠와 상호작용하면서 이용할 수 있다.

이러한 특징 때문에 멀티미디어 콘텐츠는 비선형 콘텐츠가 될 수 있다.

| 구분      | 설명                             |
|---------|--------------------------------|
| 선형 콘텐츠  | 처음부터 끝까지 순서대로 이용하는 콘텐츠         |
| 비선형 콘텐츠 | 사용자가 원하는 순서와 경로로 이동하며 이용하는 콘텐츠 |

예를 들어 웹 페이지에서 링크를 클릭해 원하는 문서로 이동하는 방식은 비선형 콘텐츠의 대표적인 예이다.

---

## 3. 비선형 콘텐츠

### 3.1 하이퍼텍스트

하이퍼텍스트(hypertext)는 서로 연관된 텍스트의 집합이다.

하이퍼링크(hyperlink)를 이용하여 텍스트 안에 있는 내용과 관련된 다른 텍스트가 서로 연결된다.

하이퍼링크는 다음 위치로 연결될 수 있다.

| 연결 대상                | 설명                  |
|----------------------|---------------------|
| 문서 내의 특정 위치          | 같은 문서 안의 다른 부분으로 이동 |
| 같은 컴퓨터 안의 다른 문서      | 로컬 파일 연결            |
| 통신망으로 연결된 다른 컴퓨터의 문서 | 웹 페이지나 원격 문서 연결     |

---

### 3.2 하이퍼미디어

하이퍼미디어(hypermedia)는 하이퍼텍스트가 오디오, 비디오, 그래픽스 등을 포함하도록 확장된 형태이다.

| 구분     | 의미                                |
|--------|-----------------------------------|
| 하이퍼텍스트 | 텍스트 중심의 비선형 연결                    |
| 하이퍼미디어 | 텍스트 + 오디오 + 비디오 + 그래픽스 등 멀티미디어 연결 |

---

### 3.3 웹과 하이퍼미디어

웹에서는 HTML로 작성된 문서를 HTTP 프로토콜에 따라 교환한다.

| 용어        | 원어                          | 의미              |
|-----------|-----------------------------|-----------------|
| HTML      | HyperText Markup Language   | 웹 문서를 작성하는 언어   |
| HTTP      | HyperText Transfer Protocol | 웹 문서를 주고받는 프로토콜 |
| Hyperlink | 하이퍼링크                       | 문서와 문서를 연결하는 링크 |

시험에서는 HTML과 HTTP가 모두 HyperText와 관련된다는 점을 기억한다.

## 4. 디지털 미디어

## 4.1 텍스트

### 4.1.1 텍스트의 표현

텍스트는 문자로 구성된 문서이다.

컴퓨터에서 텍스트를 표현하기 위해서는 문자 인코딩, 문자 속성, 문단 정렬 정보가 필요하다.

| 요소        | 설명                  |
|-----------|---------------------|
| 문자 인코딩 방식 | ASCII, EBCDIC, 유니코드 |
| 문자 속성     | 글꼴, 색상, 크기 등        |
| 문단 정렬     | 좌측, 우측, 중앙, 양쪽 맞춤 등 |

---

### 4.1.2 문자 인코딩

문자 인코딩은 문자를 컴퓨터가 처리할 수 있는 코드로 표현하는 방식이다.

| 인코딩 방식 | 특징                     |
|--------|------------------------|
| ASCII  | 영문자와 숫자, 기본 기호 표현에 사용  |
| EBCDIC | IBM 계열 시스템에서 사용된 문자 코드 |
| 유니코드   | 다양한 언어와 문자를 통합적으로 표현   |

---

### 4.1.3 HTML 문서 예

텍스트의 색상, 크기, 정렬은 HTML과 스타일 속성으로 표현할 수 있다.

    <!DOCTYPE html>
    <html>
    <head>
        <title>Sample</title>
    </head>
    <body>
        <p style="text-align:center; font-size:200%; color:red">텍스트</p>
        <p style="text-align:right">
            우측 맞춤으로 정렬된 문단 :
            텍스트를 표현하려면 문자들의 글꼴, 색상, 크기 등이 필요하다.
        </p>
    </body>
    </html>

---

## 4.2 디지털 이미지

### 4.2.1 디지털 이미지란?

디지털 이미지는 사진, 그림, 차트 등을 디지털 방식으로 표현한 것이다.

디지털 이미지는 다음 방법으로 생성할 수 있다.

| 생성 방법   | 설명                      |
|---------|-------------------------|
| 디지털 카메라 | 실제 장면을 디지털 사진으로 입력      |
| 스캐너     | 종이 사진이나 문서를 디지털 이미지로 변환 |
| 그래픽 패키지 | 포토샵 같은 프로그램으로 이미지 제작    |

---

### 4.2.2 이미지 표현 방식

디지털 이미지의 표현 방식은 크게 두 가지이다.

| 표현 방식  | 핵심                        |
|--------|---------------------------|
| 래스터 방식 | 픽셀을 가로와 세로로 나열하여 그림 표현    |
| 벡터 방식  | 점, 직선, 곡선 등 기하 도형으로 그림 표현 |

---

## 4.3 래스터 이미지

### 4.3.1 래스터 이미지의 의미

래스터 이미지는 가로와 세로로 점들을 나열하여 그림을 표현하는 방식이다.

이때 하나하나의 점을 픽셀(pixel), 또는 화소라고 한다.

| 개념      | 설명                |
|---------|-------------------|
| 픽셀      | 이미지를 구성하는 가장 작은 점 |
| 화소      | 픽셀의 다른 표현         |
| 래스터 이미지 | 픽셀들의 배열로 구성된 이미지  |

---

### 4.3.2 해상도

해상도(resolution)는 단위 길이당 구분할 수 있는 픽셀의 수를 의미한다.

또는 이미지의 가로와 세로 픽셀 수를 해상도라고 부르기도 한다.

| 구분      | 예                    |
|---------|----------------------|
| 프린터 해상도 | 300dpi 이상            |
| 모니터 해상도 | 24인치 풀HD 모니터 약 92ppi |
| 픽셀 수 표현 | 1,280 × 1,024        |

| 단위  | 의미                        |
|-----|---------------------------|
| dpi | dots per inch, 인치당 점의 수   |
| ppi | pixels per inch, 인치당 픽셀 수 |

해상도가 높을수록 이미지 데이터의 양이 커진다.

---

### 4.3.3 래스터 이미지의 확대

래스터 이미지는 이미지를 확대해도 더 상세한 그림을 얻을 수 없다.

확대하면 다음과 같은 문제가 발생할 수 있다.

- 경계가 거칠게 표현됨
- 이미지가 흐릿해짐
- 픽셀이 확대되어 보임
- 계단 현상 발생

---

## 4.4 벡터 이미지

### 4.4.1 벡터 이미지의 의미

벡터 이미지는 점, 직선, 곡선 등 기하 도형으로 그림을 표현하는 방식이다.

이미지를 저장하기 위한 데이터의 양은 이미지에 포함된 기하 도형의 수에 의해 결정된다.

---

### 4.4.2 벡터 이미지의 출력

벡터 이미지는 화면이나 프린터에 출력할 때 렌더링 알고리즘을 이용하여 래스터 이미지로 변환된다.

| 단계     | 설명                    |
|--------|-----------------------|
| 벡터 데이터 | 점, 선, 곡선, 도형 정보 저장    |
| 렌더링    | 도형 정보를 픽셀 이미지로 변환     |
| 출력     | 모니터나 프린터에 래스터 이미지로 표시 |

---

### 4.4.3 벡터 이미지의 확대

벡터 이미지는 이미지를 확대하면 기하 도형의 크기를 다시 계산하여 그린다.

따라서 래스터 이미지보다 자연스럽게 확대할 수 있다.

| 구분      | 래스터 이미지        | 벡터 이미지         |
|---------|----------------|----------------|
| 표현 방식   | 픽셀 배열          | 기하 도형          |
| 확대 시 품질 | 거칠거나 흐릿해질 수 있음 | 비교적 자연스럽게 확대   |
| 데이터 크기  | 해상도와 픽셀 수에 영향  | 도형 수에 영향       |
| 적합한 용도  | 사진             | 로고, 아이콘, 도형 그림 |

---

## 5. 이미지 파일 형식

### 5.1 BMP

BMP는 비트맵 이미지 파일 형식이다.

| 항목    | 내용                         |
|-------|----------------------------|
| 원어    | Bitmap                     |
| 특징    | 마이크로소프트 Windows나 OS/2에서 사용 |
| 압축    | 일반적으로 압축되지 않음              |
| 파일 크기 | 큼                          |
| 확장자   | .bmp                       |

BMP는 압축하지 않는 경우가 많아 품질은 좋지만 파일 크기가 크다.

---

### 5.2 GIF

GIF(Graphics Interchange Format)는 CompuServe사에서 이미지 전송 목적으로 개발한 파일 형식이다.

| 항목    | 내용                          |
|-------|-----------------------------|
| 원어    | Graphics Interchange Format |
| 개발 목적 | 이미지 전송                      |
| 사용    | 웹 페이지 이미지 전달                |
| 색상    | 최대 256색                     |
| 압축    | LZW 압축                      |
| 기능    | 간단한 애니메이션 지원                |
| 투명색   | 사용 가능                       |
| 확장자   | .gif                        |

GIF는 색상이 많지 않은 그림, 아이콘, 간단한 애니메이션에 적합하다.

---

### 5.3 PNG

PNG(Portable Network Graphics)는 GIF에서 사용하는 LZW 압축 알고리즘의 특허 문제를 피하기 위해 만들어진 웹용 이미지 파일 형식이다.

| 항목  | 내용                        |
|-----|---------------------------|
| 원어  | Portable Network Graphics |
| 목적  | 웹 이미지 표현                  |
| 압축  | 특허 문제가 없는 무손실 압축          |
| 색상  | 256색 팔레트, 트루 컬러 지원        |
| 확장자 | .png                      |

PNG는 웹에서 고품질 이미지와 투명 배경을 표현할 때 유용하다.

---

### 5.4 JPEG

JPEG(Joint Photographic Expert Group)는 손실압축 부호화 알고리즘을 사용하는 이미지 파일 형식이다.

| 항목  | 내용                              |
|-----|---------------------------------|
| 원어  | Joint Photographic Expert Group |
| 압축  | 손실압축                            |
| 특징  | 원본 데이터에 미세한 오차 발생               |
| 장점  | 높은 압축률                          |
| 용도  | 디지털 사진 저장                       |
| 확장자 | .jpg                            |

JPEG는 시각적으로 잘 느낄 수 없을 만큼의 손실을 허용하는 대신 매우 높은 비율로 압축할 수 있다.

---

### 5.5 이미지 파일 형식 비교

| 형식   | 압축 방식     | 주요 특징                | 적합한 용도         |
|------|-----------|----------------------|----------------|
| BMP  | 일반적으로 무압축 | 파일 크기 큼              | 원본 이미지 저장      |
| GIF  | LZW 압축    | 256색, 투명색, 간단한 애니메이션 | 아이콘, 단순 그래픽    |
| PNG  | 무손실 압축    | 트루 컬러, 투명 배경 지원      | 웹 그래픽, 고품질 이미지 |
| JPEG | 손실압축      | 높은 압축률, 사진에 적합       | 디지털 사진         |

시험에서는 “디지털 사진 저장에 유용한 파일 형식”이라고 나오면 JPEG를 기억한다.

## 6. 사운드

### 6.1 사운드의 의미

사운드는 음성, 음악, 음향효과 등 모든 종류의 소리를 의미한다.

초기 컴퓨터는 내부 스피커를 통해 단순한 사운드를 출력하였다.

현재는 고성능 사운드 카드를 통해 5.1채널, 7.1채널 등의 입체 음향을 제공할 수 있다.

---

### 6.2 오디오 출력 방식

| 구분     | 내용                                |
|--------|-----------------------------------|
| 초기 컴퓨터 | 내부 스피커를 통한 단순 사운드                 |
| 현재 컴퓨터 | 고성능 사운드 카드와 입체 음향                 |
| 출력 예   | 아날로그 오디오 출력, TOSLINK, Thunderbolt |

---

### 6.3 오디오 파일 형식

오디오 파일은 녹음된 오디오 데이터와 합성을 위한 오디오 데이터로 구분할 수 있다.

| 구분                  | 파일 형식                    |
|---------------------|--------------------------|
| ADC를 통해 녹음된 오디오 데이터 | WAV, Au, MP3, MP4, WMA 등 |
| 합성을 위한 오디오 데이터      | MIDI                     |

MIDI는 실제 소리를 녹음한 것이 아니라, 악기 연주 정보와 같은 합성용 데이터를 표현하는 데 사용된다.

---

## 7. 비디오

### 7.1 비디오 압축 기술

비디오는 데이터의 양이 매우 많기 때문에 높은 압축률을 낼 수 있는 손실압축기술을 사용한다.

대표적인 비디오 압축 기술은 다음과 같다.

| 압축 기술             | 설명               |
|-------------------|------------------|
| MPEG-1            | 초기 디지털 비디오 압축    |
| MPEG-2            | DVD, 디지털 방송 등    |
| MPEG-4            | 인터넷과 멀티미디어 서비스   |
| H.261, H.263      | 화상회의·화상전화 등에서 사용 |
| MPEG-4 AVC(H.264) | 고효율 비디오 압축       |
| HEVC(H.265)       | H.264보다 높은 압축 효율 |
| VVC(H.266)        | 차세대 고효율 비디오 압축   |

---

### 7.2 비디오 압축 기술의 활용

| 활용 분야   | 예                 |
|---------|-------------------|
| 저장      | DVD, Blu-ray Disc |
| 방송      | HDTV, DMB         |
| 비디오 서비스 | VOD               |
| 통신      | 화상전화, 화상회의        |

---

### 7.3 비디오 플레이어와 코덱

비디오 플레이어는 다양한 파일 형식을 재생하는 프로그램이다.

대표적인 비디오 플레이어는 다음과 같다.

- Windows Media Player
- QuickTime Player
- 네이버 미디어 플레이어
- 카카오 팟플레이어
- 곰플레이어

재생 가능한 파일 형식은 제공하는 코덱에 따라 달라진다.

---

### 7.4 코덱

코덱(codec)은 부호기(coder)와 복호기(decoder)를 의미한다.

| 기능  | 설명                     |
|-----|------------------------|
| 부호기 | 데이터를 압축하거나 특정 형식으로 변환  |
| 복호기 | 압축된 데이터를 풀어 재생 가능하게 변환 |

---

### 7.5 비디오 파일 형식

대표적인 비디오 파일 형식은 다음과 같다.

- AVI
- WMV
- MOV
- DivX
- MPEG

## 8. 애니메이션

### 8.1 애니메이션의 의미

애니메이션은 연속적인 장면을 나타내는 일련의 이미지를 순서대로 빠르게 디스플레이하여 움직이는 장면을 시뮬레이션하는 것이다.

## 10. 확장현실 응용

### 10.1 확장현실

확장현실(XR, eXtended Reality)은 가상의 환경으로 확장된 경험과 몰입감을 사용자에게 제공하는 실감형 기술이다.

확장현실은 가상현실, 증강현실, 혼합현실을 포괄하는 의미의 용어이다.

| 용어 | 원어                   | 의미      |
|----|----------------------|---------|
| XR | eXtended Reality     | 확장현실    |
| VR | Virtual Reality      | 가상현실    |
| AR | Augmented Reality    | 증강현실    |
| MR | Mixed Reality        | 혼합현실    |
| AV | Augmented Virtuality | 증강 가상현실 |

---

### 10.2 현실-가상 연속체

현실과 가상은 다음과 같은 연속선 위에서 이해할 수 있다.

    실제 환경 → 증강현실 → 증강 가상현실 → 가상 환경

| 구분      | 설명                     |
|---------|------------------------|
| 실제 환경   | 현실 세계                  |
| 증강현실    | 실제 세계에 디지털 정보 결합       |
| 증강 가상현실 | 가상 환경에 실제 세계 정보를 일부 결합 |
| 가상 환경   | 컴퓨터로 만든 가상 세계          |
| 혼합현실    | 실제와 가상이 통합되어 상호작용      |

---

## 10.3 가상현실

가상현실(VR, Virtual Reality)은 실제 세계의 환경 또는 상상 속의 환경을 컴퓨터를 이용하여 시뮬레이션하는 기술이다.

| 특징 | 설명                    |
|----|-----------------------|
| 환경 | 실제 환경 또는 상상 속 환경      |
| 방식 | 컴퓨터 시뮬레이션             |
| 목적 | 사용자가 가상 공간에 있는 것처럼 경험 |
| 예  | 가상현실 환경을 이용한 비행훈련     |

---

## 10.4 증강현실

증강현실(AR, Augmented Reality)은 실제 세계의 정보에 디지털 정보를 결합하여 현실 환경 정보의 가치를 높이는 기술이다.

| 특징    | 설명                       |
|-------|--------------------------|
| 기반    | 실제 현실                    |
| 결합 정보 | 디지털 정보                   |
| 활용    | 모바일 단말기 카메라 영상에 부가 정보 결합 |
| 출력 방식 | 스마트폰, 안경, 투명 창 등에 정보 표시  |

예를 들어 카메라로 실제 환경을 비추면 화면 위에 길 안내, 설명, 3D 객체 등이 덧붙여 표시될 수 있다.

---

## 10.5 혼합현실

혼합현실(MR, Mixed Reality)은 가상의 세계와 실제 세계가 통합된 환경의 경험을 제공하는 기술이다.

| 특징   | 설명                     |
|------|------------------------|
| 환경   | 실제 세계와 가상 세계가 통합       |
| 객체   | 가상 객체와 실제 객체가 공존       |
| 상호작용 | 실시간 상호작용 가능            |
| 몰입감  | 사용자가 실제와 가상이 섞인 환경을 경험 |

---

## 10.6 몰입 기술

몰입 기술(immersive technology)은 실제와 유사한 감각을 제공하고 가상 환경과 상호작용할 수 있게 하는 기술이다.

### 실제와 유사한 감각을 제공하기 위한 하드웨어 기술

| 감각 | 기술         |
|----|------------|
| 시각 | HMD, 홀로그래피 |
| 청각 | 3차원 오디오    |
| 촉각 | 촉각 기술      |

### 가상 환경과 상호작용하기 위한 기술

| 기술       | 설명               |
|----------|------------------|
| 동작 인식    | 사용자의 움직임 인식      |
| 음성 인식    | 음성 명령 인식         |
| 전방향 트레드밀 | 가상 공간에서 걷는 경험 제공 |

## 10.7 메타버스

메타버스(metaverse)는 `meta`와 `universe`의 합성어이다.

| 구성       | 의미 |
|----------|----|
| meta     | 초월 |
| universe | 우주 |

메타버스는 가상과 현실이 상호작용하는 환경 안에서 직업, 금융, 문화, 교육 등의 활동이 이루어지는 세상이다.

메타버스에서는 아바타, 가상세계, 아이템 등을 표현하기 위해 멀티미디어 기술이 활용된다.

| 요소       | 설명                          |
|----------|-----------------------------|
| 아바타      | 사용자를 대신하는 가상 캐릭터            |
| 가상세계     | 사용자가 활동하는 디지털 공간            |
| 아이템      | 가상세계에서 사용하는 디지털 객체          |
| 멀티미디어 기술 | 3D 그래픽, 사운드, 애니메이션, 상호작용 기술 |

---

## 정리하기

1. 멀티미디어는 두 종류 이상의 매체가 통합되어 구성된 응용 또는 저작물을 의미한다.

2. 멀티미디어의 매체에는 텍스트, 그림, 음향, 동영상, 애니메이션 등이 있다.

3. 멀티미디어의 특징은 다수 매체의 유기적 통합, 디지털 방식 표현, 사용자와 미디어 사이의 상호작용성이다.

4. 비선형 콘텐츠는 사용자가 정해진 순서가 아니라 원하는 순서와 경로로 콘텐츠를 이용할 수 있는 콘텐츠이다.

5. 하이퍼텍스트는 하이퍼링크를 이용하여 서로 관련된 텍스트가 연결된 구조이다.

6. 하이퍼미디어는 하이퍼텍스트가 오디오, 비디오, 그래픽스 등을 포함하도록 확장된 형태이다.

7. 텍스트 표현에는 문자 인코딩, 문자 속성, 문단 정렬 정보가 필요하다.

8. 디지털 이미지는 래스터 방식과 벡터 방식으로 표현할 수 있다.

9. 래스터 이미지는 픽셀 배열로 그림을 표현하고, 벡터 이미지는 기하 도형으로 그림을 표현한다.

10. 래스터 이미지는 확대하면 거칠거나 흐릿해질 수 있지만, 벡터 이미지는 비교적 자연스럽게 확대할 수 있다.

11. BMP는 일반적으로 압축되지 않아 파일 크기가 큰 비트맵 이미지 파일 형식이다.

12. GIF는 최대 256색을 사용할 수 있고, 투명색과 간단한 애니메이션을 지원한다.

13. PNG는 무손실 압축을 사용하며, 256색 팔레트와 트루 컬러를 지원한다.

14. JPEG는 손실압축을 사용하며, 디지털 사진 저장에 유용하다.

15. 사운드는 음성, 음악, 음향효과 등 모든 종류의 소리를 의미한다.

16. ADC를 통해 녹음된 오디오 데이터에는 WAV, Au, MP3, MP4, WMA 등이 있고, 합성을 위한 오디오 데이터에는 MIDI가 있다.

17. 비디오는 데이터 양이 많기 때문에 MPEG, H.264, H.265 같은 손실압축기술을 사용한다.

18. 코덱은 부호기와 복호기를 의미하며, 재생 가능한 비디오 파일 형식은 코덱에 따라 달라진다.

19. 애니메이션은 일련의 이미지를 빠르게 보여 주어 움직이는 장면을 시뮬레이션하는 것이다.

20. 멀티미디어는 프레젠테이션, 원격회의, 원격교육, OTT, 웹 콘텐츠 등에 활용된다.

21. 확장현실은 가상현실, 증강현실, 혼합현실을 포괄하는 실감형 기술이다.

22. 가상현실은 실제 또는 상상 속 환경을 컴퓨터로 시뮬레이션하는 기술이다.

23. 증강현실은 실제 세계 정보에 디지털 정보를 결합하는 기술이다.

24. 혼합현실은 가상 세계와 실제 세계가 통합되어 실시간 상호작용하는 환경을 제공한다.

25. 메타버스는 가상과 현실이 상호작용하는 환경에서 직업, 금융, 문화, 교육 등의 활동이 이루어지는 세상이다.

### Q1. 멀티미디어의 특징으로 적절하지 않은 것은?

1. 두 종류 이상의 매체가 통합된다.
2. 디지털 방식으로 저장하고 처리할 수 있다.
3. 반복적으로 복제하면 품질이 반드시 저하된다.
4. 사용자와 미디어 사이에 상호작용이 존재할 수 있다.

정답: 3

해설:
디지털 방식으로 표현된 멀티미디어는 반복적으로 재생, 전송, 복제되더라도 품질이 저하되지 않는다.

1. 하이퍼텍스트

3. 메타버스
4. 래스터 이미지

정답: 1

하이퍼텍스트는 서로 연관된 텍스트의 집합이며, 하이퍼링크를 통해 다른 텍스트와 연결된다.

### Q3. 다음 중 래스터 이미지에 대한 설명으로 적절한 것은?

1. 점, 직선, 곡선 등 기하 도형으로 그림을 표현한다.
2. 이미지를 확대해도 항상 자연스럽게 확대된다.
3. 가로와 세로로 나열된 픽셀로 그림을 표현한다.
4. 데이터의 양은 이미지에 포함된 도형 수에 의해 결정된다.

정답: 3

해설:
래스터 이미지는 픽셀을 가로와 세로로 나열하여 그림을 표현한다. 확대하면 경계가 거칠어지거나 흐릿해질 수 있다.
