## 1. 인터넷 개요

### 1.1 인터넷이란?

인터넷은 전 세계적인 정보통신망이다.

- 여러 네트워크가 서로 연결된 구조
- “네트워크의 네트워크”
- INTERNET = INTERconnected NETwork
- ARPANET에서 출발
- TCP/IP 프로토콜을 기반으로 동작
- 패킷 교환 방식의 통신 프로토콜 사용

즉, 인터넷은 전 세계의 컴퓨터와 네트워크가 TCP/IP를 기반으로 서로 연결된 거대한 정보통신망이다.

---

### 1.2 인터넷의 역사

| 시기       | 내용                              |
|----------|---------------------------------|
| 1960년대 초 | 폴 배런(Paul Baran)의 생각에서 출발       |
| 1968년    | ARPA에서 ARPANET 계획               |
| 1969년    | ARPANET 구축                      |
| 1972년    | 국제통신학회에서 일반에 공개                 |
| 1973년    | 영국과 노르웨이 사이 국제적인 통신망 형성         |
| 1983년    | MILNET과 ARPANET으로 구분            |
| 이후       | TCP/IP를 적용한 NSFNET에 민간용 네트워크 추가 |

ARPANET은 인터넷의 시초이며, 미국 국방성 산하 ARPA에서 계획하고 구축한 네트워크이다.

---

### 1.3 인터넷의 특성

| 특성         | 설명                               |
|------------|----------------------------------|
| 개방 구조      | 프로토콜, 제도, 규격 등이 완전히 개방된 통신망      |
| 호스트 간의 평등성 | 인터넷에 연결된 모든 컴퓨터는 정보 송수신에서 대등한 위치 |
| 독자적인 주소 할당 | 인터넷에 연결된 모든 컴퓨터는 고유한 IP 주소를 가짐   |

시험에서는 “인터넷에 연결된 모든 컴퓨터가 다른 컴퓨터와 주소를 공유한다”는 설명은 부적절하다.  
인터넷에 연결된 모든 컴퓨터는 고유한 IP 주소를 가져야 한다.

---

## 2. 인터넷 주소

## 2.1 IP 주소

IP 주소(Internet Protocol Address)는 인터넷에 연결된 컴퓨터를 식별하기 위한 고유한 주소이다.

- 인터넷에 연결된 모든 컴퓨터는 고유한 IP 주소를 가짐
- 숫자로 구성됨
- 컴퓨터가 실제 통신할 때 사용하는 주소

예:

    203.232.172.32

---

### 2.2 IPv4

IPv4는 32비트 주소 체계이다.

| 구분      | 내용                       |
|---------|--------------------------|
| 주소 길이   | 32비트, 4바이트               |
| 표시 방법   | 8비트씩 4부분으로 나누고 점(.)으로 구분 |
| 각 부분 범위 | 0~255                    |
| 주소 개수   | 약 43억 개, 2^32개           |
| 예       | 203.232.53.55            |
| 문제점     | 주소 고갈 현상 발생              |

IPv4 주소는 일반적으로 다음과 같은 형태이다.

    203.232.172.32

---

### 2.3 IPv6

IPv6는 IPv4 주소 고갈 문제를 해결하기 위해 등장한 주소 체계이다.

| 구분    | 내용                         |
|-------|----------------------------|
| 주소 길이 | 128비트, 16바이트               |
| 표시 방법 | 16비트씩 8부분으로 나누고 콜론(:)으로 구분 |
| 주소 개수 | 약 3.4 × 10^38개, 2^128개     |
| 다른 이름 | IPng, IP next generation   |

IPv6 주소 예시는 다음과 같은 형태이다.

    2002:0221:ABCD:DCBA:0000:0000:FFFF:4002

---

### 2.4 IPv4와 IPv6 비교

| 구분    | IPv4              | IPv6               |
|-------|-------------------|--------------------|
| 주소 길이 | 32비트              | 128비트              |
| 크기    | 4바이트              | 16바이트              |
| 표시 방식 | 10진수 4부분, 점(.) 구분 | 16진수 8부분, 콜론(:) 구분 |
| 주소 개수 | 약 43억 개           | 약 3.4 × 10^38개     |
| 특징    | 주소 고갈 문제 발생       | 매우 많은 주소 제공        |

---

## 2.5 도메인 이름

도메인 이름(Domain Name)은 숫자로 된 IP 주소를 사람이 기억하기 쉬운 영문자로 바꾼 것이다.

| 구분     | 예              |
|--------|----------------|
| IP 주소  | 203.232.172.32 |
| 도메인 이름 | www.knou.ac.kr |

도메인 이름은 인터넷에 연결된 호스트 컴퓨터의 주소를 사람이 쉽게 기억할 수 있도록 만든 주소 표현 방식이다.

---

### 2.6 도메인 이름의 계층 구조

도메인 이름은 계층 구조를 가진다.

예:

    www.knou.ac.kr

| 부분   | 의미               |
|------|------------------|
| kr   | 국가 도메인, 대한민국     |
| ac   | 교육기관             |
| knou | 기관 이름            |
| www  | 호스트 이름 또는 서비스 이름 |

시험에서는 `.ac`가 나오면 교육기관을 의미한다고 기억한다.

---

## 2.7 DNS

DNS(Domain Name System)는 도메인 이름을 IP 주소로 변환해 주는 시스템이다.

사용자가 도메인 이름을 입력하면 DNS 서버는 해당 도메인 이름에 대응하는 IP 주소를 찾아 연결시켜 준다.

예:

    www.knou.ac.kr → 203.232.172.32

### DNS의 동작 흐름

1. 사용자가 도메인 이름을 입력한다.
2. DNS 서버에 해당 도메인의 IP 주소를 요청한다.
3. DNS 서버가 도메인 이름에 대응하는 IP 주소를 알려준다.
4. 사용자는 해당 IP 주소의 서버와 통신을 시작한다.

---

## 2.8 URL

URL(Uniform Resource Locator)은 인터넷에서 특정 전산 자원을 지정하는 주소를 알아보기 쉽게 표현하는 형식이다.

URL은 인터넷상의 특정 자원에 대한 주소이다.

### URL 형식

URL은 일반적으로 다음 요소로 구성된다.

| 구성 요소   | 설명                 |
|---------|--------------------|
| 프로토콜 이름 | 어떤 방식으로 접근할 것인지 표시 |
| 도메인 이름  | 접속할 서버의 이름         |
| 경로 이름   | 해당 파일의 위치나 자원 경로   |

형식:

    protocol://domain-name/path-or-file

### URL 예

| 구분      | 예                       |
|---------|-------------------------|
| 웹 사이트   | http://www.knou.ac.kr   |
| FTP 사이트 | ftp://ftp.kaist.ac.kr   |
| 텔넷 사이트  | telnet://home.hitel.net |

시험에서는 “인터넷에서 접근 가능한 특정 자원을 지정하는 주소”라고 나오면 URL을 떠올리면 된다.

## 3. 인터넷 서비스

## 3.1 전자우편

전자우편(e-mail, electronic mail)은 인터넷을 이용하여 컴퓨터에 저장한 메시지를 주고받을 수 있게 해 주는 정보통신 서비스이다.

| 구분   | 프로토콜       |
|------|------------|
| 보낼 때 | SMTP       |
| 받을 때 | POP3, IMAP |

### 전자우편 관련 프로토콜

| 프로토콜 | 원어                               | 역할                 |
|------|----------------------------------|--------------------|
| SMTP | Simple Mail Transfer Protocol    | 전자우편 송신            |
| POP3 | Post Office Protocol 3           | 전자우편 수신            |
| IMAP | Internet Message Access Protocol | 서버에 있는 메일을 관리하며 수신 |

시험에서는 “이메일을 보낼 때 주로 사용되는 프로토콜”이라고 나오면 SMTP를 고른다.

---

## 3.2 FTP

FTP(File Transfer Protocol)는 인터넷상에서 컴퓨터 간에 파일을 교환하기 위한 표준 프로토콜이다.

- TCP/IP 응용 프로토콜 중 하나
- 파일 업로드와 다운로드에 사용
- 원격 컴퓨터와 파일을 주고받는 서비스

시험에서는 “컴퓨터 간 파일을 교환하기 위한 표준 프로토콜”이라고 나오면 FTP를 떠올린다.

---

## 3.3 Telnet

Telnet(Telecommunication Network)은 인터넷에 연결되어 있는 컴퓨터를 원거리에서 접속하여 제어하거나 조작할 수 있는 정보통신 서비스이다.

- 원격 컴퓨터 접속
- 원격 제어 및 조작 가능
- 텍스트 기반 원격 접속 서비스

---

## 3.4 Usenet

Usenet은 “이용자 네트워크(user’s network)”를 의미한다.

- 인터넷 웹과 웹브라우저가 나타나기 훨씬 이전에 등장
- 세계의 수많은 사용자가 참여하는 토론 그룹의 모임
- 인터넷 사용자의 전자게시판 역할

---

## 3.5 월드 와이드 웹

월드 와이드 웹(WWW, World Wide Web)은 1989년 스위스의 유럽입자물리연구소(CERN)에서 제안한 광역정보시스템이다.

### WWW의 목적

네트워크상에 하이퍼텍스트(hypertext)를 구축하여 모든 정보를 끊임없이 액세스할 수 있게 하는 것이다.

### WWW 관련 개념

| 개념    | 설명                         |
|-------|----------------------------|
| WWW   | World Wide Web             |
| HTTP  | 하이퍼텍스트 파일 송수신용 응용 프로토콜     |
| HTML  | Hyper-Text Markup Language |
| 클라이언트 | 웹페이지를 요청하는 쪽               |
| 웹 서버  | 웹페이지를 제공하는 쪽               |

HTTP는 웹 서비스가 가능하도록 해 주는 프로토콜이다.

## 4. 인터넷 응용

인터넷은 정보검색, 전자상거래, 교육, 전화, 방송, 의료, SNS 등 여러 분야에 활용된다.

---

## 4.1 정보검색

정보검색은 검색엔진을 사용하여 인터넷상에서 사용자가 필요로 하는 정보를 찾는 행동이다.

### 검색엔진

검색엔진(search engine)은 사용자가 제시한 검색어를 포함하는 웹사이트를 검색하고, 그 결과를 제공해 주는 소프트웨어이다.

### 검색엔진의 구성요소

| 구성요소       | 설명                            |
|------------|-------------------------------|
| 인터페이스 프로그램 | 사용자가 검색어를 입력하고 결과를 확인하는 화면 제공 |
| 크롤링 프로그램   | 웹페이지를 수집하는 프로그램               |
| 색인 프로그램    | 수집된 정보를 검색 가능하게 정리            |

크롤링 프로그램은 crawler, spider, bot이라고도 부른다.

---

## 4.2 전자상거래

전자상거래는 생산자, 중개인, 소비자가 디지털 통신망을 이용하여 상호 거래하는 시장이다.

### 전자상거래의 의미

| 구분    | 설명                                             |
|-------|------------------------------------------------|
| 좁은 의미 | 인터넷 홈페이지로 개설된 상점을 통해 실시간으로 상품을 거래하는 것          |
| 넓은 의미 | 공급자, 소비자, 금융기관, 정부기관 등 거래 관련 모든 기관의 전자적 상거래 행위 |

---

### 4.2.1 전자상거래 유형

| 유형     | 원어                   | 설명           |
|--------|----------------------|--------------|
| B to B | Business to Business | 기업 간 거래      |
| B to C | Business to Consumer | 기업과 소비자 간 거래 |
| C to C | Consumer to Consumer | 소비자 간 거래     |

#### B to B

- 기업 간 거래
- 파는 쪽과 사는 쪽이 웹사이트 등을 사용하여 거래
- 전자시장 형태

#### B to C

- 기업과 소비자 간 거래
- 웹서비스를 통해 소비자에게 제품이나 디지털 콘텐츠 판매
- 전자상점 형태

#### C to C

- 소비자 간 거래
- 인터넷에서 소비자끼리 물건을 사고파는 형태

---

## 4.3 e-Learning

e-Learning은 ICT를 활용하여 언제, 어디서나, 누구나 원하는 수준별 맞춤형 학습을 할 수 있는 교수학습 방법이다.

- 정보통신기술을 활용한 모든 형태의 학습
- 시간과 장소의 제약을 줄임
- 수준별 맞춤형 학습 가능

---

## 4.4 인터넷 전화

인터넷 전화는 VoIP(Voice over Internet Protocol)를 활용하여 일반 전화망에서 이루어졌던 음성 서비스를 IP망 기반의 패킷 데이터로 구현하는 통신기술이다.

### 인터넷 전화의 특징

| 특징       | 설명                      |
|----------|-------------------------|
| IP망 이용   | 음성을 패킷 데이터로 변환하여 전송     |
| 동시 사용 가능 | 케이블을 통해 여러 명이 동시에 사용 가능 |
| 확장성      | 서비스 확장이 쉬움              |
| 저렴한 요금   | 기존 전화에 비해 요금이 저렴함       |

시험에서는 인터넷 전화와 가장 가까운 개념은 VoIP이다.

---

## 4.5 IPTV

IPTV는 Internet Protocol TV의 약자이다.

인터넷 프로토콜(IP)을 사용하여 소비자에게 디지털 텔레비전 서비스를 제공하는 시스템이다.

### IPTV의 특징

- 인터넷 프로토콜을 이용한 TV 서비스
- 시청자가 편리한 시간에 원하는 프로그램을 선택해서 볼 수 있음
- VOD(Video on Demand) 제공
- PC 모니터 대신 텔레비전 사용
- 마우스 대신 리모컨 사용
- TV 방송의 주도권이 시청자에게 이동

---

### 4.5.1 OTT 서비스

OTT service(over-the-top media service)는 인터넷을 통해 방송 프로그램, 영화, 교육 등 각종 미디어 콘텐츠를 제공하는 서비스이다.

| 구분   | 설명                        |
|------|---------------------------|
| IPTV | 인터넷 프로토콜을 이용한 TV 서비스      |
| OTT  | 인터넷을 통해 미디어 콘텐츠를 제공하는 서비스 |

---

## 4.6 원격진료

원격진료는 환자의 위치와 관계없이 환자의 정보 및 전문적인 지식을 제공하는 시스템을 통해 환자의 감시, 검사, 관리 및 교육을 시행하는 것이다.

또한 환자나 의료진이 위치나 상호 간 거리의 제약을 받지 않고 진료할 수 있는 시스템이다.

### 원격진료의 핵심

- 환자 정보 전송
- 실시간 감시
- 검사 및 관리
- 의료진과 환자 사이의 원격 진료
- 위치와 거리의 제약 완화

---

## 4.7 SNS

SNS는 Social Network Service 또는 social media를 의미한다.

온라인 인맥구축 서비스로, 1인 미디어, 1인 커뮤니티, 정보공유 등을 포괄하는 개념이다.

- 인맥을 넓히기 위해 개설된 웹사이트
- 개인 간 관계 형성
- 정보 공유
- 커뮤니티 활동
- 1인 미디어 역할

예:

- Twitter
- Facebook
- YouTube
- Instagram
- LINE
- KakaoTalk

## 정리하기

1. 인터넷은 네트워크의 네트워크로, 미국 국방성에서 추진한 ARPANET에서 기원을 찾을 수 있다.

2. TCP/IP 프로토콜을 바탕으로 인터넷이 시작되었으며, ARPANET 호스트들은 TCP/IP 프로토콜을 사용한 인터넷 환경에서의 상호 통신이 이루어지게 되었다.

3. 인터넷에 연결된 모든 컴퓨터는 고유한 IP 주소를 가진다.

4. IP 주소는 일반적으로 숫자로 구성되어 있고, 도메인 이름은 사람이 기억하기 쉬운 문자 주소이다.

5. 도메인 이름을 실제 IP 주소로 변환해 주는 시스템은 DNS이다.

6. URL은 인터넷상의 특정 자원에 대한 주소를 계층적으로 표현한 형식이다.

7. 전자우편은 인터넷을 이용하여 메시지를 주고받는 서비스이며, 보낼 때는 SMTP, 받을 때는 POP3 또는 IMAP을 사용한다.

8. FTP는 인터넷상에서 컴퓨터 간 파일을 교환하기 위한 표준 프로토콜이다.

9. Telnet은 인터넷에 연결된 컴퓨터를 원거리에서 접속하여 제어하거나 조작할 수 있는 서비스이다.

10. Usenet은 인터넷 사용자의 전자게시판 성격을 가진 토론 그룹의 모임이다.

11. WWW는 하이퍼텍스트를 기반으로 인터넷상의 정보를 끊임없이 접근할 수 있게 하는 광역정보시스템이다.

12. 검색엔진은 사용자가 입력한 검색어에 따라 정보를 찾는 데 활용되며, 크롤링 프로그램과 색인 프로그램 등을 포함한다.

13. 전자상거래는 디지털 통신망을 이용하여 상거래를 수행하는 것이며, B2B, B2C, C2C 유형이 있다.

14. e-Learning은 ICT를 활용한 학습 방식으로, 시간과 장소에 구애받지 않고 학습할 수 있다.

15. 인터넷 전화는 VoIP 기술을 활용하여 IP망을 기반으로 음성 통화를 구현한다.

16. IPTV는 IP를 이용하여 디지털 텔레비전 서비스를 제공하며, VOD를 통해 원하는 시간에 원하는 프로그램을 선택할 수 있다.

17. 원격진료는 거리 제약 없이 환자와 의료진이 진료할 수 있도록 지원하는 시스템이다.

18. SNS는 온라인 인맥구축 서비스로, 1인 미디어, 1인 커뮤니티, 정보공유 등을 포함한다.

## 연습문제 정리

### Q1. 인터넷(Internet)에 관한 설명으로 부적절한 것은?

1. 인터넷의 어원은 Interconnected network이다.
2. 미국의 국방용 네트워크인 ARPANET이 전신이다.
3. 인터넷은 TCP/IP 프로토콜을 바탕으로 개발되었다.
4. 인터넷에 연결되는 모든 컴퓨터는 다른 컴퓨터와 주소를 공유해서 사용한다.

정답: 4

해설:
인터넷에 연결된 모든 컴퓨터는 고유한 IP 주소를 가져야 한다. 다른 컴퓨터와 주소를 공유해서 사용한다는 설명은 부적절하다.

1. 교육기관
2. 정부기관
3. 공공기관
4. 상업기관

정답: 1

해설:
도메인 이름에서 ac는 교육기관을 의미한다.

### Q3. 일반적으로 홈페이지의 주소는 문자로 구성된 주소인 (    )과 숫자로 구성된 주소인 (    )이 있다. 괄호 안에 순서대로 적합한 것은?

1. URL, 도메인
2. 도메인, IP 주소
3. URL, DNS
4. DNS, IP 주소

정답: 2

해설:
문자로 구성된 주소는 도메인 이름이고, 숫자로 구성된 주소는 IP 주소이다.

---

### Q4. 다음 중 도메인 이름을 IP 주소로 변환해 주는 것은?

1. POP3
2. TCP
3. DNS
4. FTP

정답: 3

해설:
DNS는 도메인 이름과 그에 대응하는 IP 주소를 저장한 데이터베이스를 이용하여 도메인 이름을 IP 주소로 변환해 준다.

---

### Q5. 다음 중 인터넷에서 접근 가능한 특정 자원을 지정하는 주소를 의미하는 것은?

1. Domain
2. IP 주소
3. DNS
4. URL

정답: 4

해설:
URL은 인터넷에서 특정 전산 자원을 지정하는 주소를 알아보기 쉽게 표현한 형식이다.

---

### Q6. 다음 중 이메일을 보낼 때 주로 사용되는 프로토콜은?

1. SMTP
2. DBMS
3. CRM
4. FTP

정답: 1

해설:
전자우편을 보낼 때는 SMTP(Simple Mail Transfer Protocol)를 사용한다. POP3와 IMAP은 주로 메일을 받을 때 사용한다.

---

### Q7. 다음 중 월드 와이드 웹(WWW) 서비스가 가능하도록 해 주는 프로토콜은?

1. FTP
2. POP3
3. HTML
4. HTTP

정답: 4

해설:
HTTP는 하이퍼텍스트 파일 송수신용 응용 프로토콜로, 월드 와이드 웹 서비스가 가능하도록 해 준다.

---

### Q8. 다음 중 인터넷 전화 서비스와 관계가 가장 깊은 것은?

1. VoIP
2. IPTV
3. WWW
4. Search engine

정답: 1

해설:
인터넷 전화는 VoIP(Voice over Internet Protocol)를 활용하여 IP망 기반으로 음성 통화를 구현하는 통신기술이다.
